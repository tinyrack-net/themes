# Overlay

Native Modal and anchored Layer primitives with one document-level stack.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `dialog.tr-modal` | `Modal`, `ModalContent` | Native dialog | Use for blocking decisions and focused tasks. |
| Part | `tr-modal-box`, `tr-modal-backdrop` | `ModalBox`, generated backdrop | Required | Keep the box and backdrop as dialog children. |
| Part | `tr-modal-header`, `tr-modal-title`, `tr-modal-description`, `tr-modal-body`, `tr-modal-action` | Matching compound parts | Optional | Prefer these parts for accessible labels and long content. |
| Variant | `data-placement` | `placement` | `middle` (default), `top`, `bottom`, `start`, `end` | Start and end follow logical RTL direction. |
| Size | `data-size` | `size` | `sm`, `md` (default), `lg`, `full` | Utility width classes can override convenience sizing. |
| Component | `[popover].tr-layer` | `Layer`, `LayerContent` | `auto`, `manual`, `hint` | Use for anchored, non-blocking content. |
| Behavior | `data-close-on-escape`, `data-close-on-backdrop` | `closeOnEscape`, `closeOnBackdrop` | Both `true` | Disable dismissal only when the task provides an explicit safe exit. |
| Native | `commandfor`, `popovertarget` | Trigger and close compounds | Native attributes | HTML remains usable before a framework hydrates. |

          </ModalHeader>
          <ModalDescription>Update this rack configuration.</ModalDescription>

            <ModalClose asChild>

            </ModalClose>
            <Button variant="primary">Save</Button>
          </ModalAction>
        </ModalBox>
      </ModalContent>
    </Modal>
  )}
  sources={[
    {
      label: 'React',
      language: 'tsx',
      code: String.raw`import '@tinyrack/ui/core/core.css';
import '@tinyrack/ui/components/overlay/overlay.css';
import {
  Modal,
  ModalBox,
  ModalContent,
  ModalTitle,
  ModalTrigger,
} from '@tinyrack/ui/components/overlay/react';

<Modal>
  <ModalTrigger>Open settings</ModalTrigger>
  <ModalContent>
    <ModalBox>
      <ModalTitle>Rack settings</ModalTitle>
    </ModalBox>
  </ModalContent>
</Modal>;`,
    },
    {
      label: 'HTML',
      language: 'html',
      code: String.raw`<button commandfor="rack-settings" command="show-modal">
  Open settings
</button>
<dialog
  id="rack-settings"
  class="tr-modal"
  data-tr-overlay="modal"
  data-placement="middle"
  data-close-on-escape="true"
  data-close-on-backdrop="true"
  closedby="any"
  aria-labelledby="rack-settings-title"
>
  <div class="tr-modal-box" data-size="md">
    <h2 id="rack-settings-title" class="tr-modal-title">Rack settings</h2>
  </div>
  <form method="dialog" class="tr-modal-backdrop">
    <button aria-label="Close settings">close</button>
  </form>
</dialog>`,
    },
  ]}
/>

## Guidance

<ComponentGuidance
  whenToUse={[
    'Use Modal for blocking decisions, destructive confirmations, and focused tasks that require inert page content.',
    'Use Layer for menus, inspectors, and anchored actions that should not make the page inert.',
  ]}
  avoid={[
    'Do not use a Layer when the interaction requires Modal focus and inert semantics.',
    'Do not open unrelated stacked Modals; reserve stacking for a direct child task.',
  ]}
  accessibility={[
    'Label every Modal with ModalTitle or an explicit aria-label.',
    'Keep a visible close path when Escape or backdrop dismissal is disabled.',
    'Place long scrolling content in ModalBody so the title and actions remain available.',
  ]}
/>

## API

| Prop | Values | Note |
| --- | --- | --- |
| `open`, `defaultOpen` | `boolean` | Choose controlled or uncontrolled state; SSR never emits `dialog[open]`. |
| `onOpenChange` | `(open, detail) => void` | Reports trigger, dismissal, cascade, and native reasons. |
| `placement` | `middle`, `top`, `bottom`, `start`, `end` | Modal viewport placement. |
| `size` | `sm`, `md`, `lg`, `full` | Convenience Modal box size. |
| `closeOnEscape`, `closeOnBackdrop` | `boolean` | Both default to `true`. |
| `mode` | `auto`, `manual`, `hint` | Native Layer popover mode. |
| Layer positioning | placement, offset, collision padding, anchor width | Powered by Floating UI DOM. |
| native props | dialog, div, button attributes | Passed through to compound primitives. |
