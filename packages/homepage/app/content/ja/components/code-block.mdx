## API

highlighting は hydration 後に実行されます。元の source は SSR-safe で、Shiki の読み込み前にも表示されます。ハイライトされた block は `tinyrack-light` では `github-light-high-contrast`、`tinyrack-dark` では `github-dark-high-contrast` を自動で使います。`code` や `language` を変更すると古い処理をキャンセルし、以前の async 結果が現在の source を置き換えないようにします。

| Prop | 型 | 既定値 | 説明 |
| --- | --- | --- | --- |
| `code` | `string` | 必須 | semantic な `<pre><code>` 内に表示する source です。 |
| `language` | `BundledLanguage` | `undefined` | 段階的な Shiki highlighting を有効にします。省略時は plain text です。 |
| `wrap` | `boolean` | `false` | 水平 overflow の代わりに長い行を折り返します。 |
| `ref` | `Ref<HTMLPreElement>` | — | native `pre` 要素を参照します。 |
| `className`, `style` | native `pre` props | — | consumer class、inline style、component token override を適用します。 |

その他の native `pre` attribute と event も渡されます。周囲の文脈で source を識別できない場合は `aria-label` を追加してください。highlighting は accessible な原文を変えず、language の読み込みに失敗しても SSR fallback は維持されます。

## スタイル

| Token | 用途 |
| --- | --- |
| `--tr-code-block-background` | highlighted block を含む surface |
| `--tr-code-block-border` | border color |
| `--tr-code-block-border-width` | border width |
| `--tr-code-block-color` | fallback source color |
| `--tr-code-block-font-family` | monospace font family |
| `--tr-code-block-font-size` | source size |
| `--tr-code-block-line-height` | source line height |
| `--tr-code-block-padding-x` | inline padding |
| `--tr-code-block-padding-y` | block padding |
| `--tr-code-block-radius` | corner radius |

`wrap={false}` では長い行を水平 overflow で表示します。`wrap` では長い token も利用可能な幅で折り返し、どちらの mode も parent の境界内に収まります。
