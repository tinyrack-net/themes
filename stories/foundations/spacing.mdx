# Spacing

<p className="tr-doc-lead">
  A relationship scale for compact screens where proximity explains hierarchy.
</p>

## Principle

Space should describe whether UI belongs together, repeats as a row, or starts a new
task. Keep operational surfaces tight by default, then spend larger stops only at real
section changes.

| Relationship | Range | Rule |
| --- | --- | --- |
| Detail | `xs - sm` | Repeated elements must stay visually connected. |
| Component | `md - lg` | Normal rhythm inside forms, panels, and control groups. |
| Section | `xl - 2xl` | Use only when the task or reading group changes. |

## Relationship scale

| Token | Value | Relationship | Use |
| --- | --- | --- | --- |
| `xs` | `0.25rem` | Detail | Status dots, icon-to-label gaps, tiny affordance padding. |
| `sm` | `0.5rem` | Compact | Toolbar controls, table cells, repeated inline actions. |
| `md` | `0.75rem` | Default | Default row rhythm, panel internals, field groups. |
| `lg` | `1rem` | Component | Panel padding, form blocks, larger action areas. |
| `xl` | `1.5rem` | Section | Related groups that need a scan-friendly break. |
| `2xl` | `2rem` | Layout | Page-level separation and major task changes. |
