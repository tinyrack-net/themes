# Controls

One size recipe keeps actions, fields, and navigation aligned across the component system.

## Principle

Use `sm` for dense secondary actions, `md` for the default operational interface, and `lg` only when touch comfort or emphasis requires it.

## Visual scale

<div className="grid gap-tinyrack-md" role="list">
  {['sm', 'md', 'lg'].map((size) => (

## Applied pattern

<section className="grid gap-tinyrack-lg border border-tinyrack-border p-tinyrack-lg">
  <div className="grid gap-tinyrack-sm md:grid-cols-[minmax(0,1fr)_auto]"><Field.Root><Field.Control aria-label="Rack name" placeholder="rack-a" /></Field.Root><Button variant="primary">Save rack</Button></div>
  <div className="text-tinyrack-sm text-tinyrack-text-muted">Both controls use the same 40px medium height and typography recipe.</div>
</section>

## Implementation

Override the global metric when an entire product needs a different density, or a `--tr-*` recipe token when one component needs a deliberate exception.

```css
:root {
  --tinyrack-control-height-md: 2.5rem;
}

.rack-toolbar {
  --tr-btn-height: 2.25rem;
}
```
