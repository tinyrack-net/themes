## Contract

Previous and next destinations are data. `renderLink` injects a router element without coupling the component to routing state. The landmark renders only when at least one destination exists.

| Axis | Contract |
| --- | --- |
| direction | previous links expose `rel="prev"`; next links expose `rel="next"` |
| availability | omit a missing destination; set its `disabled` field when it must remain visible but unavailable |
| routing | `renderLink(destination, direction)` composes a router link without nested anchors |
| events | `onNavigate(destination, direction)` runs only for enabled links |

## API

`previous` and `next` accept `{ path, title, label?, description?, disabled? }`. Direction text and accessible labels are localizable with `previousLabel`, `nextLabel`, `previousAriaLabel`, `nextAriaLabel`, and landmark `label`. `renderLink` receives the destination and `previous | next` direction; use it for React Router or another client router. `onNavigate` receives the same values after an enabled link is activated.

The root preserves native `nav` attributes, `className`, `style`, events, and React 19 `ref`. Cards wrap long content and switch from one column to two at the responsive breakpoint; a lone next link occupies the second desktop column. Styles use Tinyrack semantic color, spacing, typography, focus, opacity, border, and motion tokens.
