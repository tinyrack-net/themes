## Contract

`TRDocsNavigation` renders recursive `group`, `page`, and `link` items in a labelled navigation landmark. Groups containing the current or pending path open automatically, including after route state changes. Users can still disclose other groups with pointer or keyboard input.

## Accessibility

The root is a `<nav>` named by `label`. Current pages receive `aria-current="page"`; pending links expose `data-pending` and a decorative spinner. Group triggers are native buttons supplied by Collapsible and support `Enter`, `Space`, and normal tab navigation. Keep item labels descriptive and localize `label` when the default `Documentation` name is unsuitable.

## API

### `TRDocsNavigation`

| Prop | Type | Default | Purpose |
| --- | --- | --- | --- |
| `items` | `readonly TRDocsNavigationItem[]` | Required | Recursive navigation data. |
| `currentPath` | `string` | Required | Exact page path marked current. |
| `pendingPath` | `string` | `undefined` | Exact destination path marked pending. |
| `defaultGroupsOpen` | `boolean` | `false` | Initially opens groups that do not contain the current or pending path. |
| `label` | `string` | `'Documentation'` | Accessible name for the navigation landmark. |
| `renderLink` | `(item, state) => ReactElement` | Native anchor | Injects a framework router link; `state` contains `active` and `pending`. |
| `onNavigate` | `(item) => void` | `undefined` | Runs after a page or link is activated, for example to close a mobile drawer. |

`TRDocsNavigation` also forwards native `nav` props, `className`, `style`, events, data attributes, and a React 19 `ref`.

### `TRDocsNavigationItem`

`page` items have `label`, `path`, and `type: 'page'`. `link` items add optional `external`; use that metadata in `renderLink` when external destinations need a target or external-link treatment. `group` items have `label`, `type: 'group'`, and recursive `children`.

The exported data types are `TRDocsNavigationItem`, `TRDocsNavigationGroup`, `TRDocsNavigationPage`, and `TRDocsNavigationLink`. `TRDocsNavigationLinkState` contains the `active` and `pending` booleans passed to `renderLink`; `TRDocsNavigationProps` is the complete component prop type.

## Styling

Import the component stylesheet after core tokens. Useful overrides include `--tr-docs-navigation-gap`, `--tr-docs-navigation-item-gap`, `--tr-docs-navigation-group-color`, `--tr-docs-navigation-link-color`, `--tr-docs-navigation-link-hover-background`, `--tr-docs-navigation-link-hover-color`, and `--tr-docs-navigation-link-active-border-color`. Long labels wrap inside narrow sidebars and mobile drawers.

## SSR

Server output includes the current path and initially open ancestor groups. Hydrate with the same `items`, `currentPath`, and `pendingPath`; route changes after hydration open newly relevant groups without replacing the navigation tree.
