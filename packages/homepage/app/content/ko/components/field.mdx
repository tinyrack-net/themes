## 계약

`TRField`는 Base UI Field의 전체 anatomy를 래핑합니다. `Label`, `Description`, 표시 중인 `Error`는 control에 자동으로 연결됩니다. Root의 `name`과 `disabled`는 Control의 같은 prop보다 우선합니다.

| Part | 역할 |
| --- | --- |
| `Root` | name, 사용 가능 상태, validation 시점, dirty/touched/filled/focused state |
| `Label` | accessible name과 click-to-focus target |
| `Control` | native input value, event, validity, form data, reset |
| `Description` | `aria-describedby`에 포함되는 지속적인 안내 |
| `Error` | native validity 또는 외부 state에 따른 복구 메시지 |
| `Validity` | value, errors, native `ValidityState`를 읽는 render prop |
| `Item` | collection의 checkbox/radio와 label/description 배치 |

값을 focus 가능한 상태로 제출해야 하면 Control의 `readOnly`를 사용합니다. 사용할 수 없고 form data에서도 제외해야 하면 Root의 `disabled`를 사용합니다. 표준 part를 조합할 때는 `aria-labelledby`나 `aria-describedby`를 직접 추가하지 않습니다.

## API

### `TRField.Root`

| Prop | 타입 / default | 용도 |
| --- | --- | --- |
| `name` | `string` | field 이름을 지정하며 `Control.name`보다 우선 |
| `disabled` | `boolean` / `false` | control을 비활성화하고 모든 part에 `data-disabled` 부여 |
| `invalid`, `dirty`, `touched` | `boolean` | 외부에서 관리하는 field state |
| `validate` | sync/async validator | error, error 배열 또는 `null` 반환 |
| `validationMode` | `'onSubmit' \| 'onBlur' \| 'onChange'` / `'onSubmit'` | validation 시점 |
| `validationDebounceTime` | `number` / `0` | onChange validation debounce(ms) |
| `actionsRef` | `RefObject<TRFieldRootActions>` | `validate()` 노출 |
| `uiSize` | `'sm' \| 'md' \| 'lg'` / `'md'` | visual control size |

`Control`은 native input props, `value` / `defaultValue`, `onValueChange(value, details)`를 받습니다. `Label.nativeLabel={false}`는 `render`가 label이 아닌 host를 사용할 때 지정합니다. `Error.match`는 `true` 또는 `valueMissing` 같은 `ValidityState` key를 받고, `Item.disabled`는 단일 option을 비활성화하며, `Validity`는 `validity`, `value`, `initialValue`, `error`, `errors`를 제공합니다.

모든 host part는 ref, native attribute/event, 함수형 `className` / `style`, Base UI `render`를 보존합니다. namespace, named part 7개, 각 props/state type, `TRFieldUiSize`, `TRFieldRootActions`, `TRFieldValidityData`, Control change-event type을 모두 개별 export합니다. CSS는 opt-in입니다.
