## コントラクト

`Main` はページ内で唯一の main landmark を担当します。選択した breakpoint 以上では `Sidebar` が固定され、独立してスクロールします。`sidebarMode` で expanded と icon rail を切り替え、`SidebarLabel` はどちらの状態でもナビゲーション名をアクセシブルに保ちます。breakpoint 未満では `mobileSidebar="drawer"` が従来の Base UI TRDrawer を維持し、`mobileSidebar="rail"` が inline 固定 rail を有効にします。

## API

`Root` は controlled の `sidebarMode`、uncontrolled の `defaultSidebarMode`（既定値 `expanded`）、`onSidebarModeChange` を提供します。安定した契約は `data-sidebar-mode`、`data-mobile-sidebar`、`--tr-app-shell-sidebar-width`、既定値 `4rem` の `--tr-app-shell-sidebar-rail-width` です。既定の `mobileSidebar="drawer"` は modal 内で常に label を展開し、`mobileSidebar="rail"` は Drawer・Backdrop・Close・Trigger なしで同じナビゲーションを inline 表示します。`SidebarToggle` は `aria-expanded` を持つ ghost `sm` IconButton で、`SidebarLabel` は rail でも削除されず視覚的にのみ隠れます。
