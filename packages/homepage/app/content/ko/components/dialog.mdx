## 계약

| 축 | 값 | 기본값 |
| --- | --- | --- |
| `placement` on `Popup` | `middle`, `top`, `bottom`, `start`, `end` | `middle` |
| `modal` on `Root` | focus trap and scroll lock when true | `true` |
| state | controlled `open` or uncontrolled `defaultOpen` | closed |

## API

Base UI는 포커스 트랩, 스크롤 잠금, 포털, 배경 상호작용, 포커스 복원, Escape 닫기를 담당합니다. `Popup`은 모든 배치에서 하나의 반응형 너비 규칙을 사용합니다. 긴 작업 콘텐츠는 Description과 작업 행 사이의 `tr-dialog-body` 컨테이너에 배치하세요. 그러면 팝업의 `minmax(0, 1fr)` 본문만 스크롤되고 제목과 작업은 계속 표시됩니다.

| 공개 API | 용도 |
| --- | --- |
| `TRDialog.Root` / `TRDialogRoot` | 제어형 `open`, 비제어형 `defaultOpen`, 모달 동작, 닫기, 선택적 handle payload 상태를 관리합니다. |
| `TRDialog.Trigger` / `TRDialogTrigger` | root를 열거나 handle과 선택적 payload를 사용해 분리된 root를 엽니다. |
| `TRDialog.Portal` / `TRDialogPortal` | 레이어를 `document.body` 또는 사용자 지정 `container`로 이동합니다. |
| `TRDialog.Backdrop` / `TRDialogBackdrop` | 모달 배경과 외부 누르기 대상을 제공합니다. |
| `TRDialog.Viewport` / `TRDialogViewport` | 긴 콘텐츠와 모바일 dialog 레이아웃을 포함합니다. |
| `TRDialog.Popup` / `TRDialogPopup` | 이름이 지정된 `dialog`를 렌더링하고 `initialFocus`, `finalFocus`, Tinyrack `placement`를 받습니다. |
| `TRDialog.Title` / `TRDialogTitle` | 접근 가능한 이름을 제공합니다. |
| `TRDialog.Description` / `TRDialogDescription` | 접근 가능한 설명을 제공합니다. |
| `TRDialog.Close` / `TRDialogClose` | 활성 dialog를 닫고 포커스를 복원합니다. |
| `TRDialog.createHandle`, `createDialogHandle`, `TRDialogHandle<Payload>` | 분리된 trigger, 명령형 열기/닫기 호출, 타입이 지정된 payload를 연결합니다. |

외부 상호작용을 유지하려면 `modal={false}`를 사용하고, 스크롤 잠금 없이 포커스만 가두려면 `modal="trap-focus"`를 사용하세요. `disablePointerDismissal`은 외부 누르기로 dialog가 닫히지 않게 합니다. 모든 part는 ref, `render`, 이벤트 핸들러, class, style, 네이티브 속성을 보존합니다. 닫힌 portal은 SSR에 안전하며 기본으로 열린 portal도 공개 구조 변경 없이 hydrate됩니다.
