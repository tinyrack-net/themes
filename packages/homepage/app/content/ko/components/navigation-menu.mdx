## 계약

TRNavigationMenu는 브랜드, 소수의 직접 링크, 선택적인 Product 또는 Resources flyout이 있는 사이트 헤더에 사용합니다. 데스크톱 구성은 Base UI의 roving keyboard navigation을 유지하고, 모바일 구성은 같은 목적지를 TRDrawer로 제공합니다.

## API

전체 anatomy는 `Root`, `List`, `Item`, `Trigger`, `Icon`, `Content`, `Portal`, `Backdrop`, `Positioner`, `Popup`, `Arrow`, `Viewport`, `Link`입니다. 각 part, props type, state type은 named export되며 해당 part는 `TRNavigationMenu` member로도 제공됩니다.

- `Root`: controlled open state에는 `value`와 `onValueChange`, uncontrolled state에는 `defaultValue`를 사용합니다. `orientation`은 roving focus key와 layout을 함께 바꾸며, `delay`와 `closeDelay`는 pointer intent를 조정합니다.
- `Link`: 현재 destination에는 `active`(`aria-current="page"`)를 설정하고 flyout 안의 link에는 필요에 따라 `closeOnClick`을 설정합니다.
- `Positioner`: `side`, `align`, offset, collision padding, `collisionAvoidance`로 배치를 제어합니다. Tinyrack default는 popup을 viewport 안에 유지하도록 align을 shift하고 side를 flip합니다.
- `Portal`: custom container를 지정할 수 있습니다. 내부에 `Backdrop`과 `Positioner`를 두며, `Popup`이 visual surface를 소유하고 `Viewport`가 active `Content`를 받습니다.
- style: `@tinyrack/ui/components/navigation-menu.css`를 import합니다. `--tr-navigation-menu-color`, `--tr-navigation-menu-control-background`, `--tr-navigation-menu-popup-background`, `--tr-navigation-menu-popup-max-width`, `--tr-navigation-menu-backdrop` custom property를 사용할 수 있습니다.
- SSR: server render와 hydration을 지원합니다. portal content는 hydration 후 container에 mount됩니다.

TRNavigationMenu는 충분한 가로 공간이 있는 pointer·keyboard 사이트 탐색을 위한 것입니다. flyout group을 task 중심으로 유지하고 preview가 필요 없는 목적지에는 direct link를 보존하세요. 좁은 breakpoint에서는 horizontal keyboard model을 압축하지 말고 같은 destination set을 가진 TRDrawer 조합으로 바꾸세요.
