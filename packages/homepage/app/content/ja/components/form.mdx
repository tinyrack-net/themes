## 主なプロパティ

| 項目 | 説明 |
| --- | --- |
| 送信 | ネイティブの `submit` イベントと `FormData`、または `onFormSubmit` で解析されたフィールド値を使用します。 |
| 検証 | ネイティブ validity と、Base UI の `TRField.Error` コンポジションで表示する外部 `errors` を組み合わせます。 |
| リセット | ネイティブの `type="reset"` は非制御値を復元します。制御値とアプリケーション結果は `onReset` でリセットします。 |
| 状態 | control は非制御またはアプリケーション制御で利用でき、server error と送信結果はアプリケーションが所有します。 |

function RequiredRackForm() {
  const [attempted, setAttempted] = useState(false);
  const [saved, setSaved] = useState('');
  const [value, setValue] = useState('');
  const invalid = attempted && value.trim().length === 0;

        aria-invalid={invalid || undefined}
        id="required-rack"
        name="rack"
        onChange={(event) => {
          setValue(event.currentTarget.value);
          setSaved('');
        }}
        required
        value={value}
      />
      {invalid ? <p id="required-rack-error" role="alert">保存する前にラック名を入力してください。</p> : null}
      <TRButton type="submit">ラックを保存</TRButton>
      <output aria-live="polite">{saved ? saved + ' を保存しました。' : ''}</output>
    </TRForm>
  );
}` ,

## API

`TRForm` はネイティブの form プロパティ、`ref`、イベント、`className`、`style`、`render` を転送し、各コントロールは自身の値を管理します。`TRFormProps<Values>` は `onFormSubmit(values, details)` を通じてフィールド値のジェネリックを保持し、イベント詳細には `TRFormSubmitEventDetails` と `TRFormSubmitEventReason` を利用できます。サーバーの応答にはフィールド名と `errors` を使い、`validationMode` は TRForm または TRField のスコープで選びます。送信を発生させずに明示的に検証するには `TRFormActions.validate(name?)` を使います。ネイティブのリセットで復元されるのは非制御のコントロールだけなので、制御値はアプリケーションの `onReset` ハンドラーでリセットしてください。
