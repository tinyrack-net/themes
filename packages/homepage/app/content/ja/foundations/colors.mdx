## セマンティックロールを選ぶ

色の値ではなく、意味から決めてください。

- **表面:** キャンバス、表面、控えめな表面、ホバー状態、選択状態、押下状態は、レイヤーと操作を表します。それぞれ `canvas`、`surface`、`surfaceMuted`、`surfaceHover`、`surfaceSelected`、`surfacePressed` トークンを使います。
- **内容:** 基本文字、補助文字、プレースホルダー、境界線、反転の各ロールは、前景色を文脈に結び付けます。`text`、`textMuted`、`textPlaceholder`、`border`、`surfaceInverse` など、意味に合うトークンを選んでください。
- **操作:** 主操作の通常、ホバー、押下、前景色は、`primary`、`primaryHover`、`primaryPressed`、`onPrimary` で 1 つのセットになります。
- **ステータス:** 情報、成功、警告、危険には、それぞれ内容、表面、操作、境界線、前景色のロールがあります。

## テーマ構造

祖先要素に `data-theme="tinyrack-light"` または `data-theme="tinyrack-dark"` を設定してください。両方のセレクターが同じセマンティック変数を定義するため、テーマが変わっても子要素の意図は維持されます。

```html
<html data-theme="tinyrack-light">
```

通常、ホバー、押下の各ロールをセットで使います。明るいテーマから暗いテーマの値を算出したり、無関係な塗りに前景色を載せたりしないでください。

## 主操作の色を安全に拡張する

Tinyrack のテーマセレクターを維持し、別の製品用アクセントセレクターを追加します。通常、ホバー、押下、前景のコントラストを一貫したセットとして保つため、4 つのロールをすべて上書きしてください。

```css
[data-accent="rack-blue"] {
  --tinyrack-primary: #1d4ed8;
  --tinyrack-primary-hover: #1e40af;
  --tinyrack-primary-pressed: #1e3a8a;
  --tinyrack-on-primary: #ffffff;
}
```

```html
<main data-theme="tinyrack-dark" data-accent="rack-blue">
```

## 避ける使い方

- プロダクト UI でパレット値を直接使わないでください。テーマに追従しません。
- ステータス色を装飾だけに使ったり、警告と危険のロールを入れ替えたりしないでください。
- `primary` だけを上書きしないでください。状態セットが不完全だと操作フィードバックが一貫しません。
- `on-*` 色を別の背景ロール上に置かないでください。

## 実装

```css
@import "@tinyrack/ui/core.css";
.rack-status { color: var(--tinyrack-info); background: var(--tinyrack-info-surface-subtle); border-color: var(--tinyrack-info-border); }
```

接頭辞付きユーティリティは [Tailwind の色マッピング](/ja/foundations/tailwind/#colors)で確認してください。
