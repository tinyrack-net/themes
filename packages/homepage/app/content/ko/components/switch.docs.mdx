## 계약

| 축 | 동작 |
| --- | --- |
| state | uncontrolled with `defaultChecked`, controlled with `checked` and `onCheckedChange` |
| semantics | pair the switch with a visible label; use a checkbox when submitting a batch choice |
| form | supports `name`, `value`, and `required` through its hidden native input |

function RequiredSwitch() {
  const [attempted, setAttempted] = useState(false);
  const [checked, setChecked] = useState(false);
  const invalid = attempted && !checked;

  return (
    <Form noValidate onSubmit={(event) => {
      event.preventDefault();
      event.currentTarget.checkValidity();
      setAttempted(true);
    }}>
      <Field.Root invalid={invalid}>
        <Field.Label>
          <Switch.Root aria-label="Enable health monitoring" checked={checked} name="monitoring" onCheckedChange={setChecked} required>
            <Switch.Thumb />
          </Switch.Root>
          Enable health monitoring.
        </Field.Label>
        {invalid ? <Field.Error match>Enable health monitoring to continue.</Field.Error> : null}
      </Field.Root>
      <Button type="submit">Continue</Button>
      <output aria-live="polite">{attempted && checked ? 'Health monitoring enabled.' : ''}</output>
    </Form>
  );
}` }]}
/>

## API

`Root`, `Thumb`, prop type, Base UI state type은 `@tinyrack/ui/components/switch`에서 export됩니다. CSS는 opt-in으로 유지됩니다. 보이는 label text를 `Root`와 연결하고 Space로 전환하세요. 변경할 수 없는 focusable value에는 `readOnly`, 사용할 수 없는 control에는 `disabled`를 선택합니다. `name`, `value`, `required`는 form 제출과 검증을 위해 hidden native input이 전달합니다.
