## 계약

| 축 | 동작 |
| --- | --- |
| 레이블 | `aria-label`/`aria-labelledby`를 제공하거나 legend가 있는 `TRFieldset.Root` 안에 group을 배치합니다. |
| 값 | `defaultValue`로 비제어형, `value`와 `onValueChange`로 제어형 상태를 사용합니다. |
| 키보드 | 모든 화살표 키가 focus와 선택을 이동하고 disabled radio를 건너뛰며 순환합니다. Tab은 한 번만 진입하고 이탈합니다. |
| 상태 | `disabled`는 상호작용과 form data를 제거합니다. `readOnly`는 선택한 radio의 focus를 유지하지만 변경을 막습니다. |
| form | `name`, `form`, `required`가 group에 적용되며 reset은 비제어형 `defaultValue`를 복원합니다. |
| 렌더링 | wrapper는 SSR-safe하며 선택 값을 바꾸지 않고 hydrate됩니다. |

`TRRadioGroup`의 화살표 탐색은 orientation과 무관하게 두 축에서 동작합니다. `orientation` prop은 제공하지 않으며 키보드 동작을 바꾸지 않고 `className` 또는 `style`로 시각적 layout을 정렬합니다.

## API

`TRRadioGroup`, `TRRadioGroupProps`, `TRRadioGroupState`는 `@tinyrack/ui/components/radio-group`에서 export됩니다. radio module의 `TRRadio.Root`, `TRRadio.Indicator`와 조합하세요. root는 `ref`, `inputRef`, native event와 attribute, state 기반 `className`/`style`, polymorphic `render`를 보존합니다. `onValueChange(value, details)`는 변경을 시작한 native event를 제공하며 `details.cancel()`로 변경을 거부할 수 있습니다.
