## Contract

| Axis | Values | Default | Use |
| --- | --- | --- | --- |
| `variant` | `secondary`, `primary`, `danger` | `secondary` | Communicates action intent. |
| `appearance` | `solid`, `outline`, `ghost` | `solid` | Changes emphasis without changing intent. |
| `uiSize` | `sm`, `md`, `lg` | `md` | Matches the surrounding control density. |
| state | `disabled`, `loading` | interactive | Uses native disabled behavior and `aria-busy`. |

## API

- `TRButton` forwards Base UI button props, native events, `className`, `style`, and React 19 `ref`.
- Native buttons default to `type="button"`; set `type="submit"` or `type="reset"` intentionally inside forms. With `render`, set `nativeButton={false}` for a non-button element; the native `type` default is then omitted.
- `loading` disables the control, renders an `aria-hidden` spinner, and sets `aria-busy` on the named button. `loadingLabel` becomes the accessible name even when the idle button uses `aria-labelledby`; without loading, an explicit `aria-busy` value is preserved.
- Secondary outline and ghost appearances use `--tinyrack-text` for readable foreground contrast rather than the lower-contrast border token.
