## Contract

`Main` owns the page's single main landmark. `Sidebar` is sticky and independently scrollable above the selected breakpoint. `sidebarMode` switches it between the expanded width and an icon rail, while `SidebarLabel` keeps every navigation name accessible in both states. Below the breakpoint, `mobileSidebar="drawer"` preserves the Base UI TRDrawer behavior and `mobileSidebar="rail"` opts into an inline fixed rail.

## API

`Root` accepts controlled `sidebarMode` or uncontrolled `defaultSidebarMode` (`expanded` by default), plus `onSidebarModeChange`. It exposes `data-sidebar-mode`, `data-mobile-sidebar`, `--tr-app-shell-sidebar-width`, and `--tr-app-shell-sidebar-rail-width` (default `4rem`). `mobileSidebar="drawer"` remains the default and always shows expanded labels in the modal; `mobileSidebar="rail"` renders the same navigation inline without Drawer, Backdrop, Close, or Trigger anatomy. `SidebarToggle` is a ghost `sm` IconButton with `aria-expanded`; `SidebarLabel` becomes visually hidden, not removed, in rail mode.
