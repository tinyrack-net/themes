## Contract

| Axis | Behavior |
| --- | --- |
| state | uncontrolled with `defaultChecked`, controlled with `checked` and `onCheckedChange` |
| semantics | pair the switch with a visible label; use a checkbox when submitting a batch choice |
| form | supports `name`, `value`, and `required` through its hidden native input |

function RequiredSwitch() {
  const [attempted, setAttempted] = useState(false);
  const [checked, setChecked] = useState(false);
  const invalid = attempted && !checked;

  return (
    <TRForm noValidate onSubmit={(event) => {
      event.preventDefault();
      event.currentTarget.checkValidity();
      setAttempted(true);
    }}>
      <TRField.Root invalid={invalid}>
        <TRField.Label>
          <TRSwitch.Root aria-label="Enable health monitoring" checked={checked} name="monitoring" onCheckedChange={setChecked} required>
            <TRSwitch.Thumb />
          </TRSwitch.Root>
          Enable health monitoring.
        </TRField.Label>
        {invalid ? <TRField.Error match>Enable health monitoring to continue.</TRField.Error> : null}
      </TRField.Root>
      <TRButton type="submit">Continue</TRButton>
      <output aria-live="polite">{attempted && checked ? 'Health monitoring enabled.' : ''}</output>
    </TRForm>
  );
}` }]}

## API

`Root`, `Thumb`, their prop types, and Base UI state types are exported from `@tinyrack/ui/components/switch`; CSS remains opt-in. Associate visible label text with `Root`, use Space to toggle, and choose `disabled` for unavailable controls versus `readOnly` for focusable values that cannot change. `name`, `value`, and `required` are carried by the hidden native input for form submission and validation.
