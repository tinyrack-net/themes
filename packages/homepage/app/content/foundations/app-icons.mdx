# App icons

Dotweave and TinyAuth use approved product marks derived from the Tinyrack app tile. Each mark keeps the same black tile, 25% corner radius, and three-row rhythm while using one accent color to describe the product.

Use the supplied files as complete artwork. Do not rebuild the geometry from the previews or combine either mark with the Tinyrack wordmark.

## Shared construction

The master brand and both product marks use the same normalized rack bay: identical row width, 8px row height, 5.333px gaps, rounded row ends, and a black tile with a 25% corner radius. Product meaning changes only inside that shared silhouette.

## Format and size guidance

Prefer SVG whenever the destination accepts it. Choose a PNG that matches the intended rendered size; do not upscale a smaller raster file.

<Table.Root containerProps={{ 'aria-label': 'App icon file guidance', tabIndex: 0 }}>
  <thead><tr><th scope="col">File</th><th scope="col">Use</th><th scope="col">Guidance</th></tr></thead>
  <tbody>
    <tr><th scope="row">SVG</th><td>Web interfaces and scalable assets</td><td>Default choice; one canonical 64×64 vector grid.</td></tr>
    <tr><th scope="row">16px PNG</th><td>Browser favicon</td><td>Use only where SVG favicons are unavailable.</td></tr>
    <tr><th scope="row">32px PNG</th><td>High-density favicon and compact lists</td><td>Render at 32px or 16px on a 2× display.</td></tr>
    <tr><th scope="row">48px PNG</th><td>Compact launchers and shortcuts</td><td>Use at the native 48px size.</td></tr>
    <tr><th scope="row">128px PNG</th><td>Service avatars and large launchers</td><td>Use when the surface requires a raster image.</td></tr>
    <tr><th scope="row">512px PNG</th><td>Store listings and downstream export</td><td>Keep as a high-resolution raster source; scale down, never up.</td></tr>
  </tbody>
</Table.Root>
