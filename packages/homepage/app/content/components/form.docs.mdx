# Form

A native form wrapper for submission, validity, and Base UI field errors.

## Contract

| Concern | Contract |
| --- | --- |
| submission | native `submit` events and `FormData`, or parsed field values through `onFormSubmit` |
| validation | native validity plus external `errors` rendered by Base UI `Field.Error` composition |
| reset | native `type="reset"` restores uncontrolled values; clear application results and server errors in `onReset` |
| state | applications own input values and post-submit results |

  return <Form
    aria-label="Native rack form"

    onSubmit={(event) => {
      event.preventDefault();
      const value = String(new FormData(event.currentTarget).get('rack') ?? '');
      setSubmitted(value);
    }}

    <label htmlFor="rack-name">

      <Input id="rack-name" name="rack" defaultValue="rack-alpha" />
    </label>
    <Button type="submit">Submit rack</Button>
    <Button type="reset">Reset form</Button>

  </Form>;

## API

`Form` forwards native form props, refs, events, `className`, `style`, and `render`; controls keep ownership of their own values. `FormProps<Values>` preserves the field-value generic through `onFormSubmit`. Use `errors` with field names for server responses, choose `validationMode` at Form or Field scope, and use `FormActions.validate(name?)` for explicit validation without synthesizing a submit.
