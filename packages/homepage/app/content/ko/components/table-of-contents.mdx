## 핵심 속성

순서대로 정렬한 h2/h3 heading 데이터와 현재 scroll-spy 결과를 전달하세요. depth 3 항목은 바로 앞 depth 2 항목 아래에 중첩돼요. `renderLink`로 router link를 넣을 수 있고, adapter가 없으면 데스크톱 link와 모바일 select가 hash navigation을 사용해요.

intersection observer에서 `currentHeading`을 업데이트하세요. link는 native keyboard 탐색을 유지하며, `onNavigate`로 router 또는 custom scroller와 controlled active state를 동기화할 수 있어요. `items`가 비어 있으면 `null`을 반환하고 좁은 layout에서는 긴 label의 overflow를 제한해요.

## API

`items`는 순서가 있는 `{ id, label, depth }` collection이며 `depth`는 `2 | 3`으로 제한돼요. `currentHeading`은 active link에 `aria-current="location"`을 설정하고 모바일 선택을 제어해요. `label`은 navigation landmark와 데스크톱 heading을, `mobileLabel`은 모바일 select를 명명해요. `onNavigate`는 선택한 item을 받고 `renderLink`는 데스크톱 router link를 제공해요. Native nav props, events, styles, `className`, `ref`가 전달돼요. 같은 데이터로 데스크톱 list와 keyboard로 조작 가능한 모바일 select를 렌더링하며 SSR/hydration에 안전해요.
