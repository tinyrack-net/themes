## 계약

| 항목 | 계약 |
| --- | --- |
| 제출 | 네이티브 `submit` 이벤트와 `FormData`를 사용하거나 `onFormSubmit`으로 파싱된 필드 값을 받아요. |
| 유효성 검사 | 네이티브 validity와 Base UI `TRField.Error` 조합이 렌더링하는 외부 `errors`를 함께 사용해요. |
| 초기화 | 네이티브 `type="reset"`은 비제어 값을 복원해요. 제어 값과 애플리케이션 결과는 `onReset`에서 초기화해야 해요. |
| 상태 | 컨트롤은 비제어 또는 애플리케이션 제어 방식으로 사용할 수 있으며, 애플리케이션이 서버 오류와 제출 결과를 소유해요. |

  const [submitted, setSubmitted] = useState('');

  return <TRForm

import '@tinyrack/ui/core.css';

import { TRButton } from '@tinyrack/ui/components/button';

import { TRInput } from '@tinyrack/ui/components/input';

function RequiredRackForm() {
  const [attempted, setAttempted] = useState(false);
  const [saved, setSaved] = useState('');
  const [value, setValue] = useState('');
  const invalid = attempted && value.trim().length === 0;

  return (
    <TRForm
      noValidate
      onSubmit={(event) => {
        event.preventDefault();
        setAttempted(true);
        if (event.currentTarget.checkValidity()) setSaved(value);
      }}
    >
      <label htmlFor="required-rack">랙 이름</label>
      <TRInput
        aria-describedby={invalid ? 'required-rack-error' : undefined}
        aria-invalid={invalid || undefined}
        id="required-rack"
        name="rack"
        onChange={(event) => {
          setValue(event.currentTarget.value);
          setSaved('');
        }}
        required
        value={value}
      />
      {invalid ? <p id="required-rack-error" role="alert">저장하기 전에 랙 이름을 입력해요.</p> : null}
      <TRButton type="submit">랙을 저장해요</TRButton>
      <output aria-live="polite">{saved ? saved + ' 값을 저장했어요.' : ''}</output>
    </TRForm>
  );
}` }]}

function ServerValidatedRackForm() {
  const [errors, setErrors] = useState<Record<string, string>>({});
  const [result, setResult] = useState('');

  return <TRForm
    aria-label="랙 생성 폼"

    onFormSubmit={(values) => {

        setResult('');
        return;

      setErrors({});
      setResult(rack + ' 랙을 만들었어요.');
    }}
    onReset={() => { setErrors({}); setResult(''); }}
  >
    <TRField.Root name="rack">
      <TRField.Label>랙 이름</TRField.Label>
      <TRField.Control defaultValue="rack-alpha" required />
      <TRField.Description>아직 등록되지 않은 이름을 사용해요.</TRField.Description>
      <TRField.Error />
    </TRField.Root>
    <TRButton type="submit">랙을 만들어요</TRButton>
    <TRButton type="reset">폼을 초기화해요</TRButton>
    <output aria-live="polite">{result}</output>
  </TRForm>;
}` }]}

export function RackForm() {
  const actionsRef = useRef<TRFormActions>(null);
  const [result, setResult] = useState('');

  return <TRForm<{ rack: string; region: string }>

    onFormSubmit={(values) => setResult(values.region + '의 ' + values.rack + '을 검증했어요.')}

      <TRField.Control required />

    </TRField.Root>

    <TRButton type="submit">제출해요</TRButton>
    <output aria-live="polite">{result}</output>

## API

`TRForm`은 네이티브 form props, ref, event, `className`, `style`, `render`를 전달하며 컨트롤은 자체 값을 계속 소유해요. `TRFormProps<Values>`는 `onFormSubmit(values, details)`을 통해 field-value generic을 보존하고, event detail에는 `TRFormSubmitEventDetails`와 `TRFormSubmitEventReason`을 사용할 수 있어요. 서버 응답에는 field name과 함께 `errors`를 사용하고 TRForm 또는 TRField 범위에서 `validationMode`를 선택하세요. submit 없이 명시적으로 검증하려면 `TRFormActions.validate(name?)`를 사용해요. 네이티브 reset은 비제어 컨트롤만 복원하므로 제어 값은 애플리케이션의 `onReset` handler에서 초기화해야 해요.
