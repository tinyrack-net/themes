## 구현

Components choose elevation by the visual separation their surface needs. TRPopover, TRToast, elevated TRCard, TRSwitch, and TRSlider use `raised`; TRDialog, TRAlert TRDialog, TRDrawer, TRAutocomplete, Context TRMenu, Navigation TRMenu, and Preview TRCard use `overlay`. Use a public shadow token only when building a custom elevated surface; do not infer modality from the shadow or add it to normal layout sections.

```css
.rack-popover {
  box-shadow: var(--tinyrack-shadow-raised);
}
```

```html
<div popover class="rounded-tinyrack-lg shadow-tinyrack-raised">
  Rack actions
</div>
```

tooling이나 생성된 style에서 TypeScript로 같은 값이 필요하면 `tinyrackShadows.raised`와 `tinyrackShadows.overlay`를 사용하세요.
