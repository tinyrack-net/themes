## Contract

| Axis | Values | Default |
| --- | --- | --- |
| `variant` | `neutral`, `info`, `success`, `warning`, `danger` | toast `type` or `neutral` |
| `position` | block start/end × inline start/center/end | `block-end-inline-end` |
| parts | `Provider`, `Portal`, `Viewport`, `Root`, `Content`, `Title`, `Description`, `Action`, `Close`, `Positioner`, `Arrow` | compound API |

  const manager = useToastManager();
  const toastId = useRef(null);

  return <>
    <Button onClick={() => {
      toastId.current = manager.add({
        title: 'Rack operation',
        description: 'This toast closes in five seconds.',
        timeout: 5000,
      });
    }}>Show timed toast</Button>
    <Button onClick={() => {
      if (toastId.current === null) return;
      manager.update(toastId.current, {
        title: 'Rack operation updated',
        type: 'success',
      });
    }}>Update toast</Button>
    <Button onClick={() => manager.promise(Promise.resolve('Rack A'), {
      loading: { title: 'Deploying rack', type: 'info' },
      success: (rack) => ({ title: rack + ' deployed', type: 'success' }),
      error: { title: 'Deployment failed', type: 'danger' },
    })}>Run promise</Button>
    <Button onClick={() => {
      for (let index = 1; index <= 4; index += 1) {
        manager.add({
          description: 'Queued with provider limit 2.',
          title: 'Queued toast ' + index,

    }}>Queue four</Button>
    <Button onClick={() => manager.close()}>Reset toasts</Button>

            <Toast.Content><Toast.Title>{toast.title}</Toast.Title><Toast.Description>{toast.description}</Toast.Description></Toast.Content>
            <Toast.Action onClick={() => manager.close(toast.id)}>Undo</Toast.Action>
            <Toast.Close aria-label="Dismiss notification">×</Toast.Close>
          </Toast.Root>
        ))}

## API

Create or consume a toast manager inside `Provider`; Base UI owns queueing, timers, swipe gestures, focus, and live-region semantics. `Content`, `Positioner`, and `Arrow` are available for composed layouts; give every simultaneously rendered `Viewport` a distinct `aria-label`.
