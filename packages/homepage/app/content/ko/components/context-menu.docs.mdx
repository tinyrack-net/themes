## 계약

`ContextMenu`는 rack row, file, canvas 같은 content에 command를 연결합니다. 전용 trigger가 항상 command list를 열 때는 `Menu`를 사용하세요. 이 Tinyrack wrapper는 Base UI 위치 지정, focus, 닫기, React 19 ref, native props, event, `render` contract를 보존합니다. Playground control은 `label`, `open`, `disabledItem`을 다룹니다.

import { ContextMenu } from '@tinyrack/ui/components/context-menu';
import { ChevronRight, Server } from 'lucide-react';
import { useRef } from 'react';

function RackRowActions() {
  const triggerRef = useRef<HTMLDivElement>(null);

  function openActions() {
    const rect = triggerRef.current?.getBoundingClientRect();
    if (!triggerRef.current || !rect) return;
    triggerRef.current.dispatchEvent(new MouseEvent('contextmenu', {
      bubbles: true,
      button: 2,
      clientX: rect.right - 24,
      clientY: rect.top + rect.height / 2,
    }));
  }

  return (
    <div className="grid gap-2 sm:grid-cols-[minmax(0,1fr)_auto]">
      <ContextMenu.Root>
        <ContextMenu.Trigger
          aria-label="Rack Alpha, online rack. Open context menu for actions."
          className="grid w-full grid-cols-[auto_minmax(0,1fr)_auto] items-center gap-3"
          ref={triggerRef}
          render={<Button appearance="ghost" type="button" />}
        >
          <Server aria-hidden="true" />
          <span>Rack Alpha</span>
          <span>10.42.0.18 · Seoul</span>
          <span>Online</span>
        </ContextMenu.Trigger>
        <Button appearance="outline" onClick={openActions} uiSize="sm">
          More actions
        </Button>
        <ContextMenu.Portal>
          <ContextMenu.Backdrop />
          <ContextMenu.Positioner>
            <ContextMenu.Popup>
              <ContextMenu.Arrow />
              <ContextMenu.Group>
                <ContextMenu.GroupLabel>Rack Alpha</ContextMenu.GroupLabel>
                <ContextMenu.LinkItem href="/racks/alpha">Open details</ContextMenu.LinkItem>
                <ContextMenu.Item>Copy address</ContextMenu.Item>
                <ContextMenu.Item>Restart</ContextMenu.Item>
              </ContextMenu.Group>
              <ContextMenu.Separator />
              <ContextMenu.SubmenuRoot>
                <ContextMenu.SubmenuTrigger>
                  Move to <ChevronRight aria-hidden="true" />
                </ContextMenu.SubmenuTrigger>
                <ContextMenu.Portal>
                  <ContextMenu.Positioner>
                    <ContextMenu.Popup>
                      <ContextMenu.Arrow />
                      <ContextMenu.Item>Production</ContextMenu.Item>
                      <ContextMenu.Item>Staging</ContextMenu.Item>
                    </ContextMenu.Popup>
                  </ContextMenu.Positioner>
                </ContextMenu.Portal>
              </ContextMenu.SubmenuRoot>
              <ContextMenu.Separator />
              <ContextMenu.Item variant="danger">Remove rack</ContextMenu.Item>
            </ContextMenu.Popup>
          </ContextMenu.Positioner>
        </ContextMenu.Portal>
      </ContextMenu.Root>
    </div>
  );
}` }]}
/>

## API

모든 compound part와 prop/state type은 `@tinyrack/ui/components/context-menu`에서 export됩니다. style은 대응 CSS subpath를 통한 opt-in으로 유지됩니다. 파괴적 명령에는 `Item variant="danger"`를 사용하세요. `Backdrop`, `Positioner`, `Popup`, `Arrow`는 layer anatomy를 공유하며 `Popup`은 제한된 scroll viewport이기도 합니다. Shift+F10을 위해 target을 focus 가능하게 만들고 touch 사용자가 오른쪽 클릭을 할 수 없을 때 같은 context-menu action을 실행하는 명시적 control을 제공하세요.
