## 階層を選ぶ

読む順序を先に決め、その階層を表せる最小限のロールを選んでください。`display` は大きな導入見出し、見出しロールは入れ子になったセクション、`body` と `bodySm` は本文、`caption` は補足、`label` は短い識別子に使います。

## 行長とローカライズ

段落は読み取りやすい幅に抑え、ラベルは内容に応じて伸縮できるようにしてください。日本語や韓国語は英語と異なる位置で改行されます。英語の文字列を基準に幅を固定したり、説明文を 1 行に強制したりしないでください。大文字の字間は短いラテン文字のラベルだけに使い、日本語の本文には適用しません。

## フォント読み込みの責任

Tinyrack のコア CSS は、`body`、`heading`、`mono` トークンに `"IBM Plex Sans"` というファミリー名だけを宣言します。フォントファイルは提供も読み込みもしません。利用するアプリで、必要なウェイトと言語サブセットを読み込んでください。このドキュメントアプリでは、IBM Plex Sans と韓国語・日本語用のファミリーを読み込んでいます。

## 実装とリファレンス

```css
@import "@tinyrack/ui/core.css";
.rack-title { font: var(--tinyrack-weight-heading) var(--tinyrack-text-2xl) / var(--tinyrack-leading-sm) var(--tinyrack-font-heading); }
```
