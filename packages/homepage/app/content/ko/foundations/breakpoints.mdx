## 스케일

브레이크포인트 토큰은 레이아웃에 여유 공간이 생기는 지점을 정의합니다. 모바일 우선으로 구성하고 콘텐츠에 필요한 시점에만 <TRCode>{'*-min'}</TRCode> 규칙을 추가합니다.

## 선택 기준

- 특정 기기가 시작되는 지점이 아니라 콘텐츠가 작업을 수행하기 어려워지는 지점에 브레이크포인트를 추가합니다.
- 페이지 셸과 애플리케이션 영역에는 뷰포트 브레이크포인트를 사용합니다. 재사용 컴포넌트의 적응은 해당 컴포넌트나 컨테이너 안에서 처리합니다.
- 시각적 재배치는 CSS를 우선합니다. 동작, 데이터 로딩, 상호작용 방식까지 바뀌어야 할 때만 TypeScript에서 공통 값을 읽습니다.

## TypeScript

```ts
import { tinyrackBreakpoints } from '@tinyrack/ui/core';

const desktopQuery = `(width >= ${tinyrackBreakpoints.lg})`;
```

## CSS와 Tailwind

Tinyrack CSS는 Tailwind의 <TRCode>@variant</TRCode> 지시어를 사용합니다. <TRCode>@tinyrack/ui/core.css</TRCode>를 컴포넌트 스타일보다 먼저 불러오면 Tailwind가 공통 브레이크포인트 스케일을 해석합니다.

```css
@variant md {
  /* Tinyrack source authoring */
}
```

Tailwind는 <TRCode>--breakpoint-xs</TRCode>부터 <TRCode>--breakpoint-xl</TRCode>까지 동일한 값을 사용합니다.

```tsx
<section className="grid gap-4 md:grid-cols-2 max-md:grid-cols-1" />
```

모바일 우선 규칙을 추가할 때는 생성된 <TRCode>xs:</TRCode>부터 <TRCode>xl:</TRCode>까지의 variant를 사용하고, 임계값 아래에 적용할 때는 <TRCode>max-md:</TRCode> 같은 variant를 사용해요. CSS의 기준은 <TRCode>--breakpoint-*</TRCode>이고 TypeScript의 기준은 <TRCode>tinyrackBreakpoints</TRCode>예요.

## 반응형 품질 확인

변경한 레이아웃은 브레이크포인트 바로 아래, 정확한 경계, 바로 위에서 확인합니다. 390px 모바일 뷰포트, 200% 확대, 긴 현지화 레이블, 키보드 포커스 순서를 포함하고 레이아웃이 접힐 때 필수 액션이 사라지지 않는지 검증합니다.
