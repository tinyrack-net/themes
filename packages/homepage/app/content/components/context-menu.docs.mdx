## Contract

`ContextMenu` attaches commands to content such as a rack row, file, or canvas. Use `Menu` when a dedicated trigger always opens the command list. This Tinyrack wrapper preserves Base UI positioning, focus, dismissal, React 19 refs, native props, events, and the `render` contract. Playground controls cover `label`, `open`, and `disabledItem`.

<ComponentExampleTabs

  title="Rack row actions"
  description="Right-click the rack row or focus it and press Shift+F10. The separate More actions button opens the same commands for touch users."
  preview={<Stories.ContextMenuPreview label="Rack Alpha" open={false} disabledItem={false} />}
  sources={[{ label: 'React', language: 'tsx', code: String.raw`import { Button } from '@tinyrack/ui/components/button';
import { ContextMenu } from '@tinyrack/ui/components/context-menu';
import { ChevronRight, Server } from 'lucide-react';
import { useRef } from 'react';

function RackRowActions() {
  const triggerRef = useRef<HTMLDivElement>(null);

  function openActions() {
    const rect = triggerRef.current?.getBoundingClientRect();
    if (!triggerRef.current || !rect) return;
    triggerRef.current.dispatchEvent(new MouseEvent('contextmenu', {
      bubbles: true,
      button: 2,
      clientX: rect.right - 24,
      clientY: rect.top + rect.height / 2,
    }));
  }

  return (
    <div className="grid gap-2 sm:grid-cols-[minmax(0,1fr)_auto]">
      <ContextMenu.Root>
        <ContextMenu.Trigger
          aria-label="Rack Alpha, online rack. Open context menu for actions."
          className="grid w-full grid-cols-[auto_minmax(0,1fr)_auto] items-center gap-3"
          ref={triggerRef}
          render={<Button appearance="ghost" type="button" />}
        >
          <Server aria-hidden="true" />
          <span>Rack Alpha</span>
          <span>10.42.0.18 · Seoul</span>
          <span>Online</span>
        </ContextMenu.Trigger>
        <Button appearance="outline" onClick={openActions} uiSize="sm">
          More actions
        </Button>
        <ContextMenu.Portal>
          <ContextMenu.Backdrop />
          <ContextMenu.Positioner>
            <ContextMenu.Popup>
              <ContextMenu.Arrow />
              <ContextMenu.Group>
                <ContextMenu.GroupLabel>Rack Alpha</ContextMenu.GroupLabel>
                <ContextMenu.LinkItem href="/racks/alpha">Open details</ContextMenu.LinkItem>
                <ContextMenu.Item>Copy address</ContextMenu.Item>
                <ContextMenu.Item>Restart</ContextMenu.Item>
              </ContextMenu.Group>
              <ContextMenu.Separator />
              <ContextMenu.SubmenuRoot>
                <ContextMenu.SubmenuTrigger>
                  Move to <ChevronRight aria-hidden="true" />
                </ContextMenu.SubmenuTrigger>
                <ContextMenu.Portal>
                  <ContextMenu.Positioner>
                    <ContextMenu.Popup>
                      <ContextMenu.Arrow />
                      <ContextMenu.Item>Production</ContextMenu.Item>
                      <ContextMenu.Item>Staging</ContextMenu.Item>
                    </ContextMenu.Popup>
                  </ContextMenu.Positioner>
                </ContextMenu.Portal>
              </ContextMenu.SubmenuRoot>
              <ContextMenu.Separator />
              <ContextMenu.Item variant="danger">Remove rack</ContextMenu.Item>
            </ContextMenu.Popup>
          </ContextMenu.Positioner>
        </ContextMenu.Portal>
      </ContextMenu.Root>
    </div>
  );
}` }]}
/>

## API

Every compound part and prop/state type is exported from `@tinyrack/ui/components/context-menu`; styles remain opt-in through the matching CSS subpath. Use `Item variant="danger"` for destructive commands. `Backdrop`, `Positioner`, `Popup`, and `Arrow` share the layer anatomy; `Popup` is also the constrained scrolling viewport. Make the target focusable for Shift+F10 and provide an explicit control that dispatches the same context-menu action when touch users cannot right-click.
