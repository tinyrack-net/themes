## コントラクト

TRNavigationMenu は、ブランド、少数の直接リンク、任意の Product または Resources flyout を持つサイトヘッダーに使用します。デスクトップ構成では Base UI の roving keyboard navigation を維持し、モバイル構成では同じリンク先を TRDrawer で表示します。

## API

完全な anatomy は `Root`、`List`、`Item`、`Trigger`、`Icon`、`Content`、`Portal`、`Backdrop`、`Positioner`、`Popup`、`Arrow`、`Viewport`、`Link` です。各 part、props type、state type は named export され、該当する part は `TRNavigationMenu` の member でもあります。

- `Root`: controlled な open state には `value` と `onValueChange`、uncontrolled state には `defaultValue` を使います。`orientation` は roving focus key と layout の両方を変更し、`delay` と `closeDelay` は pointer intent を調整します。
- `Link`: 現在の destination には `active`（`aria-current="page"`）を設定し、flyout 内の link には必要に応じて `closeOnClick` を設定します。
- `Positioner`: `side`、`align`、offset、collision padding、`collisionAvoidance` で配置を制御します。Tinyrack の default は popup を viewport 内に保つため align を shift し、side を flip します。
- `Portal`: custom container を指定できます。その中に `Backdrop` と `Positioner` を置きます。`Popup` が visual surface を持ち、`Viewport` が active な `Content` を受け取ります。
- style: `@tinyrack/ui/components/navigation-menu.css` を import します。`--tr-navigation-menu-color`、`--tr-navigation-menu-control-background`、`--tr-navigation-menu-popup-background`、`--tr-navigation-menu-popup-max-width`、`--tr-navigation-menu-backdrop` を custom property として利用できます。
- SSR: server render と hydration に対応しています。portal content は hydration 後に container へ mount されます。

TRNavigationMenu は十分な横幅で pointer と keyboard によるサイトナビゲーションを行うためのものです。flyout group は task 指向に保ち、preview が不要な destination には direct link を残します。狭い breakpoint では horizontal keyboard model を圧縮せず、同じ destination set を持つ TRDrawer 構成に切り替えます。
