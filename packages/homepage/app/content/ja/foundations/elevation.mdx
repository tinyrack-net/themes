## 実装

カスタムサーフェスに必要な視覚的分離に応じてシャドウを選び、操作の所有順に応じてレイヤーを選びます。モーダルサーフェスには、フォーカス管理、閉じる動作、`--tinyrack-scrim` を使うバックドロップも必要です。シャドウだけではコンテンツはモーダルになりません。

### レイヤー順

公開されている `tinyrackLayers` は、`base`、`backdrop`、`dropdown`、`popover`、`dialog`、`toast`、`tooltip` の順序を定義します。バックドロップはダイアログより下に置き、ポータル UI と競合する独自の値を作らないでください。

<TRTable.Root containerProps={{ 'aria-label': 'レイヤー順リファレンス', tabIndex: 0 }} data-foundation-reference="layers">
  <thead><tr><th scope="col">レイヤー</th><th scope="col">値</th><th scope="col">CSS 変数</th></tr></thead>
  <tbody>{Object.entries(tinyrackLayers).map(([name, value]) => <tr key={name}><th scope="row"><TRCode>{name}</TRCode></th><td>{value}</td><td><TRCode>{`--tinyrack-layer-${name}`}</TRCode></td></tr>)}</tbody>
</TRTable.Root>

```css
.rack-popover {
  box-shadow: var(--tinyrack-shadow-raised);
}
```

```html
<div popover class="rounded-tinyrack-lg shadow-tinyrack-raised">
  Rack actions
</div>
```

tooling や生成 style で TypeScript の同じ値が必要な場合は `tinyrackShadows.raised` と `tinyrackShadows.overlay` を使います。
