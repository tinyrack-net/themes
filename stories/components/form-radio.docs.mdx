# Radio

Mutually exclusive native choices with vertical or horizontal group layout.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `fieldset.tr-radio-group` | `RadioGroup` | fieldset attributes | Name related choices with a legend. |
| Behavior | `data-orientation` | `RadioGroup.orientation` | `vertical`, `horizontal`; default `vertical` | Use horizontal only when labels fit at narrow widths. |
| Component | `label.tr-radio` | `Radio` | native radio plus optional label | Give every choice a visible label. |
| Part | `tr-radio-input` | native input props | `type="radio"`; required part | Use one shared `name` for mutually exclusive choices. |
| Part | `tr-radio-control`, `tr-radio-label` | generated control, `children` | control required; label optional | Keep the custom control decorative. |
| Size | `data-size` | `Radio.size` | `sm`, `md`, `lg`; default `md` | Shares the choice-control scale with Checkbox and Switch. |
| State | `data-invalid`, `aria-invalid`, `disabled` | `Radio.invalid`, native props | invalid unset; disabled default `false` | Put state on each affected native radio. |

## Guidance

<ComponentGuidance
  whenToUse={[
    'Use RadioGroup when users must choose exactly one option from a short, visible set.',
    'Default to vertical orientation; use horizontal orientation only when every label fits comfortably at narrow widths.',
  ]}
  avoid={[
    'Do not use radio buttons for independent choices or give radios in one group different name values.',
    'Do not use aria-label instead of a visible legend when the group question can be shown on screen.',
  ]}
  accessibility={[
    'Wrap related radios in RadioGroup/fieldset and provide a legend that states the group question.',
    'Keep every native radio focusable with a visible label; the custom control remains aria-hidden.',
    'Preserve native keyboard behavior: Tab enters the group, arrow keys move selection, and Space selects the focused radio.',
  ]}
/>

## API

| Prop | Values | Note |
| --- | --- | --- |
| `RadioGroup.orientation` | `vertical`, `horizontal` | Default `vertical`; sets `data-orientation`. |
| `Radio.size` | `sm`, `md`, `lg` | Default `md`; sets `data-size`. |
| `Radio.invalid` | `boolean` | Sets `aria-invalid` on input and `data-invalid` on wrapper. |
| `children` | `ReactNode` | Renders `tr-radio-label` when present. |
| native radio props | `InputHTMLAttributes<HTMLInputElement>` except `children`, `size`, `type` | Passed through to the native radio input. |
| native fieldset props | `FieldsetHTMLAttributes<HTMLFieldSetElement>` | Passed through to `RadioGroup`. |
