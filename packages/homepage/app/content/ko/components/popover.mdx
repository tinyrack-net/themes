## API

Base UI가 anchor 위치 지정, collision 회피, focus 동작, portal, dialog 관계, outside/Escape 닫기를 담당합니다. 보조 액션에는 표면을 non-modal로 유지하고 뒤의 page를 막아야 하는 작업에는 TRDialog를 사용하세요.

### `TRPopover.Root`

`open` 또는 `defaultOpen` 상태, `onOpenChange`, `modal`, imperative `actionsRef`, trigger ID와 typed `handle`을 관리하며 DOM 요소는 렌더링하지 않습니다.

### `TRPopover.Trigger`

열기 button을 렌더링하며 `handle`, `payload`, `openOnHover`, `delay`, `closeDelay`, `render`, native button prop과 `ref`를 받습니다.

### `TRPopover.Portal`

기본적으로 layer를 `document.body`로 이동합니다. `container`로 host를 지정하고 `keepMounted`로 숨겨진 콘텐츠를 유지합니다.

### `TRPopover.Positioner`

`side`, `align`, `sideOffset`, `alignOffset`, `collisionAvoidance`, `collisionBoundary`, `collisionPadding`, `anchor`, `sticky`, `disableAnchorTracking`, `arrowPadding`을 받습니다.

### `TRPopover.Popup`

non-modal `dialog`를 렌더링하고 title/description 관계와 open, closed, side, alignment, transition data attribute를 노출합니다.

### `TRPopover.Arrow`

배치된 arrow를 렌더링합니다. `--tr-layer-arrow-size`로 크기를 변경합니다.

### `TRPopover.Backdrop`

선택적 backdrop을 렌더링하며 주로 `modal` 또는 `trap-focus` 표현에 사용합니다.

### `TRPopover.Title`

`aria-labelledby`를 통해 Popup의 label을 제공합니다.

### `TRPopover.Description`

`aria-describedby`를 통해 Popup을 설명합니다.

### `TRPopover.Close`

활성 popover를 닫고 Base UI dismissal 동작에 따라 focus를 복원합니다.

### `TRPopover.Viewport`

handle 기반 popover가 여러 trigger 사이에서 전환될 때 animated content transition을 담습니다.

### `TRPopover.createHandle`

분리된 trigger용 `TRPopoverHandle<Payload>`를 만듭니다. 같은 handle을 Root와 Trigger에 전달하면 Trigger의 `payload`가 Root render 함수에 제공됩니다.
