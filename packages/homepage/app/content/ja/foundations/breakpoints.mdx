## スケール

Breakpoint token はレイアウトに利用可能なスペースが増える位置を定義します。Mobile-first で構成し、コンテンツに必要な時点でのみ <TRCode>{'*-min'}</TRCode> rule を追加します。

<BreakpointReference locale="ja" />

## TypeScript

```ts
import { tinyrackBreakpoints } from '@tinyrack/ui/core';

const desktopQuery = `(width >= ${tinyrackBreakpoints.lg})`;
```

## CSS と Tailwind

Tinyrack CSS は Tailwind の <TRCode>@variant</TRCode> directive を使用します。<TRCode>@tinyrack/ui/core.css</TRCode> を component style より先に import すると、Tailwind が共通 breakpoint scale を解決します。

```css
@variant md {
  /* Tinyrack source authoring */
}
```

Tailwind は <TRCode>--breakpoint-xs</TRCode> から <TRCode>--breakpoint-xl</TRCode> まで同じ値を使用します。

```tsx
<section className="grid gap-4 md:grid-cols-2 max-md:grid-cols-1" />
```

互換 alias の <TRCode>xs-at-most</TRCode> と <TRCode>sm-at-most</TRCode> は境界値を含みます。新しい rule では <TRCode>{'*-min'}</TRCode> の補集合となる strict な <TRCode>{'*-max'}</TRCode> を使います。
