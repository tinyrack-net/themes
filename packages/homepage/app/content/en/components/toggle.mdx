## Contract

| Axis | Values | Default |
| --- | --- | --- |
| state | controlled `pressed` or uncontrolled `defaultPressed` | off |
| availability | enabled or `disabled` | enabled |
| label | visible text | required |

## API

TRToggle preserves Base UI button behavior, refs, native props, events, and `render`. Use controlled `pressed` with `onPressedChange` or uncontrolled `defaultPressed`; native Space/Enter activation and `disabled` semantics follow button behavior. Give each toggle a visible label, TRToggleGroup items also need a stable `value`, and polymorphic renders must preserve button keyboard semantics. Use a TRCheckbox or TRSwitch when a value must participate in form submission or required validation.
