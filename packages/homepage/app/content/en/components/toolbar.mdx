## Contract

This Tinyrack wrapper preserves Base UI behavior, React 19 refs, native props, events, and the `render` contract. Playground controls cover the visual `disabled` and `orientation` states; the accessible label remains semantic and focus looping retains its default behavior.

## API

Every compound part and prop/state type is exported from `@tinyrack/ui/components/toolbar`; styles remain opt-in through the matching CSS subpath. `TRToolbar.Root` defaults to horizontal arrow-key focus with looping enabled; vertical toolbars use Up/Down. `TRToolbar.Group` names related controls, `TRToolbar.Input` keeps normal text-editing keys, and disabled groups/items suppress activation. In a horizontal toolbar, use at most one `TRToolbar.Input` and place it last so text-editing keys do not block later toolbar items. Keep a focusable disabled `TRToolbar.Button` visibly unavailable, and give every `TRToolbar` and nested `Group` an accessible name.
