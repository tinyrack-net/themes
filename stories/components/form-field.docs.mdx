# Field

Composition primitives for labels, native controls, and supporting messages.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `tr-field` | `Field` | `data-size`; default `md` | Sets vertical rhythm for one complete field. |
| Part | `tr-label` | `Label` | native label attributes; no implicit association | Provide `for`/`htmlFor`, or wrap the control. |
| Part | `tr-form-message` | `FormMessage` | paragraph attributes; variant default `neutral` | Give the message an ID when it describes a control. |
| Size | `tr-field[data-size]` | `Field.size` | `sm`, `md`, `lg`; default `md` | Match the nested control size. |
| Variant | `tr-form-message[data-variant]` | `FormMessage.variant` | `neutral`, `error`; default `neutral` | Use `error` only for an actual validation error. |
| Native | `for`, `id`, `aria-describedby` | native props | pass-through | Build label and description relationships explicitly. |

## Usage

<ComponentExampleTabs
  id="form-field-basic"
  title="Basic field"
  description="A complete field groups an explicit label, native control, and associated supporting message."
  previewLayout="stretch"
  preview={(
    <Field size="md">
      <Label htmlFor="rack-name">Rack name</Label>
      <Input aria-describedby="rack-name-help" id="rack-name" placeholder="rack-a-01" />
      <FormMessage id="rack-name-help">Use a stable rack identifier.</FormMessage>
    </Field>
  )}
  sources={[
    {
      label: 'React',
      language: 'tsx',
      code: String.raw`import '@tinyrack/ui/core/core.css';
import '@tinyrack/ui/components/form/form.css';
import { Field, FormMessage, Input, Label } from '@tinyrack/ui/components/form/react';

<Field size="md">
  <Label htmlFor="rack-name">Rack name</Label>
  <Input aria-describedby="rack-name-help" id="rack-name" placeholder="rack-a-01" />
  <FormMessage id="rack-name-help">Use a stable rack identifier.</FormMessage>
</Field>;`,
    },
    {
      label: 'HTML',
      language: 'html',
      code: String.raw`<div class="tr-field" data-size="md">
  <label class="tr-label" for="rack-name">Rack name</label>
  <input class="tr-input" data-size="md" id="rack-name" aria-describedby="rack-name-help" />
  <p class="tr-form-message" data-variant="neutral" id="rack-name-help">
    Use a stable rack identifier.
  </p>
</div>`,
    },
  ]}
/>

## Guidance

<ComponentGuidance
  whenToUse={[
    'Use Field to keep one label, one control, and its help or error message in a consistent vertical rhythm.',
    'Use a neutral message for durable help text and the error variant only after validation identifies a problem.',
  ]}
  avoid={[
    'Do not use Field as a semantic replacement for fieldset when several related controls form one group.',
    'Do not show an error message without also marking the associated native control invalid.',
  ]}
  accessibility={[
    'Connect Label to the control with htmlFor/id and FormMessage with aria-describedby.',
    'For required or invalid fields, set required and aria-invalid on the native control and keep the message visible.',
    'Use fieldset and legend for checkbox or radio groups; disabled must remain a native control attribute.',
  ]}
/>

## API

| Prop | Values | Note |
| --- | --- | --- |
| `Field.size` | `sm`, `md`, `lg` | Default `md`; sets `data-size`. |
| `Label.htmlFor` | `string` | Native label association. |
| `FormMessage.variant` | `neutral`, `error` | Default `neutral`; sets `data-variant`. |
| native props | div, label, paragraph attributes | Passed through to each primitive. |
