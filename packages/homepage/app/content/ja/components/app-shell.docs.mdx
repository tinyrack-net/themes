## コントラクト

`Main` owns the page's single main landmark. `Sidebar` is sticky with its own scroll position at and above the selected breakpoint, so document scrolling does not move or reset its navigation. Below the breakpoint it uses Base UI Drawer for focus, dismissal, scroll lock, and focus return. The `open` state only affects that mobile Drawer; it does not hide the desktop sidebar.

## API

`Root` accepts controlled or uncontrolled open state, `sm` or `lg` breakpoints, and `header-first` or `sidebar-first` layout. Pass an optional `portalContainer` to keep the mobile Drawer inside a preview or other host element. `Trigger` and `閉じる` default to a ghost `sm` IconButton and require an accessible name; an explicit `appearance` or `size` still wins. They are not rendered on desktop, so inline display styles cannot leak them back into the layout. On desktop, `Sidebar` stays sticky within the viewport while its plain `ScrollArea` scrolls independently from the document. On mobile it becomes a modal Drawer. Put visual padding on a child inside `Sidebar`, not on `Sidebar` itself, so the scrollbar stays aligned with the outer edge.
