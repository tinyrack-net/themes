## 계약

`Main`은 페이지의 단일 main landmark를 담당합니다. 선택한 breakpoint 이상에서 `Sidebar`는 고정되고 독립적으로 스크롤됩니다. `sidebarMode`로 expanded와 icon rail을 전환하며, `SidebarLabel`은 두 상태 모두에서 탐색 이름을 접근 가능하게 유지합니다. breakpoint 아래에서는 `mobileSidebar="drawer"`가 기존 Base UI TRDrawer 동작을 보존하고, `mobileSidebar="rail"`이 inline 고정 rail을 활성화합니다.

## API

`Root`는 `breakpoint`(`sm` 또는 `lg`), `layout`(`header-first` 또는 `sidebar-first`), `mobileSidebar`(`drawer` 또는 `rail`)를 받습니다. Drawer 상태는 `open`, `defaultOpen`, `onOpenChange`로, 사이드바 너비는 `sidebarMode`, `defaultSidebarMode`, `onSidebarModeChange`로 제어합니다. 포함형 preview나 embedded app에서는 `portalContainer`를, popup 스타일에는 `drawerPopupClassName`을 사용합니다.

`Header`는 header landmark를 렌더링합니다. `Sidebar`는 독립적으로 스크롤되는 `aside`이며 mobile dialog에도 접근 가능한 이름을 전달합니다. `Trigger`는 mobile drawer를 열고 `Close`는 닫습니다. `SidebarToggle`은 focus를 유지하고 `aria-expanded`를 노출하면서 expanded/rail mode를 전환합니다. `SidebarLabel`은 rail에서 제거되지 않고 시각적으로 숨겨집니다. `Main`은 main landmark와 Base UI `render` 계약을 제공합니다.

안정적인 스타일 계약은 `--tr-app-shell-background`, `--tr-app-shell-sidebar-background`, `--tr-app-shell-sidebar-width`, `--tr-app-shell-sidebar-rail-width`, `--tr-app-shell-drawer-block-size`입니다. Root 상태는 `data-breakpoint`, `data-layout`, `data-mobile-sidebar`, `data-sidebar-mode`에 반영됩니다.

SSR에서는 module 평가 중 browser global을 읽지 않고 desktop/static 구조를 렌더링한 뒤 hydration합니다. client가 선택한 media query를 적용하므로 application render에서 `window`에 따라 child markup을 바꾸지 마세요. Modal drawer는 focus를 가두고 Escape나 backdrop으로 닫히며 focus를 `Trigger`로 복원하고 열린 동안 page scroll을 막습니다.
