## Contract

| Axis | Behavior |
| --- | --- |
| state | input text, selected value, and open state are separate controllable axes |
| input anatomy | Label → InputGroup → optional InputAdornment + TRInput + Clear + Trigger; Icon belongs inside Trigger |
| multiple anatomy | Chips → Value → Chip + ChipRemove; grid collections compose Collection → Row → Item |
| popup anatomy | Portal → Positioner → Popup → Status + List; groups, indicators, separators, and Empty cover collection feedback |

## API

The complete label, input, popup, multiple-chip, row, collection, group, item-indicator, status, empty-state, and arrow anatomy is exported from `@tinyrack/ui/components/combobox` with matching generic prop and state types. Use `InputAdornment` for decorative content beside the input; `Icon` is the popup indicator and belongs inside `Trigger`. Use a native `label` or `TRField.Label` for an editable `TRCombobox.Input`; `TRCombobox.Label` specifically names a trigger-only select anatomy.
