## Contract

`TRDocsSearch.Trigger` is the standalone button and `TRDocsSearch.Dialog` is the controlled, portalled search surface. The dialog owns Ctrl/⌘+K, loading, empty, error, fallback, keyboard selection, focus restoration, and stale-request cancellation. Your `onSearch(query, signal)` function owns the data source and must pass its `AbortSignal` to cancellable work.

### Disabled trigger

Disable every opening path, including the dialog shortcut, when search is unavailable.

<ComponentExampleTabs id="docs-search-disabled" data-docs-example-item-count="1" title="Unavailable search" preview={<Stories.DocsSearchDisabled />} sources={[{ label: 'React', language: 'tsx', code: String.raw`import { useRef, useState } from 'react';
import '@tinyrack/ui/components/docs-search.css';
import { TRDocsSearch } from '@tinyrack/ui/components/docs-search';

export function DisabledSearch() {
  const [open, setOpen] = useState(false);
  const triggerRef = useRef<HTMLButtonElement>(null);

  return <>
    <TRDocsSearch.Trigger
      aria-label="Search unavailable"
      disabled
      label="Search docs"
      onClick={() => setOpen(true)}
      ref={triggerRef}
    />
    <TRDocsSearch.Dialog
      enableShortcut={false}
      onOpenChange={setOpen}
      onSearch={async () => []}
      onSelect={() => setOpen(false)}
      open={open}
      returnFocusRef={triggerRef}
    />
  </>;
}` }]} />

- `ArrowDown` and `ArrowUp` move the active result while focus remains in the combobox; `Enter` selects it and `Escape` closes the dialog.
- `enableShortcut={false}` disables the global shortcut, which is useful for embedded previews or when a disabled trigger must disable every opening path.
- `compact` hides the visible label and shortcut while preserving the trigger's accessible name. `uiSize="sm" | "md" | "lg"` keeps compact triggers square.
- The dialog portals to `document.body`. Pass the trigger through `returnFocusRef` to restore focus after dismissal.
- Closed dialogs and triggers are SSR-safe and hydrate without rendering the portal until opened.
- Set `fallback` when the primary search index is unavailable to explain why a reduced local result set is shown. Rejected searches use the localized `messages.error` state demonstrated in the Usage source.

## API

| Export | Key props | Purpose |
| --- | --- | --- |
| `TRDocsSearch` | `Trigger`, `Dialog` | Compound namespace. |
| `TRDocsSearch.Trigger` | `compact`, `label`, `shortcutLabel`, `uiSize`, plus button props and `ref` | Opens search from a visible or compact button. `disabled` prevents button activation. |
| `TRDocsSearch.Dialog` | `open`, `onOpenChange`, `onSearch`, `onSelect`, `messages`, `fallback`, `enableShortcut`, `popupRef`, `returnFocusRef` | Controlled search dialog and result list. |
| `TRDocsSearchResult` | `id`, `url`, `title`, `section`, `excerpt`, and optional match ranges | Search result rendered by the dialog. Match ranges use end-exclusive offsets. |
| `TRDocsSearchMessages` | `title`, `trigger`, `placeholder`, `close`, `results`, `idle`, `loading`, `empty`, `error`, `fallback` | Complete localizable visible and accessible copy. |

`onSearch` receives `(query: string, signal: AbortSignal)` and returns a promise of readonly results. Rejections show `messages.error`; aborted or superseded requests never replace newer results.
