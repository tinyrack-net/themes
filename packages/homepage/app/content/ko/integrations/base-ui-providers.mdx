## 설치

```sh
pnpm add @tinyrack/ui tailwindcss react react-dom
```

## 콘텐츠 보안 정책

모든 server response마다 새 nonce를 생성하고 같은 값을 HTTP `Content-Security-Policy`
header에 포함하세요. 값을 변경하지 말고 server render와 hydration에 전달합니다.
`TRCSPProvider`는 HTTP header를 생성하지 않습니다.

```tsx
import { randomBytes } from 'node:crypto';
import type { ReactNode } from 'react';
import { TRCSPProvider } from '@tinyrack/ui/providers/csp';

export function createRequestCsp() {
  const nonce = randomBytes(16).toString('base64');

  return {
    nonce,
    header: [
      "default-src 'self'",
      `style-src 'self' 'nonce-${nonce}'`,
      `script-src 'self' 'nonce-${nonce}'`,
    ].join('; '),
  };
}

export function AppProviders({
  children,
  nonce,
}: {
  children: ReactNode;
  nonce: string;
}) {
  return <TRCSPProvider nonce={nonce}>{children}</TRCSPProvider>;
}
```

요청마다 `createRequestCsp()`를 한 번 호출하고 같은 HTTP response에 `header`를 넣으세요.
`nonce`를 server-rendered `AppProviders`에 전달하고 hydration에도 같은 nonce를 직렬화합니다.
두 번째 client nonce를 생성하지 마세요. 기존 CSP를 이 최소 예제로 교체하지 말고 애플리케이션의
전체 정책에 맞게 directive를 조정하세요.

inline style element가 금지된 경우 nonce를 생략하고 명시적으로 opt out하세요.

```tsx
import type { ReactNode } from 'react';
import { TRCSPProvider } from '@tinyrack/ui/providers/csp';

export function AppProviders({ children }: { children: ReactNode }) {
  return <TRCSPProvider disableStyleElements>{children}</TRCSPProvider>;
}
```

그러면 Base UI 내부 구현이 주입했을 style을 애플리케이션이 직접 관리합니다.

## 텍스트 방향

native document direction과 Base UI context를 동기화하세요. `dir` attribute는 browser의
텍스트·layout 의미를 제어하고 `TRDirectionProvider`는 방향을 인식하는 Base UI 동작을 제어합니다.
`useDirection`은 provider 값만 읽습니다.

```tsx
import type { ReactNode } from 'react';
import { TRDirectionProvider } from '@tinyrack/ui/providers/direction';

export function DirectionDocument({
  children,
  direction,
}: {
  children: ReactNode;
  direction: 'ltr' | 'rtl';
}) {
  return (
    <html dir={direction} lang={direction === 'rtl' ? 'ar' : 'en'}>
      <body>
        <TRDirectionProvider direction={direction}>{children}</TRDirectionProvider>
      </body>
    </html>
  );
}
```

Provider import는 의도적으로 component import와 분리되어 있습니다. CSS를 제공하지 않으며
package root에서 re-export되지 않습니다. 활성 Base UI context 값이 필요한 하위 요소 안에서만
direction subpath에서 `useDirection`을 import하세요.
