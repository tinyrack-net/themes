## Implementation

Choose shadow from the separation a custom surface needs. Choose layer order from interaction ownership. A modal surface also needs focus management, dismissal behavior, and a backdrop that uses `--tinyrack-scrim`; a shadow alone never makes content modal.

### Layer order

The exported `tinyrackLayers` contract orders `base`, `backdrop`, `dropdown`, `popover`, `dialog`, `toast`, and `tooltip`. Keep a backdrop below its dialog, and avoid inventing local values that compete with portalled UI.

```css
  box-shadow: var(--tinyrack-shadow-raised);
}
```

```html

</div>
```

Use `tinyrackShadows.raised` and `tinyrackShadows.overlay` when tooling or generated styles need the same values in TypeScript.
