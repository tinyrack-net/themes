# Badge

Compact status text for operational metadata, health states, and dense labels.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `tr-badge` on `<span>` | `<Badge>` | Required component surface | Use for compact, inline status metadata. |
| Size | `data-size` | `size` | `sm`, `md`; default `sm` | Use `sm` in dense tables and `md` near controls. |
| Variant | `data-variant` | `variant` | `neutral`, `primary`, `danger`; default `neutral` | Communicate operational state rather than decoration. |
| Native | Native `<span>` attributes | `HTMLAttributes<HTMLSpanElement>` | Passed through | Preserve inline semantics and add an accessible name only when surrounding text does not explain the status. |

## Guidance

<ComponentGuidance
  whenToUse={[
    'Show compact rack health, queue, or deployment state beside a resource name.',
    'Label dense metadata where a short word is easier to scan than explanatory prose.',
  ]}
  avoid={[
    'Do not use a badge as a button, filter, or other interactive control.',
    'Do not rely on color alone or place long resource names inside the badge.',
  ]}
  accessibility={[
    'Use explicit state text such as "healthy" or "blocked" so meaning survives without color.',
    'Keep the badge next to the resource or sentence it describes; add native span attributes only when extra context is required.',
  ]}
/>

## API

| Prop | Values | Note |
| --- | --- | --- |
| `size` | `sm`, `md` | Default `sm`; sets `data-size`. |
| `variant` | `neutral`, `primary`, `danger` | Default `neutral`; sets `data-variant`. |
| `className` | `string` | Merged with `tr-badge`. |
| native span props | `HTMLAttributes<HTMLSpanElement>` | Passed through to `<span>`. |
