## Contract

Pass ordered h2/h3 heading data and the current scroll-spy result. Depth-3 items nest under the preceding depth-2 item. `renderLink` can inject a router link; without an adapter, desktop links and the mobile select use hash navigation.

Update `currentHeading` from your intersection observer. Links remain native and keyboard accessible; `onNavigate` lets a router or custom scroller coordinate navigation with the controlled active state. The component returns `null` when `items` is empty and constrains long labels on narrow layouts.

## API

`items` is the ordered `{ id, label, depth }` collection, with `depth` limited to `2 | 3`. `currentHeading` marks the active link with `aria-current="location"` and controls the mobile selection. `label` names the navigation landmark and desktop heading; `mobileLabel` names the mobile select. `onNavigate` receives the chosen item, while `renderLink` supplies a router link for desktop. Native nav props, events, styles, `className`, and `ref` are forwarded. The component renders a desktop list and keyboard-operable mobile select from the same data and is SSR/hydration safe.
