## Contract

| Axis | Behavior |
| --- | --- |
| composition | `TRRadio.Root` is always placed inside `TRRadioGroup`; `TRRadio.Indicator` is optional visual anatomy inside the root |
| selection | the group owns `value`, `defaultValue`, and `onValueChange`; each root contributes one unique `value` |
| interaction | click a label or radio to select it; arrow keys move focus and selection within the group |
| state | roots expose checked, unchecked, invalid, read-only, disabled, required, focus, and field state through Base UI data attributes |
| forms | put `name`, `form`, and group-level `required` on `TRRadioGroup`; use a root `inputRef` only when its hidden native input is needed |

Do not use a radio as a standalone boolean control. Use `TRCheckbox` or `TRSwitch` when the user can turn one independent choice on and off.

## Usage

<ComponentExampleTabs
  id="radio-basic"
  title="Choose a support plan"
  preview={<Stories.RadioPlanExample />}
  sources={[{ label: 'React', language: 'tsx', code: String.raw`import { useId, useState } from 'react';
import { TRRadio } from '@tinyrack/ui/components/radio';
import { TRRadioGroup } from '@tinyrack/ui/components/radio-group';
import '@tinyrack/ui/components/radio.css';
import '@tinyrack/ui/components/radio-group.css';

export function SupportPlanField() {
  const [value, setValue] = useState('standard');
  const id = useId();
  const plans = [
    ['standard', 'Standard · Community support'],
    ['priority', 'Priority · 4-hour response'],
    ['critical', 'Critical · 24/7 response'],
  ];

  return (
    <TRRadioGroup
      aria-label="Support plan"
      name="support-plan"
      onValueChange={(nextValue) => setValue(String(nextValue))}
      value={value}
    >
      {plans.map(([planValue, label]) => (
        <label htmlFor={id + '-' + planValue} key={planValue}>
          <TRRadio.Root id={id + '-' + planValue} value={planValue}>
            <TRRadio.Indicator aria-hidden="true" />
          </TRRadio.Root>
          {label}
        </label>
      ))}
      <output aria-live="polite">Selected plan: {value}</output>
    </TRRadioGroup>
  );
}` }]}
/>

## API

`TRRadio` exports `Root` and `Indicator`, their prop types, their state types, and `TRRadioUiSize`. `Root` preserves Base UI's `render`, functional `className`/`style`, root `ref`, hidden-input `inputRef`, native attributes, and event handlers. Use `readOnly` or `disabled` on an individual root only when that option alone is unavailable; prefer the matching group prop when every option is unavailable. `Indicator` mounts only while selected unless `keepMounted` is set.

Selection events belong to `TRRadioGroup.onValueChange`, not an individual root. Give the group an accessible name and every root an accessible label. Consumer tokens include `--tr-radio-size`, `--tr-radio-indicator-size`, base/hover/checked/invalid border and background tokens, and indicator background.
