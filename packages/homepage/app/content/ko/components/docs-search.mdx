## 계약

`TRDocsSearch.Trigger`는 독립 실행형 버튼이고 `TRDocsSearch.Dialog`는 제어되는 포털 검색 화면이에요. 대화상자는 Ctrl/⌘+K, 로딩, 빈 결과, 오류, `fallback`, 키보드 선택, 포커스 복원, 오래된 요청 취소를 관리해요. 데이터 소스는 `onSearch(query, signal)`이 관리하며, 취소 가능한 작업에 전달받은 `AbortSignal`을 사용해야 해요.

## 플레이그라운드

컨트롤에서 `disabled`와 `compact` 트리거, 세 가지 크기, 표시 레이블을 변경할 수 있어요. 대화상자를 연 뒤 **초기화**를 누르면 컨트롤이 복원되고 내부 상호 작용 상태가 초기화돼요.

### 비활성 트리거예요

검색할 수 없을 때는 대화상자 단축키를 포함한 모든 열기 경로를 비활성화해요.

<ComponentExampleTabs id="docs-search-disabled" data-docs-example-item-count="1" title="비활성 검색" preview={<Stories.DocsSearchDisabled />} sources={[{ label: 'React', language: 'tsx', code: String.raw`import { useRef, useState } from 'react';
import '@tinyrack/ui/components/docs-search.css';
import { TRDocsSearch } from '@tinyrack/ui/components/docs-search';

export function DisabledSearch() {
  const [open, setOpen] = useState(false);
  const triggerRef = useRef<HTMLButtonElement>(null);

  return <>
    <TRDocsSearch.Trigger
      aria-label="문서 검색을 사용할 수 없음"
      disabled
      label="문서 검색"
      onClick={() => setOpen(true)}
      ref={triggerRef}
    />
    <TRDocsSearch.Dialog
      enableShortcut={false}
      onOpenChange={setOpen}
      onSearch={async () => []}
      onSelect={() => setOpen(false)}
      open={open}
      returnFocusRef={triggerRef}
    />
  </>;
}` }]} />

- 포커스는 콤보박스에 유지되며 `ArrowDown`과 `ArrowUp`으로 활성 결과를 이동해요. `Enter`는 결과를 선택하고 `Escape`는 대화상자를 닫아요.
- `enableShortcut={false}`는 전역 단축키를 꺼요. 임베드된 미리 보기나 비활성 트리거에서 모든 열기 경로를 막을 때 사용해요.
- `compact`는 접근 가능한 이름을 유지하면서 표시 레이블과 단축키를 숨겨요. `uiSize="sm" | "md" | "lg"`에서 컴팩트 트리거는 정사각형을 유지해요.
- `Dialog`는 `document.body`에 포털돼요. 닫힌 뒤 트리거로 포커스를 복원하려면 `returnFocusRef`를 전달해요.
- 닫힌 대화상자와 트리거는 SSR에서도 안전하고, 열리기 전에는 포털을 렌더링하지 않은 채 하이드레이션돼요.
- 기본 검색 색인을 사용할 수 없을 때 축소된 로컬 결과가 표시되는 이유를 설명하려면 `fallback`을 설정해요. 검색 요청이 거부되면 사용법 소스에 나온 지역화된 `messages.error` 상태를 사용해요.

## API

| 내보내기 | 주요 속성 | 용도 |
| --- | --- | --- |
| `TRDocsSearch` | `Trigger`, `Dialog` | 복합 네임스페이스예요. |
| `TRDocsSearch.Trigger` | `compact`, `label`, `shortcutLabel`, `uiSize`, `button` 속성, `ref` | 표시형 또는 컴팩트 검색 버튼이에요. `disabled`는 버튼 활성화를 막아요. |
| `TRDocsSearch.Dialog` | `open`, `onOpenChange`, `onSearch`, `onSelect`, `messages`, `fallback`, `enableShortcut`, `popupRef`, `returnFocusRef` | 제어되는 검색 대화상자와 결과 목록이에요. |
| `TRDocsSearchResult` | `id`, `url`, `title`, `section`, `excerpt`, 선택적 일치 범위 | `Dialog`가 렌더링하는 검색 결과예요. 일치 범위의 끝 오프셋은 포함하지 않아요. |
| `TRDocsSearchMessages` | `title`, `trigger`, `placeholder`, `close`, `results`, `idle`, `loading`, `empty`, `error`, `fallback` | 표시 및 접근성 문구 전체를 지역화해요. |

`onSearch`는 `(query: string, signal: AbortSignal)`을 받고 읽기 전용 결과의 `Promise`를 반환해요. 요청이 거부되면 `messages.error`가 표시되고, 취소되거나 교체된 요청은 최신 결과를 덮어쓰지 않아요.
