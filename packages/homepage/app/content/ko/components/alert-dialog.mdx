## 계약

이 Tinyrack wrapper는 Base UI 동작, React 19 refs, native props, events, `render` contract를 보존해요. Playground control은 `label`과 `disabled`를 다뤄요. `open`은 component event로 동기화되는 preview-owned interaction state로 유지되고 Reset은 초기 상태와 result를 복원해요.

## API

anatomy는 `Root`, `Trigger`, `Portal`, `Backdrop`, `Viewport`, `Popup`, `Title`, `Description`, `Close`예요. 모든 part와 prop/state type은 `@tinyrack/ui/components/alert-dialog`에서 export되고 style은 대응하는 CSS subpath로 opt-in해요.

`Root`는 항상 modal이에요. 열린 동안 focus는 내부에 머물고 Escape와 `Close`는 닫기를 요청하며 outside interaction으로는 닫히지 않아요. 닫힌 뒤 focus는 `Trigger`로 돌아가요. 첫 `Close`는 안전한 취소로 사용하고 danger confirmation `Close`에만 파괴적 작업을 연결하세요. disabled trigger와 close는 동작하지 않아요.

`Root` 밖의 trigger에는 `TRAlertDialog.createHandle<Payload>()`로 typed bridge를 만들고 `Trigger`와 `Root` 모두에 handle을, `Trigger`에는 payload를 전달하세요. `Portal`은 custom container를 받을 수 있어요. 확인 컨트롤을 `tr-alert-dialog-actions`로 그룹화하면 좁은 viewport에서 읽기 쉬운 action으로 줄바꿈돼요.
