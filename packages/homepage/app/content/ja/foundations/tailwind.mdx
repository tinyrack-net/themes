<h2 className="tr-mdx-h2" id="setup">セットアップ</h2>

Tailwind を先に読み込み、続けて Tinyrack のコア CSS を読み込みます。コアスタイルシートはランタイム <TRCode>--tinyrack-*</TRCode> 変数を維持しながら、Tailwind v4 の静的テーマ名前空間として公開します。

```css
@import "tailwindcss";
@import "@tinyrack/ui/core.css";
```

コンポーネントスタイルは分離したまま、使用するものだけを読み込みます。

```css
@import "@tinyrack/ui/components/button.css";
```

<h2 className="tr-mdx-h2" id="choose-utilities">ユーティリティの選択</h2>

- カラー、タイポグラフィ、スペーシング、形状、エレベーション、モーションがデザインシステムの判断を表す場合は <TRCode>tinyrack-*</TRCode> ユーティリティを使います。
- ビューポートしきい値には共通の <TRCode>xs:</TRCode> から <TRCode>xl:</TRCode> までのレスポンシブ変形を使います。
- 一時的なグリッド列数など、Tinyrack トークンを表さない補助的なレイアウトには Tailwind のデフォルトユーティリティを使います。
- その他の Tailwind デフォルトテーマはグローバルに上書きしません。接頭辞により既存アプリケーションと外部サンプルの意味を安定して保ちます。

```tsx
<section className="grid gap-tinyrack-lg rounded-tinyrack-lg border-tinyrack-default border-tinyrack-border bg-tinyrack-surface p-tinyrack-xl">
  <h2 className="text-tinyrack-xl font-tinyrack-heading">クラスター状態</h2>
  <p className="text-tinyrack-sm leading-tinyrack-md text-tinyrack-text-muted">
    すべてのノードを利用できます。
  </p>
</section>
```

<h2 className="tr-mdx-h2" id="arbitrary-values">任意の値</h2>

コンポーネント範囲の <TRCode>--tr-*</TRCode> 変数や他の CSS 値に自然な Tailwind テーマ名前空間がない場合は型ヒントを使います。解決済みの値をコピーせず、変数を基準値として維持します。

```tsx
<div className="bg-[color:var(--tr-card-background)] [width:var(--tr-layer-anchor-width)]">
  ...
</div>
```

以下のリファレンスに名前付きユーティリティがある場合は、それを優先します。任意の値は意図的なコンポーネントのカスタマイズ用であり、別のトークン体系ではありません。
