<h2 className="tr-mdx-h2" id="setup">설정</h2>

Tinyrack을 사용하는 애플리케이션에 Tailwind CSS와 Vite 플러그인을 설치하세요.

```sh
pnpm add -D tailwindcss @tailwindcss/vite
```

애플리케이션의 Vite 설정에 플러그인을 추가하세요.

```ts
import tailwindcss from '@tailwindcss/vite';
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [tailwindcss()],
});
```

Tailwind를 먼저 불러오고 Tinyrack 핵심 CSS를 이어서 불러오세요. 사용하는 컴포넌트 스타일시트는 각각 따로 불러오세요.

```css
@import "tailwindcss";
@import "@tinyrack/ui/core.css";
@import "@tinyrack/ui/components/card.css";
```

<h2 className="tr-mdx-h2" id="choose-utilities">값 선택</h2>

1. 색상, 타이포그래피, 간격, 모서리 반경, 고도, 모션처럼 디자인 시스템의 결정을 나타내는 값에는 이름이 있는 <TRCode>tinyrack-*</TRCode> 유틸리티를 선택하세요.
2. <TRCode>grid</TRCode>나 <TRCode>grid-cols-2</TRCode>처럼 부수적인 레이아웃에는 Tailwind 기본 유틸리티를 선택하세요. Tinyrack 뷰포트 임계값에는 <TRCode>xs:</TRCode>부터 <TRCode>xl:</TRCode>까지 사용해요.
3. 실제 컴포넌트 변수나 일회성 CSS 값에 이름 있는 유틸리티가 없을 때만 임의 값을 선택하세요. 별도의 토큰 체계를 만들지 마세요.

<h2 className="tr-mdx-h2" id="practical-example">실전 예시</h2>

이 카드는 시스템 결정에 이름 있는 Tinyrack 유틸리티를 사용하고 레이아웃에는 Tailwind 기본 유틸리티를 사용해요. 위 설정을 마치면 그대로 붙여 넣을 수 있어요.

```tsx
export function ClusterHealth() {
  return (
    <section className="grid gap-tinyrack-lg rounded-tinyrack-lg border-tinyrack-default border-tinyrack-border bg-tinyrack-surface p-tinyrack-xl md:grid-cols-2">
      <h2 className="text-tinyrack-xl font-tinyrack-heading">클러스터 상태</h2>
      <p className="text-tinyrack-sm text-tinyrack-text-muted">
        모든 노드를 사용할 수 있어요.
      </p>
    </section>
  );
}
```

<h2 className="tr-mdx-h2" id="arbitrary-values">임의 값</h2>

공개 컴포넌트 변수에 자연스러운 Tailwind 이름 공간이 없으면 유형 힌트를 사용하세요. 계산된 값을 복사하지 말고 변수를 기준값으로 유지해요. 예를 들어 Card는 <TRCode>--tr-card-background</TRCode>를 제공해요.

```tsx
<div className="bg-[color:var(--tr-card-background)]">...</div>
```

<h2 className="tr-mdx-h2" id="reference">참고표</h2>
