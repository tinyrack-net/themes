## 계약

표면에는 `Root/Trigger/Portal/Positioner/Popup`을 사용하고 내부에 `Item` 또는 `Separator`를 조합하세요. nested menu는 자체 `Portal/Positioner/Popup` 표면을 `SubmenuRoot`와 `SubmenuTrigger`로 감쌉니다.

## API

`Root`는 controlled / uncontrolled open state, orientation, modality, focus loop, dismissal을 관리합니다. `Trigger`가 menu를 열고 `createHandle`이 분리된 trigger와 typed payload를 root에 연결합니다.

`Portal`, `Backdrop`, `Positioner`, `Popup`, `Arrow`, 선택적 `Viewport`가 collision-aware layer를 구성합니다. `Group`, `GroupLabel`, `Separator`는 내용을 정리합니다. command에는 `Item`, navigation에는 `LinkItem`을 사용하고 link 선택 시 menu를 닫으려면 `closeOnClick`을 지정합니다.

독립 선택에는 `CheckboxItem`과 `CheckboxItemIndicator`를 사용합니다. 단일 선택은 `RadioGroup` 안에 `RadioItem`과 `RadioItemIndicator`를 배치합니다. `SubmenuRoot`와 `SubmenuTrigger`는 nested `Portal` / `Positioner` / `Popup`을 연결합니다. Base UI가 roving focus, typeahead, keyboard activation, disabled item 건너뛰기, portal 위치 지정, outside/Escape dismissal을 담당합니다.
