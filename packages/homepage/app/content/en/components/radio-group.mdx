## Contract

| Axis | Behavior |
| --- | --- |
| label | provide `aria-label`/`aria-labelledby`, or place the group in `TRFieldset.Root` with a legend |
| value | uncontrolled with `defaultValue`, controlled with `value` and `onValueChange` |
| keyboard | all arrow keys move focus and selection, skip disabled radios, and wrap; Tab enters and leaves once |
| state | `disabled` removes interaction and form data; `readOnly` keeps the selected radio focusable without allowing changes |
| form | `name`, `form`, and `required` apply to the group; reset restores an uncontrolled `defaultValue` |
| rendering | the wrapper is SSR-safe and hydrates without changing the selected value |

`TRRadioGroup` uses orientation-independent, two-axis arrow navigation. It does not expose an `orientation` prop; arrange the visual layout with `className` or `style` without changing keyboard behavior.

## API

`TRRadioGroup`, `TRRadioGroupProps`, and `TRRadioGroupState` are exported from `@tinyrack/ui/components/radio-group`; compose the group with `TRRadio.Root` and `TRRadio.Indicator` from the radio module. The root preserves `ref`, `inputRef`, native events and attributes, state-driven `className`/`style`, and polymorphic `render`. `onValueChange(value, details)` receives the originating native event and can call `details.cancel()` to reject a change.
