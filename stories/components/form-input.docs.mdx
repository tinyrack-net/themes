# Input

Native input wrapper for text-like, numeric, date, and time controls.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `input.tr-input` | `Input` | native input element | Use the native input type that matches the data. |
| Size | `data-size` | `size` | `sm`, `md`, `lg`; default `md` | Shares the control scale with Textarea and Select. |
| State | `data-invalid`, `aria-invalid` | `invalid` | `true` or unset; default unset | Pair invalid state with an associated error message. |
| Native | `type` | `type` | native input types; browser default | Preserves browser keyboards, validation, and value behavior. |
| Native | `required`, `disabled`, `aria-describedby` | native input props | pass-through | Put semantics on the native input, not its wrapper. |

## Usage

<ComponentExampleTabs
  id="form-input-basic"
  title="Basic input field"
  description="Pair the native input with a visible label and associated help text."
  previewLayout="stretch"
  preview={(

  sources={[
    {
      label: 'React',

  <Label htmlFor="ops-email">Notification email</Label>
  <Input
    aria-describedby="ops-email-help"
    id="ops-email"
    placeholder="ops@example.com"
    required
    type="email"
  />
  <FormMessage id="ops-email-help">Used for deployment and maintenance alerts.</FormMessage>
</Field>;`,
    },
    {
      label: 'HTML',
      language: 'html',
      code: String.raw`<div class="tr-field" data-size="md">
  <label class="tr-label" for="ops-email">Notification email</label>
  <input
    aria-describedby="ops-email-help"
    class="tr-input"
    data-size="md"
    id="ops-email"
    placeholder="ops@example.com"
    required
    type="email"
  />
  <p class="tr-form-message" id="ops-email-help">
    Used for deployment and maintenance alerts.
  </p>
</div>`,
    },
  ]}
/>

## Guidance

<ComponentGuidance
  whenToUse={[
    'Use Input for single-line values and choose a native type such as email, number, date, or time that matches the data.',
    'Compose production inputs with Field, Label, and FormMessage so help and validation remain close to the control.',
  ]}
  avoid={[
    'Do not use placeholder text as the only accessible name or use a text input when Select or Textarea better matches the task.',
    'Do not set invalid styling without a specific error message and an opportunity to correct the value.',
  ]}
  accessibility={[
    'Associate the visible label with htmlFor/id and help or error text with aria-describedby.',
    'Use native required and disabled attributes, and set invalid/aria-invalid only when validation has failed.',
    'Preserve native keyboard, browser validation, autofill, and input-mode behavior by selecting the correct type.',
  ]}
/>

## API

| Prop | Values | Note |
| --- | --- | --- |
| `size` | `sm`, `md`, `lg` | Default `md`; sets `data-size`. |
| `invalid` | `boolean` | Sets `aria-invalid` and `data-invalid`. |
| `type` | native input type | Passed through to `<input>`. |
| native input props | `InputHTMLAttributes<HTMLInputElement>` except native `size` | Passed through to `<input>`. |
