<h2 className="tr-mdx-h2" id="setup">セットアップ</h2>

Tinyrack を使用するアプリケーションに Tailwind CSS と Vite プラグインをインストールしてください。

```sh
pnpm add -D tailwindcss @tailwindcss/vite
```

アプリケーションの Vite 設定にプラグインを追加してください。

```ts
import tailwindcss from '@tailwindcss/vite';
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [tailwindcss()],
});
```

Tailwind を先に読み込み、続けて Tinyrack のコア CSS を読み込んでください。使用するコンポーネントのスタイルシートは個別に読み込みます。

```css
@import "tailwindcss";
@import "@tinyrack/ui/core.css";
@import "@tinyrack/ui/components/card.css";
```

<h2 className="tr-mdx-h2" id="choose-utilities">値の選択</h2>

1. カラー、タイポグラフィ、スペーシング、角丸、エレベーション、モーションなど、デザインシステムの判断を表す値には、名前付きの <TRCode>tinyrack-*</TRCode> ユーティリティを選んでください。
2. <TRCode>grid</TRCode> や <TRCode>grid-cols-2</TRCode> などの補助的なレイアウトには、Tailwind のデフォルトユーティリティを選んでください。Tinyrack のビューポートしきい値には <TRCode>xs:</TRCode> から <TRCode>xl:</TRCode> までを使います。
3. 実在するコンポーネント変数や一時的な CSS 値に名前付きユーティリティがない場合のみ、任意の値を選んでください。別のトークン体系は作成しません。

<h2 className="tr-mdx-h2" id="practical-example">実践例</h2>

このカードでは、システム上の判断に名前付きの Tinyrack ユーティリティを使い、レイアウトには Tailwind のデフォルトを使います。上記のセットアップ後、そのまま貼り付けられます。

```tsx
export function ClusterHealth() {
  return (
    <section className="grid gap-tinyrack-lg rounded-tinyrack-lg border-tinyrack-default border-tinyrack-border bg-tinyrack-surface p-tinyrack-xl md:grid-cols-2">
      <h2 className="text-tinyrack-xl font-tinyrack-heading">クラスター状態</h2>
      <p className="text-tinyrack-sm text-tinyrack-text-muted">
        すべてのノードを利用できます。
      </p>
    </section>
  );
}
```

<h2 className="tr-mdx-h2" id="arbitrary-values">任意の値</h2>

公開コンポーネント変数に自然な Tailwind 名前空間がない場合は、型ヒントを使ってください。解決済みの値をコピーせず、変数を基準値として維持します。たとえば、Card は <TRCode>--tr-card-background</TRCode> を提供します。

```tsx
<div className="bg-[color:var(--tr-card-background)]">...</div>
```
