## 実装

Components choose elevation by the visual separation their surface needs. Popover, Toast, elevated Card, Switch, and Slider use `raised`; Dialog, Alert Dialog, Drawer, Autocomplete, Context Menu, Navigation Menu, and Preview Card use `overlay`. Use a public shadow token only when building a custom elevated surface; do not infer modality from the shadow or add it to normal layout sections.

```css
.rack-popover {
  box-shadow: var(--tinyrack-shadow-raised);
}
```

```html
<div popover class="rounded-tinyrack-lg shadow-tinyrack-raised">
  Rack actions
</div>
```

tooling や生成 style で TypeScript の同じ値が必要な場合は `tinyrackShadows.raised` と `tinyrackShadows.overlay` を使います。
