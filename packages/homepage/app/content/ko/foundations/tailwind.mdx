<h2 className="tr-mdx-h2" id="setup">설정</h2>

Tailwind를 먼저 불러오고 Tinyrack 핵심 CSS를 이어서 불러옵니다. 핵심 스타일시트는 런타임 <TRCode>--tinyrack-*</TRCode> 변수를 유지하면서 Tailwind v4의 정적 테마 이름 공간으로 노출합니다.

```css
@import "tailwindcss";
@import "@tinyrack/ui/core.css";
```

컴포넌트 스타일은 계속 분리해서 사용한 컴포넌트만 불러옵니다.

```css
@import "@tinyrack/ui/components/button.css";
```

<h2 className="tr-mdx-h2" id="choose-utilities">유틸리티 선택</h2>

- 색상, 타이포그래피, 간격, 형태, 고도, 모션이 디자인 시스템 결정을 나타내면 <TRCode>tinyrack-*</TRCode> 유틸리티를 사용합니다.
- 뷰포트 임계값에는 공통 <TRCode>xs:</TRCode>부터 <TRCode>xl:</TRCode>까지의 반응형 변형을 사용합니다.
- 일회성 격자 열 개수처럼 Tinyrack 토큰을 의미하지 않는 부수적인 레이아웃에는 Tailwind 기본 유틸리티를 유지합니다.
- 그 밖의 Tailwind 기본 테마는 전역에서 재정의하지 않습니다. 접두사가 기존 애플리케이션 코드와 외부 예제의 의미를 안정적으로 보존합니다.

```tsx
<section className="grid gap-tinyrack-lg rounded-tinyrack-lg border-tinyrack-default border-tinyrack-border bg-tinyrack-surface p-tinyrack-xl">
  <h2 className="text-tinyrack-xl font-tinyrack-heading">클러스터 상태</h2>
  <p className="text-tinyrack-sm leading-tinyrack-md text-tinyrack-text-muted">
    모든 노드를 사용할 수 있습니다.
  </p>
</section>
```

<h2 className="tr-mdx-h2" id="arbitrary-values">임의 값</h2>

컴포넌트 범위의 <TRCode>--tr-*</TRCode> 변수나 다른 CSS 값에 자연스러운 Tailwind 테마 이름 공간이 없으면 유형 힌트를 사용합니다. 계산된 값을 복사하지 말고 변수를 기준값으로 유지합니다.

```tsx
<div className="bg-[color:var(--tr-card-background)] [width:var(--tr-layer-anchor-width)]">
  ...
</div>
```

아래 참조에 이름 있는 유틸리티가 있으면 그것을 우선합니다. 임의 값은 의도적인 컴포넌트 맞춤 설정을 위한 것이며 별도의 토큰 체계가 아닙니다.
