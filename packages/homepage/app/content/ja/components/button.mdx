## コントラクト

| 軸 | 値 | 既定値 | 用途 |
| --- | --- | --- | --- |
| `intent` | `neutral`, `primary`, `info`, `success`, `warning`, `danger` | `neutral` | アクションの意図を伝えます。 |
| `appearance` | `solid`, `outline`, `ghost` | `solid` | 意図を変えずに強調度を変更します。 |
| `uiSize` | `sm`, `md`, `lg` | `md` | 周囲のコントロール密度に合わせます。 |
| 状態 | `disabled`, `loading` | 操作可能 | ネイティブの disabled 動作と `aria-busy` を使います。 |

## API

- `TRButton` は Base UI button props、ネイティブ event、`className`、`style`、React 19 `ref` を転送します。
- ネイティブ button の `type` の既定値は `button` です。form 内では `type="submit"` または `type="reset"` を意図的に設定します。`render` で button 以外の要素を使う場合は `nativeButton={false}` を設定し、ネイティブの `type` 既定値は省略されます。
- `loading` は control を無効化し、`aria-hidden` spinner をレンダリングし、名前付き button に `aria-busy` を設定します。待機中の button が `aria-labelledby` を使っていても `loadingLabel` がアクセシブル名になり、loading 中でなければ明示した `aria-busy` 値が保持されます。
- `intent` は `appearance` と独立して semantic color を決定します。neutral の outline と ghost は primary より控えめな muted foreground を使い、すべての intent は対応する hover/pressed surface を維持します。
- deprecated の `variant` prop は互換性を維持します。`secondary` は `neutral` に、`primary` と `danger` は同名の intent にマッピングされます。
