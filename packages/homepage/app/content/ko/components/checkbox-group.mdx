## 계약

| 축 | 동작 |
| --- | --- |
| value | `defaultValue`를 사용하면 비제어형, `value`와 `onValueChange`를 사용하면 제어형으로 동작합니다. |
| form | 각 checkbox에 같은 `name`을 지정합니다. 외부 `form` 소유자를 사용할 때도 체크된 모든 항목이 자신의 `value`를 제공합니다. |
| availability | group의 `disabled`가 하위 항목에 적용되며, 개별 checkbox에는 `readOnly`를 지정할 수 있습니다. |
| accessibility | group에 이름을 지정합니다. `TRCheckbox.Root`를 label로 감싸거나 native input ID를 `htmlFor`로 가리키면 시각적 checkbox role의 이름이 됩니다. |
| parent | `allValues`를 전달하고 `parent` checkbox를 렌더링하면 checked, empty, mixed 상태를 표현할 수 있습니다. |
| reset | native form reset은 `defaultValue`를 복원합니다. 제어형 group의 상태는 계속 `value`가 소유합니다. |
| validation | 제품에서 최소 또는 최대 선택 수가 필요하면 제어형 값과 TRForm/TRField 규칙을 조합합니다. |

  sources={[{ label: 'React', language: 'tsx', code: String.raw`const featureOptions = [
  ['metrics', 'Metrics'],
  ['alerts', 'Alerts'],
  ['backups', 'Automated backups'],
];

function FeatureForm() {
  const [attempted, setAttempted] = useState(false);
  const [selected, setSelected] = useState(['metrics']);
  const [submitted, setSubmitted] = useState('');
  const invalid = attempted && (selected.length < 1 || selected.length > 2);

  return (
    <TRForm onSubmit={(event) => {
      event.preventDefault();
      setAttempted(true);
      const values = new FormData(event.currentTarget).getAll('features');
      setSubmitted(values.length >= 1 && values.length <= 2 ? values.join(', ') : '');
    }}>
      <TRCheckboxGroup
        aria-describedby={invalid ? 'feature-error' : undefined}
        aria-invalid={invalid || undefined}
        aria-label="Included features"
        onValueChange={(values) => { setSelected(values); setSubmitted(''); }}
        value={selected}
      >
        {featureOptions.map(([value, label]) => {
          const inputId = 'feature-' + value;
          const labelId = inputId + '-label';
          return <div key={value}>
            <TRCheckbox.Root
              aria-labelledby={labelId}
              id={inputId}
              name="features"
              value={value}
            >
              <TRCheckbox.Indicator aria-hidden="true">✓</TRCheckbox.Indicator>
            </TRCheckbox.Root>
            <label htmlFor={inputId} id={labelId}>{label}</label>
          </div>;
        })}
      </TRCheckboxGroup>
      {invalid ? <p id="feature-error" role="alert">
        {selected.length < 1 ? 'Select at least one feature.' : 'Select no more than two features.'}
      </p> : null}
      <TRButton type="submit">Save features</TRButton>
      <output aria-live="polite">{submitted}</output>
    </TRForm>
  );
}` }]}
/>

## API

`TRCheckboxGroup`, `TRCheckboxGroupProps`, `TRCheckboxGroupState`는 `@tinyrack/ui/components/checkbox-group`에서 export됩니다. `onValueChange`는 다음 `string[]`와 Base UI event details를 받습니다. 고유한 `value` prop과 접근 가능한 이름을 가진 `TRCheckbox.Root` children을 렌더링하세요.
