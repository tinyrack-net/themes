## コントラクト

この Tinyrack wrapper は Base UI の動作、React 19 refs、ネイティブ props、events、`render` contract を保ちます。Playground の control は `label` と `disabled` を扱います。`open` は component event で同期される preview-owned interaction state のまま維持され、Reset は初期状態と result を復元します。

## API

anatomy は `Root`、`Trigger`、`Portal`、`Backdrop`、`Viewport`、`Popup`、`Title`、`Description`、`Close` です。すべての part と prop/state type は `@tinyrack/ui/components/alert-dialog` から export され、style は対応する CSS subpath で opt-in のままです。

`Root` は常に modal です。open 中は focus が内部に留まり、Escape と `Close` は close を要求しますが、outside interaction では閉じません。close 後は focus が `Trigger` に戻ります。最初の `Close` を安全な Cancel とし、破壊的な処理は danger confirmation の `Close` のみに接続してください。disabled の trigger と close は動作しません。

`Root` の外に trigger を置く場合は `TRAlertDialog.createHandle<Payload>()` で typed bridge を作り、`Trigger` と `Root` の両方に handle、`Trigger` に payload を渡します。`Portal` は custom container を受け取れます。確認コントロールを `tr-alert-dialog-actions` にまとめると、狭い viewport でも読みやすい action に折り返されます。
