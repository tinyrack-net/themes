# Progress

A native progress indicator for determinate work and indeterminate activity.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `tr-progress` on `<progress>` | `Progress` | Native progress semantics | Supply an accessible label for the operation. |
| Size | `data-size` | `size` | `sm`, `md`, `lg`; default `md` | Use size to match the surrounding density. |
| Variant | `data-variant` | `variant` | `neutral`, `primary`, `danger`; default `primary` | Use danger when progress represents a failed or destructive operation. |
| State | Native `value` and `max` | Native progress props | Omit `value` for indeterminate | Keep the browser's native determinate and indeterminate semantics. |

## Guidance

<ComponentGuidance
  whenToUse={[
    'Use determinate progress when the operation exposes a meaningful value and maximum.',
    'Use indeterminate progress when the operation is active but the completion amount is unknown.',
  ]}
  avoid={[
    'Do not use progress as a decorative loading animation without an operation label.',
    'Do not communicate failure only through color; pair danger progress with text or an Alert.',
  ]}
  accessibility={[
    'Provide an accessible label through aria-label or an associated visible label.',
    'Keep native value, max, and indeterminate behavior intact.',
  ]}
/>
