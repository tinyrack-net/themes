# Avatar

A compact identity surface for users, racks, nodes, and other operational resources.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `tr-avatar` on `<span>` | `Avatar` | Child content is caller-owned | Use initials or an image with an appropriate accessible name. |
| Size | `data-size` | `size` | `sm`, `md`, `lg`; default `md` | Match avatar size to the adjacent control or metadata density. |
| Shape | `data-shape` | `shape` | `circle`, `square`; default `circle` | Use square for resource thumbnails and circle for people. |
| Image | Child `<img>` | Native child content | `object-fit: cover` | Provide meaningful alt text for informative images. |

## Guidance

<ComponentGuidance
  whenToUse={[
    'Use Avatar for compact identity or resource context beside a name, status, or action.',
    'Use initials when no image is available and keep them short enough for the chosen size.',
  ]}
  avoid={[
    'Do not add image loading or fallback behavior to this presentational primitive.',
    'Do not use Avatar as the only label for a critical action.',
  ]}
  accessibility={[
    'Give informative images meaningful alt text and give initials an aria-label when needed.',
    'Mark decorative avatar content as hidden when nearby text already supplies the identity.',
  ]}
/>
