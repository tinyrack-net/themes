## コントラクト

| 軸 | 値 | 既定値 |
| --- | --- | --- |
| state | controlled `open` or uncontrolled `defaultOpen` | closed |
| availability | `disabled` | interactive |
| lifecycle | unmount, `keepMounted`, or `hiddenUntilFound` | unmount |

## API

### `TRCollapsible.Root`

| Prop | 型 / 既定値 | 用途 |
| --- | --- | --- |
| `open` / `defaultOpen` | `boolean` / `undefined`, `false` | controlled expansion または uncontrolled の初期状態 |
| `onOpenChange` | `(open, details) => void` | trigger と browser find による変更を通知 |
| `disabled` | `boolean` / `false` | trigger を focus で検出可能に保ちつつ状態変更を無効化 |
| `render`, `className`, `style`, `ref` | Base UI root props | 動作を保ったまま `<div>` host をカスタマイズ |

### `TRCollapsible.Trigger`

`aria-expanded` と `aria-controls` を持つ button を render します。native button props、`render`、style、ref を受け取ります。Enter と Space で開閉しても focus は trigger に残り、disabled trigger は `data-disabled` と `aria-disabled` を公開します。

### `TRCollapsible.Panel`

閉じた content を DOM に残す場合は `keepMounted`、browser page search から表示可能にする場合は `hiddenUntilFound` を使います。両方を指定すると `hiddenUntilFound` が優先されます。既定では closing transition 後に unmount します。panel は `--collapsible-panel-height`、`--collapsible-panel-width`、`data-starting-style`、`data-ending-style` を公開します。

閉じた SSR output は選択した lifecycle に従い、public relationship を変えずに hydrate されます。Tinyrack CSS は design-system motion token を使用し、reduced motion では装飾的な transition を無効にします。`TRCollapsibleRoot`、`TRCollapsibleTrigger`、`TRCollapsiblePanel`、各 prop type と Base UI state type は `@tinyrack/ui/components/collapsible` から export され、CSS は opt-in です。
