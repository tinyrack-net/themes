# Colors

Role-based colors keep compact operational surfaces readable across light and dark themes without turning raw palette steps into product decisions.

## Implementation

Colors are available through all three core surfaces. Prefer semantic CSS variables or prefixed utilities in rendered UI, and use TypeScript metadata for tooling or non-CSS consumers.

```css
@import "tailwindcss";
@import "@tinyrack/ui/core/core.css";

.rack-panel {
  background: var(--tinyrack-surface);
  color: var(--tinyrack-text);
  border-color: var(--tinyrack-border);
}
```

```html
<section class="border-tinyrack-border bg-tinyrack-surface text-tinyrack-text">
  <span class="text-tinyrack-text-muted">Backup completed</span>
</section>
```

```ts
import { tinyrackPalettes, tinyrackSemanticColors } from '@tinyrack/ui/core';

const darkAction = tinyrackSemanticColors.dark.primary;
const neutral950 = tinyrackPalettes.neutral[950];
```
