# Alert

An operational message surface for neutral guidance and semantic status feedback.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `tr-alert` on `<div>` | `Alert` | Required message surface | Keep the message short and actionable. |
| Variant | `data-variant` | `variant` | `neutral`, `info`, `success`, `warning`, `danger`; default `neutral` | Match the status meaning and reserve danger for errors. |
| Native | Native `<div>` attributes | `HTMLAttributes<HTMLDivElement>` | Passed through | Caller chooses `role="status"` or `role="alert"` when announcement is needed. |

## Guidance

<ComponentGuidance
  whenToUse={[
    'Use Alert when a message belongs to the current operational context and needs a bounded surface.',
    'Choose role="status" for informative updates and role="alert" for urgent errors.',
  ]}
  avoid={[
    'Do not announce every static message as an alert.',
    'Do not choose success, warning, info, or danger only for visual emphasis; match the message meaning.',
  ]}
  accessibility={[
    'Choose the live-region role based on urgency rather than visual variant alone.',
    'Make the message understandable without relying on border or text color.',
  ]}
/>
