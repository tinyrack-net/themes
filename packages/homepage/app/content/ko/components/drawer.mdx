## 계약

`TRDrawer`는 Base UI의 controlled/uncontrolled state, swipe와 snap-point gesture, focus 관리, keyboard dismissal, portal, React 19 ref와 `render` prop을 보존해요. motion은 `swipeDirection`과 reduced-motion 설정을 따라요. Playground는 drawer event로 `open`과 `snapPoint`를 동기화하며 Reset은 닫힌 full-height downward 기본값을 복원해요.

## 구조

`Root`는 state를 소유하고 `Trigger`는 drawer를 열어요. `Portal`은 overlay를 body 또는 지정 container에 mount해요. `Backdrop`, `Viewport`, `Popup`, scroll 가능한 `Content`가 modal surface를 구성해요. `Title`과 `Description`은 dialog label을 제공하고 `Close`는 닫아요. `SwipeArea`는 edge opening을 제공해요. `Provider`, `Indent`, `IndentBackground`는 nested page effect를 조정하고 `VirtualKeyboardProvider`는 mobile keyboard viewport change를 처리해요. `createHandle`은 typed payload를 사용하는 detached trigger를 연결해요.

## 동작

- controlled visibility에는 `open`과 `onOpenChange`, uncontrolled visibility에는 `defaultOpen`을 사용하세요.
- controlled position에는 `snapPoint`와 `onSnapPointChange`, uncontrolled position에는 `defaultSnapPoint`를 사용하고 허용값은 `snapPoints`에 둡니다.
- `swipeDirection`은 edge와 dismissal 방향을 정해요. `snapToSequentialPoints`는 velocity 기반 point skipping을 막아요.
- Escape, `Close`, backdrop press, non-modal focus-out, swipe가 기본 dismissal이에요. outside pointer dismissal을 막으려면 `disablePointerDismissal`을 사용하세요.
- `modal` 기본값은 `true`예요. outside interaction에는 `false`, scroll lock 없이 focus만 가두려면 `trap-focus`를 사용하세요.
- `Popup`의 `initialFocus`와 `finalFocus`로 focus 이동을 지정하세요. 기본적으로 focus는 drawer로 들어가고 trigger로 돌아가요.
- `actionsRef`는 imperative `close`와 `unmount`를 제공해요. 일반 flow에서는 declarative state를 사용하세요.
- `Portal`은 client에서 mount되고 닫힌 SSR markup은 hydration-safe해요. `Viewport`는 visual viewport와 safe area 안에 surface를 유지해요.

## API

모든 compound part와 prop/state type은 `@tinyrack/ui/components/drawer`에서 export돼요. CSS는 `@tinyrack/ui/components/drawer.css`로 opt-in하며 `--tr-drawer-popup-max-width`, `--tr-drawer-popup-background`, `--tr-drawer-backdrop`, `--tr-drawer-control-background`, `--tr-drawer-motion-duration`, `--tr-drawer-motion-easing`으로 appearance와 motion을 override할 수 있어요.
