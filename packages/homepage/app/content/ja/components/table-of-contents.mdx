## 主なプロパティ

順序付けた h2/h3 の見出しデータと現在の scroll-spy の結果を渡します。depth 3 の項目は直前の depth 2 の項目の下にネストされます。`renderLink` で router link を注入でき、adapter がなければデスクトップ link とモバイル select は hash navigation を使用します。

intersection observer から `currentHeading` を更新します。link はネイティブの keyboard 操作を維持し、`onNavigate` で router または custom scroller と controlled active state を同期できます。`items` が空の場合は `null` を返し、狭い layout では長い label の overflow を抑えます。

## API

`items` は順序付けた `{ id, label, depth }` collection で、`depth` は `2 | 3` に制限されます。`currentHeading` は active link に `aria-current="location"` を設定し、モバイルの選択を制御します。`label` は navigation landmark とデスクトップ heading、`mobileLabel` はモバイル select の名前です。`onNavigate` は選択項目を受け取り、`renderLink` はデスクトップ用 router link を提供します。Native nav props、events、styles、`className`、`ref` は引き継がれます。同じデータからデスクトップ list と keyboard 操作可能なモバイル select をレンダリングし、SSR/hydration に対応します。
