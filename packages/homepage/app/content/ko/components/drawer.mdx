## 계약

`TRDrawer`는 Base UI의 controlled/uncontrolled state, swipe와 snap-point gesture, focus 관리, keyboard dismissal, portal, React 19 ref와 `render` prop을 보존합니다. motion은 `swipeDirection`과 reduced-motion 설정을 따릅니다. Playground는 drawer event로 `open`과 `snapPoint`를 동기화하며 Reset은 닫힌 full-height downward 기본값을 복원합니다.

## 구조

`Root`는 state를 소유하고 `Trigger`는 drawer를 엽니다. `Portal`은 overlay를 body 또는 지정 container에 mount합니다. `Backdrop`, `Viewport`, `Popup`, scroll 가능한 `Content`가 modal surface를 구성합니다. `Title`과 `Description`은 dialog label을 제공하고 `Close`는 닫습니다. `SwipeArea`는 edge opening을 제공합니다. `Provider`, `Indent`, `IndentBackground`는 nested page effect를 조정합니다. `VirtualKeyboardProvider`는 mobile keyboard viewport change를 처리합니다. `createHandle`은 typed payload를 사용하는 detached trigger를 연결합니다.

## 동작

- controlled visibility에는 `open`과 `onOpenChange`, uncontrolled visibility에는 `defaultOpen`을 사용합니다.
- controlled position에는 `snapPoint`와 `onSnapPointChange`, uncontrolled position에는 `defaultSnapPoint`를 사용하고 허용값은 `snapPoints`에 둡니다.
- `swipeDirection`은 edge와 dismissal 방향을 정합니다. `snapToSequentialPoints`는 velocity 기반 point skipping을 막습니다.
- Escape, `Close`, backdrop press, non-modal focus-out, swipe가 기본 dismissal입니다. outside pointer dismissal을 막으려면 `disablePointerDismissal`을 사용합니다.
- `modal` 기본값은 `true`입니다. outside interaction에는 `false`, scroll lock 없이 focus만 가두려면 `trap-focus`를 사용합니다.
- `Popup`의 `initialFocus`와 `finalFocus`로 focus 이동을 지정합니다. 기본적으로 focus는 drawer로 들어가고 trigger로 돌아갑니다.
- `actionsRef`는 imperative `close`와 `unmount`를 제공합니다. 일반 flow에서는 declarative state를 권장합니다.
- `Portal`은 client에서 mount되고 닫힌 SSR markup은 hydration-safe합니다. `Viewport`는 visual viewport와 safe area 안에 surface를 유지합니다.

## API

모든 compound part와 prop/state type은 `@tinyrack/ui/components/drawer`에서 export됩니다. CSS는 `@tinyrack/ui/components/drawer.css`로 opt-in하며 `--tr-drawer-popup-max-width`, `--tr-drawer-popup-background`, `--tr-drawer-backdrop`, `--tr-drawer-control-background`, `--tr-drawer-motion-duration`, `--tr-drawer-motion-easing`으로 appearance와 motion을 override할 수 있습니다.
