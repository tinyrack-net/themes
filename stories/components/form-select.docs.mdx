# Select

Native select for compact option sets, sharing text control size and state styling.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `select.tr-select` | `Select` | native select element | Use native option elements for a compact choice list. |
| Part | `option`, `optgroup` | native children | browser behavior | Give every option a concise, distinct label. |
| Size | `data-size` | `size` | `sm`, `md`, `lg`; default `md` | Shares the control scale with Input and Textarea. |
| State | `data-invalid`, `aria-invalid` | `invalid` | `true` or unset; default unset | Pair invalid state with an associated error message. |
| Native | `required`, `disabled`, `aria-describedby` | native select props | pass-through | Keeps browser selection, validation, and keyboard semantics. |

## Usage

<ComponentExampleTabs
  id="form-select-basic"
  title="Basic select field"
  description="Pair native options with a visible label and associated selection guidance."
  previewLayout="stretch"
  preview={(
    <Field>
      <Label htmlFor="deploy-region">Deployment region</Label>
      <Select aria-describedby="deploy-region-help" defaultValue="seoul" id="deploy-region">
        <option value="seoul">Seoul</option>
        <option value="tokyo">Tokyo</option>
      </Select>
      <FormMessage id="deploy-region-help">New workloads start in this region.</FormMessage>
    </Field>
  )}
  sources={[
    {
      label: 'React',
      language: 'tsx',
      code: String.raw`import '@tinyrack/ui/core/core.css';
import '@tinyrack/ui/components/form/form.css';
import { Field, FormMessage, Label, Select } from '@tinyrack/ui/components/form/react';

<Field>
  <Label htmlFor="deploy-region">Deployment region</Label>
  <Select aria-describedby="deploy-region-help" defaultValue="seoul" id="deploy-region">
    <option value="seoul">Seoul</option>
    <option value="tokyo">Tokyo</option>
  </Select>
  <FormMessage id="deploy-region-help">New workloads start in this region.</FormMessage>
</Field>;`,
    },
    {
      label: 'HTML',
      language: 'html',
      code: String.raw`<div class="tr-field" data-size="md">
  <label class="tr-label" for="deploy-region">Deployment region</label>
  <select
    aria-describedby="deploy-region-help"
    class="tr-select"
    data-size="md"
    id="deploy-region"
  >
    <option value="seoul">Seoul</option>
    <option value="tokyo">Tokyo</option>
  </select>
  <p class="tr-form-message" id="deploy-region-help">
    New workloads start in this region.
  </p>
</div>`,
    },
  ]}
/>

## Guidance

<ComponentGuidance
  whenToUse={[
    'Use Select for a compact list of mutually exclusive options whose labels are known in advance.',
    'Compose it with Field, Label, and FormMessage when the choice needs persistent help or validation text.',
  ]}
  avoid={[
    'Do not use Select when users need to compare many options at once or enter a value that is not in the list.',
    'Do not use a disabled placeholder option as the only label or rely on invalid color without an error message.',
  ]}
  accessibility={[
    'Associate a visible label with htmlFor/id and help or error text with aria-describedby.',
    'Use native required, disabled, option, and optgroup semantics; set invalid/aria-invalid only after validation fails.',
    'Preserve the browser select keyboard behavior, including focus, arrow-key selection, and type-ahead.',
  ]}
/>

## API

| Prop | Values | Note |
| --- | --- | --- |
| `size` | `sm`, `md`, `lg` | Default `md`; sets `data-size`. |
| `invalid` | `boolean` | Sets `aria-invalid` and `data-invalid`. |
| `children` | native `option` elements | Keep browser select semantics. |
| native select props | `SelectHTMLAttributes<HTMLSelectElement>` except native `size` | Passed through to `<select>`. |
