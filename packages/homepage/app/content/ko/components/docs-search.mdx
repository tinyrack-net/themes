## Contract

`TRDocsSearch.Trigger`는 독립 실행형 버튼이고 `TRDocsSearch.Dialog`는 제어되는 포털 검색 화면입니다. Dialog는 Ctrl/⌘+K, 로딩, 빈 결과, 오류, fallback, 키보드 선택, 포커스 복원, 오래된 요청 취소를 관리합니다. 데이터 소스는 `onSearch(query, signal)`이 관리하며, 취소 가능한 작업에 전달받은 `AbortSignal`을 사용해야 합니다.

## Examples

- 포커스는 combobox에 유지되며 `ArrowDown`과 `ArrowUp`으로 활성 결과를 이동합니다. `Enter`는 결과를 선택하고 `Escape`는 dialog를 닫습니다.
- `enableShortcut={false}`는 전역 단축키를 끕니다. 임베드된 preview 또는 disabled trigger에서 모든 열기 경로를 막을 때 사용하세요.
- `compact`는 접근 가능한 이름을 유지하면서 표시 레이블과 단축키를 숨깁니다. `uiSize="sm" | "md" | "lg"`에서 compact trigger는 정사각형을 유지합니다.
- Dialog는 `document.body`에 포털됩니다. 닫힌 뒤 trigger로 포커스를 복원하려면 `returnFocusRef`를 전달하세요.
- 닫힌 dialog와 trigger는 SSR에 안전하며, 열리기 전에는 포털을 렌더링하지 않고 hydration됩니다.
- 기본 검색 색인을 사용할 수 없을 때 축소된 로컬 결과가 표시되는 이유를 설명하려면 `fallback`을 설정하세요. 검색 요청이 reject되면 Usage 소스에 나온 지역화된 `messages.error` 상태를 사용합니다.

## API

| 내보내기 | 주요 props | 용도 |
| --- | --- | --- |
| `TRDocsSearch` | `Trigger`, `Dialog` | 복합 namespace입니다. |
| `TRDocsSearch.Trigger` | `compact`, `label`, `shortcutLabel`, `uiSize`, button props, `ref` | 표시형 또는 compact 검색 버튼입니다. `disabled`는 버튼 활성화를 막습니다. |
| `TRDocsSearch.Dialog` | `open`, `onOpenChange`, `onSearch`, `onSelect`, `messages`, `fallback`, `enableShortcut`, `popupRef`, `returnFocusRef` | 제어되는 검색 dialog와 결과 목록입니다. |
| `TRDocsSearchResult` | `id`, `url`, `title`, `section`, `excerpt`, 선택적 일치 범위 | Dialog가 렌더링하는 검색 결과입니다. 일치 범위의 끝 offset은 포함하지 않습니다. |
| `TRDocsSearchMessages` | `title`, `trigger`, `placeholder`, `close`, `results`, `idle`, `loading`, `empty`, `error`, `fallback` | 표시 및 접근성 문구 전체를 지역화합니다. |

`onSearch`는 `(query: string, signal: AbortSignal)`을 받고 readonly 결과 promise를 반환합니다. Reject되면 `messages.error`가 표시되고, 취소되거나 교체된 요청은 최신 결과를 덮어쓰지 않습니다.
