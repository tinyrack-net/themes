## 계약

`TRInput`은 네이티브 `<input>`을 렌더링하며 props, ref, 이벤트, 유효성 및 form 동작을 보존합니다. label은 별도의 네이티브 요소입니다. 보이는 `<label>`을 `htmlFor`와 `id`로 연결하고 안내나 오류에는 `aria-describedby`를 사용하세요.

밀도 높은 컨트롤에는 `uiSize="sm"`, 일반 form에는 기본값인 `md`, 더 큰 조작 영역이 필요할 때는 `lg`를 사용하세요. `disabled`, `readOnly`, `required`, `aria-invalid`는 네이티브 의미와 Tinyrack 상태 스타일을 유지합니다.

  title="필수 제출과 복구"
  preview={<Stories.InputValidationPreview />}
  sources={[{ label: 'React', language: 'tsx', code: String.raw`import '@tinyrack/ui/components/button.css';
import '@tinyrack/ui/components/form.css';
import '@tinyrack/ui/components/input.css';
import { TRButton } from '@tinyrack/ui/components/button';
import { TRForm } from '@tinyrack/ui/components/form';
import { TRInput } from '@tinyrack/ui/components/input';
import { useId, useRef, useState } from 'react';

export function RequiredInput() {
  const inputId = useId();
  const errorId = useId();
  const inputRef = useRef<HTMLInputElement>(null);
  const [attempted, setAttempted] = useState(false);
  const [submitted, setSubmitted] = useState(false);
  const [value, setValue] = useState('');
  const invalid = attempted && value.trim().length === 0;

  return (
    <TRForm
      noValidate
      onSubmit={(event) => {
        event.preventDefault();
        const valid = event.currentTarget.checkValidity();
        setAttempted(true);
        setSubmitted(valid);
        if (!valid) inputRef.current?.focus();
      }}
    >
      <label htmlFor={inputId}>
        Rack name
        <TRInput
          aria-describedby={invalid ? errorId : undefined}
          aria-invalid={invalid || undefined}
          id={inputId}
          name="rack"
          onValueChange={(nextValue) => {
            setValue(nextValue);
            setSubmitted(false);
          }}
          onInvalid={(event) => event.preventDefault()}
          ref={inputRef}
          required
          value={value}
        />
      </label>
      {invalid ? <p id={errorId} role="alert">Rack name is required.</p> : null}
      <TRButton type="submit">Continue</TRButton>
      <output aria-live="polite">
        {submitted ? 'Ready to create ' + value + '.' : ''}
      </output>
    </TRForm>
  );
}` }]}
/>

## API

`TRInputProps`는 Base UI Input props를 확장하고 기본값이 `md`인 `uiSize?: 'sm' | 'md' | 'lg'`를 추가합니다. `name`, `type`, `autoComplete`, `disabled`, `readOnly`, `required` 같은 네이티브 input 속성과 `ref`, `className`, `style`, 이벤트 핸들러를 전달합니다.

제어된 텍스트에는 `value`와 `onValueChange(value, details)`를 함께 사용하고, 네이티브 비제어 및 form reset 동작에는 `defaultValue`를 사용하세요. 이벤트 객체가 필요하면 네이티브 `onChange`를 사용합니다. `aria-invalid="true"`, Base UI의 `data-invalid`, 네이티브 `:invalid`는 모두 invalid border를 활성화합니다. `render` prop은 렌더링 요소를 교체하며 Base UI가 동작, 상태, ref, `.tr-input` 스타일을 병합합니다.
