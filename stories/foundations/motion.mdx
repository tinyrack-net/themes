# Motion

Motion explains state change without slowing down compact operational work.

## Principle

Use the fast duration for direct interaction, normal for anchored layers, slow for modal entry and exit, and loading for repeating progress or skeleton feedback. Reduced-motion preferences always remove decorative transitions.

## Applied pattern

<div className="flex flex-wrap gap-tinyrack-sm"><Button>Fast interaction</Button><Button style={{ ['--tr-btn-duration']: tinyrackMotion.duration.slow }}>Deliberate exception</Button></div>

## Implementation

Components consume duration and easing variables. Prefer the semantic duration names over authoring milliseconds inside component CSS.

```css
.rack-layer {
  transition: opacity var(--tinyrack-duration-normal) var(--tinyrack-ease-out);
}
```
