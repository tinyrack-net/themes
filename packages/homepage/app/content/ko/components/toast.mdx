## API

`TRToast`는 `Provider`, `Portal`, `Viewport`, `Root`, `Content`, `Title`, `Description`, `Action`, `Close`, `Positioner`, `Arrow`를 포함합니다. `createToastManager`와 `useToastManager`도 제공하며 named helper export로도 사용할 수 있습니다.

`Provider`는 `timeout`, `limit`, global `toastManager`를 받습니다. `useToastManager`는 reactive `toasts`와 `add`, `update`, `close`, `promise`를 반환합니다. `add`에는 title, description, 의미형 `type`, timeout, priority, callback, `actionProps`, custom data, anchor `positionerProps`를 전달할 수 있습니다. 같은 `id`를 다시 쓰면 기존 toast를 update합니다. `close(id)`는 하나를, `close()`는 전체를 닫습니다.

Stacked와 anchored notification을 함께 쓸 때는 별도 provider로 분리하세요. 동시에 표시하는 각 `Viewport`에는 고유한 `aria-label`을 지정하세요. Base UI는 timer, queue, swipe dismiss, F6 focus 진입/복원, Escape dismiss, portal, polite/high-priority live announcement를 담당하고 Tinyrack은 variant, token, layout, motion을 담당합니다.
