## Content Security Policy

Generate a fresh nonce for every server response, include the same value in the
HTTP `Content-Security-Policy` header, and pass it through the server render and
hydration without changing it. `CSPProvider` does not create the HTTP header.

```tsx
import { randomBytes } from 'node:crypto';
import type { ReactNode } from 'react';
import { CSPProvider } from '@tinyrack/ui/providers/csp';

export function createRequestCsp() {
  const nonce = randomBytes(16).toString('base64');

  return {
    nonce,
    header: [
      "default-src 'self'",
      `style-src 'self' 'nonce-${nonce}'`,
      `script-src 'self' 'nonce-${nonce}'`,
    ].join('; '),
  };
}

export function AppProviders({
  children,
  nonce,
}: {
  children: ReactNode;
  nonce: string;
}) {
  return <CSPProvider nonce={nonce}>{children}</CSPProvider>;
}
```

Call `createRequestCsp()` once for a request, put its `header` on that same HTTP
response, pass its `nonce` to the server-rendered `AppProviders`, and serialize
the same nonce for hydration. Never generate a second client nonce. Adapt the
directives to the application's complete policy rather than replacing an
existing CSP with this minimal example.

When inline style elements are forbidden, omit the nonce and opt out explicitly:

```tsx
import type { ReactNode } from 'react';
import { CSPProvider } from '@tinyrack/ui/providers/csp';

export function AppProviders({ children }: { children: ReactNode }) {
  return <CSPProvider disableStyleElements>{children}</CSPProvider>;
}
```

The application then owns any styling those Base UI internals would otherwise
inject.

## Text direction

Keep native document direction and Base UI context synchronized. The `dir`
attribute controls browser text/layout semantics; `DirectionProvider` controls
direction-aware Base UI behavior. `useDirection` reads only the provider value.

```tsx
import type { ReactNode } from 'react';
import { DirectionProvider } from '@tinyrack/ui/providers/direction';

export function DirectionDocument({
  children,
  direction,
}: {
  children: ReactNode;
  direction: 'ltr' | 'rtl';
}) {
  return (
    <html dir={direction} lang={direction === 'rtl' ? 'ar' : 'en'}>
      <body>
        <DirectionProvider direction={direction}>{children}</DirectionProvider>
      </body>
    </html>
  );
}
```

Provider imports are intentionally separate from component imports. They do not
ship CSS and are not re-exported from the package root. Import `useDirection`
from the direction subpath only inside descendants that need the active Base UI
context value.
