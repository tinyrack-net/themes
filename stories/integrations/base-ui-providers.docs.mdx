# Base UI Providers

Tinyrack exposes the two application-level Base UI providers without adding a
component barrel. Place them near the root of the React application when their
behavior is needed.

## Content Security Policy

Use `CSPProvider` when a nonce is required for inline style elements created by
Base UI positioning and interaction internals.

```tsx
import { CSPProvider } from '@tinyrack/ui/providers/csp';

<CSPProvider nonce={cspNonce}>
  <App />
</CSPProvider>;
```

## Text direction

`DirectionProvider` supplies a consistent `ltr` or `rtl` direction to every
Tinyrack component backed by Base UI. `useDirection` reads the active value.

```tsx
import {
  DirectionProvider,
  useDirection,
} from '@tinyrack/ui/providers/direction';

<DirectionProvider direction="rtl">
  <App />
</DirectionProvider>;
```

Provider imports are intentionally separate from component imports. They do not
ship CSS and are not re-exported from the package root.
