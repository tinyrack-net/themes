# Tinyrack UI

A CSS-first design system for compact operational interfaces. Start with semantic foundations, import only the component CSS you render, and keep native markup at the center of every implementation.

## Principles

## Start in 5 minutes

Choose the surface your app actually renders. Every path starts with `@tinyrack/ui`; optional peers stay explicit.

<ComponentInstall
  surfaces={[
    {
      label: 'CSS / HTML',
      install: 'pnpm add @tinyrack/ui tailwindcss',
      imports: [
        '@import "tailwindcss";',
        '@import "@tinyrack/ui/core/core.css";',
        '@import "@tinyrack/ui/components/button/button.css";',
      ],
      note: 'Use prefixed Tinyrack utilities for foundation roles and class contracts for owned components.',
    },
    {
      label: 'React',

        "import '@tinyrack/ui/core/core.css';",

        "import { Button } from '@tinyrack/ui/components/button/react';",
      ],
      note: 'React wrappers preserve native element props and require the same explicit CSS imports.',
    },
    {
      label: 'React MDX',

        "import '@tinyrack/ui/core/core.css';",

      ],
      note: 'Pass the Astro component map to rendered content; highlighting remains server-side.',
    },
  ]}

## Use boundary

<div className="grid gap-3 md:grid-cols-2">
  <section className="grid content-start gap-2 border border-tinyrack-border p-4">
    <h3 className="m-0 text-tinyrack-lg font-semibold leading-tinyrack-sm">Use Tinyrack UI when</h3>
    <ul className="m-0 grid gap-2 pl-5 text-tinyrack-sm leading-tinyrack-md text-tinyrack-text-muted">
      <li>The product needs compact operational hierarchy and durable light/dark roles.</li>
      <li>The app can compose explicit CSS imports and native markup.</li>
      <li>Owned component behavior benefits from small isolated React wrappers.</li>
    </ul>
  </section>
  <section className="grid content-start gap-2 border border-tinyrack-border p-4">
    <h3 className="m-0 text-tinyrack-lg font-semibold leading-tinyrack-sm">Choose another layer when</h3>
    <ul className="m-0 grid gap-2 pl-5 text-tinyrack-sm leading-tinyrack-md text-tinyrack-text-muted">
      <li>The goal is to wrap another component system or expose root-level React exports.</li>
      <li>The app expects visual primitives that are not part of the public Tinyrack contract.</li>
      <li>A framework adapter would hide the native element behavior the workflow depends on.</li>
    </ul>
  </section>
</div>
