# Typography

<div className="tr-doc-lead">
  Named type roles for dense operational UI that needs hierarchy without noise.
</div>

## Role ladder

| Role | Token recipe | Use |
| --- | --- | --- |
| Display | `5xl / sm / none` | One major page title or high-level documentation entry. |
| Heading large | `3xl / sm / none` | Top-level product sections and dense dashboard groups. |
| Heading medium | `2xl / sm / none` | Panel titles, settings groups, and documentation subsections. |
| Heading small | `lg / sm / none` | Compact panel headings where the surrounding UI is already dense. |
| Body | `md / md / none` | Default reading text, form copy, and operational explanation. |
| Body small | `sm / md / none` | Dense panels, table cells, helper text, and secondary summaries. |
| Caption | `xs / sm / none` | Timestamps, minor metadata, and low-emphasis support. |
| Label | `xs / xs / lg` | Short uppercase labels, status eyebrows, and compact field labels. |
| Code | `sm / lg / none` | Commands, package names, log fragments, and terse config values. |

## Reading rules

| Rule | Default | Why |
| --- | --- | --- |
| Hierarchy | Weight before color | Size and weight establish order. Color is not the only emphasis. |
| Line length | Short columns | Operational copy should scan inside panels and forms. |
| Labels | Terse uppercase | Tracking is only for short labels, never reading text. |
| Code | Mono for commands | Commands and package paths need a different texture from prose. |
