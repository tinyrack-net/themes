## Contract

Callouts separate supporting guidance from the main document flow. Use `note` for context, `tip` for optional advice, `caution` for avoidable risk, and `danger` when an action can cause serious harm such as data loss. They are static content by default, not live announcements.

## API

### Content and variants

| Prop | Type | Default | Purpose |
| --- | --- | --- | --- |
| `children` | `ReactNode` | Required | Inline or block-level guidance content. |
| `variant` | `'note' \| 'tip' \| 'caution' \| 'danger'` | `'note'` | Selects the semantic heading and visual treatment. |
| `title` | `ReactNode` | Variant heading | Replaces the default English heading, including for localization. |

### HTML and live regions

`TRCallout` also forwards native `div` props, events, `className`, `style`, and React 19 `ref`. For dynamically inserted urgent or polite announcements, explicitly provide `role="alert"` or `role="status"`; do not add a live-region role to static document guidance.
