## Contract

`TRColorSchemeToggle` is a controlled two-state button. Its icon and accessible name describe the next action: `light` offers dark, and `dark` offers light. The component never reads `localStorage`, queries the `system` preference, or mutates the document; resolve and persist those concerns in the application adapter.

## API

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `value` | `"light" \| "dark"` | required | Current resolved color scheme. |
| `onValueChange` | `(value: TRColorScheme) => void` | required | Receives the opposite scheme after uncancelled activation. |
| `darkLabel` | `string` | `"Use dark color scheme"` | Accessible action label while the next value is dark. |
| `lightLabel` | `string` | `"Use light color scheme"` | Accessible action label while the next value is light. |
| `aria-labelledby` | `string` | — | Replaces the generated label with an external accessible name. |
| `disabled` | `boolean` | `false` | Prevents focus and activation. |
| `uiSize` | `"sm" \| "md" \| "lg"` | `"md"` | Inherited IconButton size. |
| `ref` | `Ref<HTMLButtonElement>` | — | Ref forwarded to the button. |
| `onClick` | `MouseEventHandler<HTMLButtonElement>` | — | Runs before `onValueChange`; `preventDefault()` cancels the value change. |

Other safe `TRIconButton` button attributes, native events, `className`, `style`, and customization tokens are inherited. The visual appearance remains `ghost` and the control remains a native button.
