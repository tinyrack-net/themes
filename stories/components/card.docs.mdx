# Card

An operational surface for grouping related content without adding layout-specific subcomponents.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `tr-card` on `<div>` | `Card` | Required surface; variant default `default` | Use for a bounded group of related content. |
| Variant | `data-variant` | `variant` | `default`, `muted`; default `default` | Use muted surfaces for secondary operational context. |
| Padding | `data-padding` | `padding` | `sm`, `md`, `lg`; default `md` | Match padding to content density. |
| Native | Native `<div>` attributes | `HTMLAttributes<HTMLDivElement>` | Passed through | Keep card semantics supplied by the surrounding heading and content. |

## Guidance

<ComponentGuidance
  whenToUse={[
    'Use Card for a bounded operational surface such as a resource summary or log block.',
    'Keep headings, actions, and layout composition in the card content.',
  ]}
  avoid={[
    'Do not use Card as a page-wide layout primitive; use Tailwind layout utilities for that.',
    'Do not create nested cards when a spacing or divider relationship is enough.',
