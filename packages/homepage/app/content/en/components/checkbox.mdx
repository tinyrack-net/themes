## Contract

| Axis | Values |
| --- | --- |
| state | unchecked, checked, `indeterminate` |
| availability | enabled, `readOnly`, `disabled` |
| size | `sm`, `md`, `lg` |
| form | `name`, `value`, `uncheckedValue`, `form`, `required`, reset and native validity |

Use a visible label or an accessible name. Space toggles a focused checkbox; Enter follows native checkbox behavior and does not toggle it. Use `readOnly` when a value must remain focusable and submitted, and `disabled` when it must be unavailable and omitted from submission.

  const invalid = attempted && !checked;

  return (
    <TRForm noValidate onSubmit={(event) => { event.preventDefault(); event.currentTarget.checkValidity(); setAttempted(true); }}>

          <TRCheckbox.Root checked={checked} name="terms" onCheckedChange={setChecked} required>
            <TRCheckbox.Indicator aria-hidden="true">✓</TRCheckbox.Indicator>
          </TRCheckbox.Root>
          I accept the maintenance window.
        </TRField.Label>
        {invalid ? <TRField.Error match>Accept the maintenance window to continue.</TRField.Error> : null}
      </TRField.Root>
      <TRButton type="submit">Continue</TRButton>
    </TRForm>
  );
}` }]}

## API

### `TRCheckbox.Root`

| Prop | Type / default | Purpose |
| --- | --- | --- |
| `checked` / `defaultChecked` | `boolean` / `undefined`, `false` | controlled or uncontrolled selection |
| `indeterminate` | `boolean` / `false` | exposes `aria-checked="mixed"`; clear it in `onCheckedChange` when users resolve the mixed state |
| `onCheckedChange` | `(checked, details) => void` | reports attempted changes; controlled consumers update `checked` |
| `disabled` / `readOnly` | `boolean` / `false` | unavailable, or focusable and immutable |
| `required` | `boolean` / `false` | participates in native constraint validation |
| `name`, `value`, `uncheckedValue`, `form` | `string` | configures native form serialization, including external forms and explicit off values |
| `inputRef` / `ref` | `Ref<HTMLInputElement>` / `Ref<HTMLElement>` | accesses the hidden native input or visual root |
| `uiSize` | `'sm' \| 'md' \| 'lg'` / `'md'` | sets the visual control size |
| `render` | Base UI render prop | replaces the root host while preserving behavior and state |

Uncontrolled roots restore `defaultChecked` when their owning form resets. Controlled roots remain governed by `checked`.

### `TRCheckbox.Indicator`

The Indicator renders while checked or indeterminate. Use `keepMounted` for transition styling, or its `render(props, state)` callback when the mixed glyph must follow `state.indeterminate`. Root and Indicator expose Base UI state through render callbacks, functional `className`/`style`, and `data-checked`, `data-unchecked`, `data-indeterminate`, `data-disabled`, `data-readonly`, and `data-invalid` attributes.

`TRCheckboxRoot`, `TRCheckboxIndicator`, `TRCheckboxRootProps`, `TRCheckboxIndicatorProps`, `TRCheckboxUiSize`, `TRCheckboxRootState`, and `TRCheckboxIndicatorState` are exported from `@tinyrack/ui/components/checkbox`. CSS remains opt-in.
