## 原則

密度の高い補助アクションには `sm`、標準の運用インターフェースには `md`、タッチの快適さや強調が必要な場合だけ `lg` を使用します。

## 視覚的スケール

<div className="grid gap-tinyrack-md" role="list">
  {['sm', 'md', 'lg'].map((size) => (
    <section className="grid items-center gap-tinyrack-md border border-tinyrack-border p-tinyrack-lg md:grid-cols-[7rem_minmax(0,1fr)]" key={size} role="listitem">
      <div><strong>{size}</strong><div className="text-tinyrack-sm text-tinyrack-text-muted"><Code>{tinyrackControlMetrics[size].height}</Code></div></div>
      <div className="flex flex-wrap items-center gap-tinyrack-sm"><Button uiSize={size}>Deploy</Button><Field.Root><Field.Control aria-label={`${size} rack name`} data-uiSize={size} placeholder="rack-a" /></Field.Root></div>
    </section>
  ))}
</div>

## 適用パターン

<section className="grid gap-tinyrack-lg border border-tinyrack-border p-tinyrack-lg">
  <div className="grid gap-tinyrack-sm md:grid-cols-[minmax(0,1fr)_auto]"><Field.Root><Field.Control aria-label="Rack 名" placeholder="rack-a" /></Field.Root><Button variant="primary">Save rack</Button></div>
  <div className="text-tinyrack-sm text-tinyrack-text-muted">両方のコントロールは同じ 40px の medium 高さとタイポグラフィレシピを使用します。</div>
</section>

## 実装

製品全体に異なる密度が必要な場合はグローバル metric を上書きし、1 つのコンポーネントだけに意図的な例外が必要な場合は `--tr-*` レシピトークンを使います。

```css
:root {
  --tinyrack-control-height-md: 2.5rem;
}

.rack-toolbar {
  --tr-btn-height: 2.25rem;
}
```
