## 계약

`TRColorSchemeToggle`은 제어형 2상태 버튼입니다. 아이콘과 접근 가능한 이름은 다음 동작을 설명합니다. `light`에서는 dark를, `dark`에서는 light를 제공합니다. 컴포넌트는 `localStorage`를 읽거나 `system` 설정을 조회하거나 document를 변경하지 않습니다. 이 작업은 애플리케이션 어댑터에서 해결하고 저장하세요.

## API

| Prop | Type | Default | 설명 |
| --- | --- | --- | --- |
| `value` | `"light" \| "dark"` | 필수 | 현재 해석된 color scheme입니다. |
| `onValueChange` | `(value: TRColorScheme) => void` | 필수 | 취소되지 않은 활성화 후 반대 scheme을 전달합니다. |
| `darkLabel` | `string` | `"Use dark color scheme"` | 다음 값이 dark일 때의 접근 가능한 동작 이름입니다. |
| `lightLabel` | `string` | `"Use light color scheme"` | 다음 값이 light일 때의 접근 가능한 동작 이름입니다. |
| `aria-labelledby` | `string` | — | 생성된 label 대신 외부의 접근 가능한 이름을 사용합니다. |
| `disabled` | `boolean` | `false` | focus와 활성화를 막습니다. |
| `uiSize` | `"sm" \| "md" \| "lg"` | `"md"` | 상속된 IconButton 크기입니다. |
| `ref` | `Ref<HTMLButtonElement>` | — | button으로 전달되는 ref입니다. |
| `onClick` | `MouseEventHandler<HTMLButtonElement>` | — | `onValueChange`보다 먼저 실행되며 `preventDefault()`로 값 변경을 취소합니다. |

그 밖의 안전한 `TRIconButton` button attribute, native event, `className`, `style`, customization token을 상속합니다. 시각적 appearance는 `ghost`로, 컨트롤은 native button으로 유지됩니다.
