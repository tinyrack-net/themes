## 原則

密度の高い補助アクションには `sm`、標準の運用インターフェースには `md`、タッチの快適さや強調が必要な場合だけ `lg` を使用します。

## 選択基準

- 主要アクションを意図的に強調する場合を除き、一つのアクション行では同じサイズレシピを使います。
- ポインター利用が想定され密度が重要なら `sm`、標準は `md`、タッチ中心または強調の強いタスクには `lg` を使います。
- 高さだけを上書きせず、高さ、内側余白、間隔、文字サイズ、行間を含むレシピ全体を変更します。

## 状態とタッチターゲット

見た目を独自に再現せず、コンポーネントの意味的な状態を使います。ホバーは操作可能であることを予告し、押下は入力中であることを示し、選択は決定を保持します。無効化は利用できない操作を止め、フォーカスはキーボード操作中も見える必要があります。ホバーだけを手がかりにしないでください。タッチ中心のレイアウトでは `lg` を選ぶか、操作領域を縮めずにコントロール周辺の間隔を広げます。

## 実装

製品全体に異なる密度が必要な場合はグローバル寸法を上書きし、一つのコンポーネントだけに意図的な例外が必要な場合は `--tr-*` レシピトークンを使います。

```css
:root {
  --tinyrack-control-height-md: 2.5rem;
}

.rack-toolbar {
  --tr-btn-height: 2.25rem;
}
```

## アクセシビリティ

すべてのコントロールには、見えるラベルまたはアクセシブルな名前、明確なフォーカス状態、色だけに頼らない状態表現が必要です。コンポーネントのキーボード動作を保ち、タッチ中心のフローでより大きなターゲットが必要なら `lg` を使います。見える表面だけを大きくして、実際の操作領域を残さないでください。
