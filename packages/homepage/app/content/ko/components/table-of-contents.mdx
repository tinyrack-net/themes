## 계약

순서가 있는 h2/h3 heading 데이터와 현재 scroll-spy 결과를 전달하세요. depth 3 항목은 앞선 depth 2 항목 아래에 중첩됩니다. `renderLink`로 router link를 주입할 수 있고, adapter가 없으면 데스크톱 link와 모바일 select가 hash navigation을 사용합니다.

## 예시

intersection observer에서 `currentHeading`을 업데이트하세요. link는 native keyboard 탐색을 유지하며, `onNavigate`로 router 또는 custom scroller와 controlled active state를 동기화할 수 있습니다. `items`가 비어 있으면 `null`을 반환하고 좁은 layout에서는 긴 label의 overflow를 제한합니다.

## API

`items`는 순서가 있는 `{ id, label, depth }` collection이며 `depth`는 `2 | 3`으로 제한됩니다. `currentHeading`은 active link에 `aria-current="location"`을 설정하고 모바일 선택을 제어합니다. `label`은 navigation landmark와 데스크톱 heading을, `mobileLabel`은 모바일 select를 명명합니다. `onNavigate`는 선택한 item을 받고 `renderLink`는 데스크톱 router link를 제공합니다. Native nav props, events, styles, `className`, `ref`가 전달됩니다. 같은 데이터로 데스크톱 list와 keyboard로 조작 가능한 모바일 select를 렌더링하며 SSR/hydration에 안전합니다.
