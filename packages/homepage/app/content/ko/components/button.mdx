## 계약

| 축 | 값 | 기본값 | 용도 |
| --- | --- | --- | --- |
| `variant` | `secondary`, `primary`, `danger` | `secondary` | 동작 의도를 전달합니다. |
| `appearance` | `solid`, `outline`, `ghost` | `solid` | 의도를 바꾸지 않고 강조 수준을 변경합니다. |
| `uiSize` | `sm`, `md`, `lg` | `md` | 주변 컨트롤 밀도에 맞춥니다. |
| 상태 | `disabled`, `loading` | 상호작용 가능 | 네이티브 disabled 동작과 `aria-busy`를 사용합니다. |

## API

- `TRButton`은 Base UI button props, native event, `className`, `style`, React 19 `ref`를 전달합니다.
- 네이티브 button의 `type` 기본값은 `button`입니다. form 안에서는 `type="submit"` 또는 `type="reset"`을 의도적으로 설정하세요. `render`로 button이 아닌 요소를 사용할 때는 `nativeButton={false}`를 설정하며, 이 경우 네이티브 `type` 기본값은 생략됩니다.
- `loading`은 control을 비활성화하고 `aria-hidden` spinner를 렌더링하며 이름 있는 button에 `aria-busy`를 설정합니다. 유휴 button이 `aria-labelledby`를 사용해도 `loadingLabel`이 접근 가능한 이름이 되며, loading 중이 아닐 때는 명시한 `aria-busy` 값이 유지됩니다.
- Secondary outline과 ghost appearance는 대비가 낮은 border token 대신 읽기 쉬운 foreground 대비를 위해 `--tinyrack-text`를 사용합니다.
