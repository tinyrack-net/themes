# Tinyrack UI

Install the CSS-first Tinyrack package, import the core token CSS once, then add component CSS only for the components you render.

## Installation

### Supported Surfaces

| Surface | Install | What it enables |
| --- | --- | --- |
| CSS / HTML | `pnpm add @tinyrack/ui tailwindcss` | Theme variables, prefixed utilities, and class-based component contracts. |
| React components | `pnpm add @tinyrack/ui react react-dom` | SSR-safe React wrappers for owned components. |
| React MDX renderer | `pnpm add @tinyrack/ui @mdx-js/react shiki` | Markdown element overrides for React, including optional highlighted code. |
| Astro MDX renderer | `pnpm add @tinyrack/ui astro @astrojs/mdx shiki` | Markdown element overrides for Astro content collections. |
| CodeBlock syntax highlight | `pnpm add shiki` | Optional client highlighting through `@tinyrack/ui/components/code-block/shiki-react`. |

```sh
pnpm add @tinyrack/ui
pnpm add tailwindcss
pnpm add react react-dom
pnpm add shiki
```

Tinyrack UI recommends Lucide for product icons, but apps install the icon
package directly instead of receiving it through `@tinyrack/ui`.

| Surface | Icon install | Use |
| --- | --- | --- |
| React, CRA, SSR React | `pnpm add lucide-react` | Use named imports inside Tinyrack React components. |
| Astro pages | `pnpm add @lucide/astro` | Use in `.astro` files; use `lucide-react` inside React islands. |
| HTML and non-React | `pnpm add lucide-static` | Inline or copy SVGs when there is no component runtime. |

```sh
pnpm add lucide-react
pnpm add @lucide/astro
pnpm add lucide-static
```

## Usage

### CSS / HTML

Import Tailwind first, then Tinyrack core CSS and the component CSS used by the page.

```css
@import "tailwindcss";
@import "@tinyrack/ui/core/core.css";
@import "@tinyrack/ui/components/button/button.css";
@import "@tinyrack/ui/components/form/form.css";
@import "@tinyrack/ui/components/link/link.css";
@import "@tinyrack/ui/components/table/table.css";
@import "@tinyrack/ui/components/tabs/tabs.css";
```

```html
<section data-theme="tinyrack-dark" class="bg-tinyrack-surface text-tinyrack-text">
  <button class="tr-btn" data-size="md" data-variant="primary" data-appearance="solid">
    Deploy
  </button>
</section>
```

Use Tailwind utilities directly for layout and spacing.

```html
<section class="mx-auto grid max-w-5xl gap-6 px-4">
  <div class="flex flex-wrap items-center gap-2">...</div>
</section>
```

### React Components

```tsx
import '@tinyrack/ui/core/core.css';
import '@tinyrack/ui/components/button/button.css';
import '@tinyrack/ui/components/form/form.css';
import { Button } from '@tinyrack/ui/components/button/react';
import { Field, FormMessage, Input, Label } from '@tinyrack/ui/components/form/react';

export function RackForm() {
  return (
    <Field>
      <Label htmlFor="rack-name">Rack name</Label>
      <Input id="rack-name" placeholder="rack-a-01" />
      <FormMessage>Use a stable rack identifier.</FormMessage>
      <Button variant="primary">Deploy</Button>
    </Field>
  );
}
```

### React MDX

```tsx
import '@tinyrack/ui/core/core.css';
import '@tinyrack/ui/mdx/mdx.css';
import { MDXProvider } from '@mdx-js/react';
import { tinyrackMdxComponents } from '@tinyrack/ui/mdx/react';

<MDXProvider components={tinyrackMdxComponents}>
  <Article />
</MDXProvider>;
```

### Astro MDX

```astro
---
import { tinyrackAstroMdxComponents } from '@tinyrack/ui/mdx/astro';
const { Content } = await entry.render();
---

<Content components={tinyrackAstroMdxComponents} />
```

## Use Boundary

| Use this when | Do not use this for |
| --- | --- |
| Product surfaces need compact color, type, spacing, and radius decisions. | Another component system wrapper or adapter gallery. |
| Apps can compose CSS explicitly from stable package paths. | Root-level React exports from `@tinyrack/ui`. |
| Owned component workflows still care about native markup. | Visual primitives outside the public core export. |
