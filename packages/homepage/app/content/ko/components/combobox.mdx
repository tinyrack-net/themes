## 계약

| 축 | 동작 |
| --- | --- |
| 상태 | 입력 텍스트, 선택 값, 열림 상태는 각각 제어 또는 비제어로 사용할 수 있어요. |
| 입력 구조 | native label 또는 Field.Label → InputGroup → 선택적 InputAdornment + Input + Clear + Trigger 순서이며 Icon은 Trigger 안에 둬요. |
| 다중 선택 구조 | Chips → Value → Chip + ChipRemove를 사용하고 grid collection은 Collection → Row → Item으로 구성해요. |
| popup 구조 | Portal → 선택적 Backdrop + Positioner → Popup → Arrow + Status + List 순서이며 Group, Indicator, Separator, Collection, Empty를 조합해요. |
| 상호작용 | pointer, 방향키, Home/End, Enter, Escape, 필터링, focus 복원, disabled item, read-only root 동작은 Base UI가 담당해요. |
| form | Root에서 `name`, `form`, `required`, `disabled`, `readOnly`, object 직렬화, native validation, submit, reset을 지원해요. |

import { useState } from 'react';

const racks = ['Rack A', 'Rack B', 'Rack C'];

function MatchCount() {
  const matches = TRCombobox.useFilteredItems<string>();
  return <output aria-live="polite">일치 항목 {matches.length}개</output>;

function ControlledRackCombobox() {

## API

| surface | 용도 |
| --- | --- |
| `Root` | item, filter, single/multiple 값, 입력 텍스트, 열림 상태, validation, form 직렬화, locale, object 비교/문자열 변환을 관리해요. |
| `InputGroup`, `InputAdornment`, `Input`, `Clear`, `Trigger`, `Icon` | 편집 가능한 control을 구성해요. `InputAdornment`는 장식이고 `Icon`은 `Trigger` 안에 둬요. |
| `Portal`, `Backdrop`, `Positioner`, `Popup`, `Arrow` | 닫을 수 있는 overlay를 mount하고 배치해요. Portal은 custom container를 지원하며 inline이면 생략할 수 있어요. |
| `Status`, `List`, `Group`, `GroupLabel`, `Collection`, `Row`, `Item`, `ItemIndicator`, `Separator`, `Empty` | list/grid collection과 접근성 피드백을 렌더링해요. |
| `Chips`, `Value`, `Chip`, `ChipRemove` | 여러 선택 값을 표시하고 제거해요. |
| `useFilter`, `useFilteredItems` | locale 기반 `contains`, `startsWith`, `endsWith` predicate를 만들고 Root 안에서 generic filtered item 배열을 읽어요. |

모든 part는 개별 `TRCombobox*` export와 props/state type을 제공해요. Root action, 상태 변경 및 highlight의 detail/reason type과 `TRComboboxFilter`, `TRComboboxFilterOptions`도 export해요.

편집 가능한 `Input`에는 native `label` 또는 `TRField.Label`을 사용해요. `TRCombobox.Label`은 trigger-only select 구조 전용이에요. Disabled root/item은 변경할 수 없고 read-only root는 focus할 수 있지만 입력하거나 지울 수 없어요. Object 값은 안정적인 identity를 유지하고 `{ label, value }` 규칙으로 충분하지 않으면 `itemToStringLabel`, `itemToStringValue`, `isItemEqualToValue`를 지정해요.

Combobox module은 SSR-safe예요. Portal은 hydration 후 mount되며 server/client의 초기 controlled 값이 같아야 해요. 스타일은 `combobox.css`로 별도 제공해요. `--tr-combobox-input-background`, `--tr-combobox-content-background`, `--tr-combobox-content-max-height`, `--tr-combobox-content-max-width`, `--tr-combobox-option-min-height`로 appearance를 조정할 수 있고 light/dark semantic token으로 fallback돼요.
