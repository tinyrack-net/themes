<h2 className="tr-mdx-h2" id="setup">Setup</h2>

Install Tailwind CSS and its Vite plugin in the application that consumes Tinyrack.

```sh
pnpm add -D tailwindcss @tailwindcss/vite
```

Add the plugin to the application's Vite configuration.

```ts
import tailwindcss from '@tailwindcss/vite';
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [tailwindcss()],
});
```

Import Tailwind first, then Tinyrack core CSS. Import each component stylesheet separately when the component is used.

```css
@import "tailwindcss";
@import "@tinyrack/ui/core.css";
@import "@tinyrack/ui/components/card.css";
```

<h2 className="tr-mdx-h2" id="choose-utilities">Choose a value</h2>

1. Choose a named <TRCode>tinyrack-*</TRCode> utility when the value represents a design-system decision such as color, type, spacing, radius, elevation, or motion.
2. Choose a Tailwind default for incidental layout, such as <TRCode>grid</TRCode> or <TRCode>grid-cols-2</TRCode>. Use <TRCode>xs:</TRCode> through <TRCode>xl:</TRCode> for Tinyrack's viewport thresholds.
3. Choose an arbitrary value only when a real component variable or one-off CSS value has no named utility. Do not create a second token vocabulary.

<h2 className="tr-mdx-h2" id="practical-example">Practical example</h2>

This card uses named Tinyrack utilities for system decisions and Tailwind defaults for its layout. The example is ready to paste after the setup above.

```tsx
export function ClusterHealth() {
  return (
    <section className="grid gap-tinyrack-lg rounded-tinyrack-lg border-tinyrack-default border-tinyrack-border bg-tinyrack-surface p-tinyrack-xl md:grid-cols-2">
      <h2 className="text-tinyrack-xl font-tinyrack-heading">Cluster health</h2>
      <p className="text-tinyrack-sm text-tinyrack-text-muted">
        All nodes are available.
      </p>
    </section>
  );
}
```

<h2 className="tr-mdx-h2" id="arbitrary-values">Arbitrary values</h2>

Use a type hint when a public component variable has no natural Tailwind namespace. Keep the variable as the source of truth instead of copying its resolved value. For example, Card exposes <TRCode>--tr-card-background</TRCode>.

```tsx
<div className="bg-[color:var(--tr-card-background)]">...</div>
```
