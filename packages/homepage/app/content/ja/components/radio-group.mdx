## コントラクト

| 軸 | 動作 |
| --- | --- |
| ラベル | `aria-label`/`aria-labelledby` を指定するか、legend を持つ `TRFieldset.Root` 内に group を配置します。 |
| 値 | `defaultValue` で非制御、`value` と `onValueChange` で制御します。 |
| キーボード | すべての矢印キーが focus と選択を移動し、disabled radio を飛ばして循環します。Tab は一度だけ出入りします。 |
| 状態 | `disabled` は操作と form data を無効にします。`readOnly` は選択済み radio を focus 可能なまま変更を防ぎます。 |
| form | `name`、`form`、`required` は group に適用され、reset は非制御の `defaultValue` を復元します。 |
| レンダリング | wrapper は SSR-safe で、選択値を変えずに hydrate されます。 |

`TRRadioGroup` の矢印ナビゲーションは orientation に依存せず、2 軸で動作します。`orientation` prop は公開していません。キーボード動作を変えずに `className` または `style` で視覚的な layout を整えてください。

## API

`TRRadioGroup`、`TRRadioGroupProps`、`TRRadioGroupState` は `@tinyrack/ui/components/radio-group` から export されます。radio module の `TRRadio.Root` と `TRRadio.Indicator` を組み合わせてください。root は `ref`、`inputRef`、ネイティブ event と attribute、state ベースの `className`/`style`、polymorphic `render` を保持します。`onValueChange(value, details)` は変更元のネイティブ event を受け取り、`details.cancel()` で変更を拒否できます。
