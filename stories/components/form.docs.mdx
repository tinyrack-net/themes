# Form

SSR-friendly field, text control, choice control, and message primitives over native form elements.

## Contract

| Surface | API | Values | Default | Rule |
| --- | --- | --- | --- | --- |
| CSS import | `form.css` | every form primitive | required | One stylesheet covers field, text, and choice controls. |
| Field class | `tr-field` | `data-size` | `md` | Groups label, control, and message rhythm. |
| Text controls | `tr-input`, `tr-textarea`, `tr-select` | `data-size`, `data-invalid` | `md`, unset | Native controls keep native behavior. |
| Choice controls | `tr-checkbox`, `tr-radio`, `tr-switch` | `data-size`, `data-invalid` | `md`, unset | Native inputs remain focusable. |
| Radio group | `tr-radio-group` | `data-orientation` | `vertical` | Use fieldset semantics for grouped choices. |
| Message | `tr-form-message` | `data-variant` | `neutral` | Use `error` for validation support text. |

## Install

| Surface | Install | Import |
| --- | --- | --- |
| CSS / HTML | `pnpm add @tinyrack/ui tailwindcss` | `@tinyrack/ui/core/core.css` and `@tinyrack/ui/components/form/form.css` |
| React | `pnpm add @tinyrack/ui react react-dom` | `@tinyrack/ui/components/form/react` |
| React MDX | `pnpm add @tinyrack/ui @mdx-js/react` | `@tinyrack/ui/mdx/react` maps task-list checkboxes. |
| Astro MDX | `pnpm add @tinyrack/ui astro @astrojs/mdx` | `@tinyrack/ui/mdx/astro` maps task-list checkboxes. |

```css
@import "tailwindcss";
@import "@tinyrack/ui/core/core.css";
@import "@tinyrack/ui/components/form/form.css";
```

## Usage

<ComponentExampleTabs
  title="Basic field"
  description="Compose label, native control, and message primitives around the shared form stylesheet."
  preview={(
    <Field>
      <Label htmlFor="rack-name">Rack name</Label>
      <Input id="rack-name" placeholder="rack-a-01" />
      <FormMessage>Use a stable rack identifier.</FormMessage>
    </Field>
  )}
  sources={[
    {
      label: 'React',
      language: 'tsx',
      code: String.raw`import '@tinyrack/ui/core/core.css';
import '@tinyrack/ui/components/form/form.css';
import { Field, FormMessage, Input, Label } from '@tinyrack/ui/components/form/react';

<Field>
  <Label htmlFor="rack-name">Rack name</Label>
  <Input id="rack-name" placeholder="rack-a-01" />
  <FormMessage>Use a stable rack identifier.</FormMessage>
</Field>;`,
    },
    {
      label: 'HTML',
      language: 'html',
      code: String.raw`<div class="tr-field" data-size="md">
  <label class="tr-label" for="rack-name">Rack name</label>
  <input class="tr-input" data-size="md" id="rack-name" />
  <p class="tr-form-message" data-variant="neutral">
    Use a stable rack identifier.
  </p>
</div>

<label class="tr-checkbox" data-size="md">
  <input class="tr-checkbox-input" type="checkbox" checked />
  <span aria-hidden="true" class="tr-checkbox-control"></span>
  <span class="tr-checkbox-label">Monitoring enabled</span>
</label>`,
    },
  ]}
/>

## API

| Component | Main props | Note |
| --- | --- | --- |
| `Field` | `size` | Field layout wrapper. |
| `Label` | native label props | Use `htmlFor` or wrap the control. |
| `Input`, `Textarea`, `Select` | `size`, `invalid`, native props | Text controls share sizing and validation styling. |
| `Checkbox`, `Radio`, `Switch` | `size`, `invalid`, `disabled`, native input props | Native input remains focusable. |
| `RadioGroup` | `orientation`, fieldset props | Renders a `fieldset`. |
| `FormMessage` | `variant` | Renders supporting or error text. |
