# Form

SSR-friendly field, text control, choice control, and message primitives over native form elements.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `tr-field` | `Field` | `data-size`; `sm`, `md`, `lg`; default `md` | Groups a label, control, and associated message. |
| Part | `tr-label`, `tr-form-message` | `Label`, `FormMessage` | message variants `neutral`, `error`; default `neutral` | Associate labels and messages with the control they explain. |
| Component | `tr-input`, `tr-textarea`, `tr-select` | `Input`, `Textarea`, `Select` | `data-size`, `data-invalid`; size default `md` | Keeps native text-control behavior and validation semantics. |
| Component | `tr-checkbox`, `tr-radio`, `tr-switch` | `Checkbox`, `Radio`, `Switch` | `data-size`, `data-invalid`; size default `md` | Keeps the native input focusable and keyboard operable. |
| Behavior | `tr-radio-group[data-orientation]` | `RadioGroup.orientation` | `vertical`, `horizontal`; default `vertical` | Group related choices with a fieldset and an accessible name. |
| State | `data-invalid`, `aria-invalid`, `required`, `disabled` | `invalid` and native props | invalid unset; native props default `false` | Expose state semantically instead of relying on color alone. |

## Usage

<ComponentExampleTabs
  id="form-basic"
  title="Basic field"
  description="Compose a complete field with an explicit label and programmatically associated help text."
  previewLayout="stretch"
  preview={(
    <Field>
      <Label htmlFor="rack-name">Rack name</Label>
      <Input aria-describedby="rack-name-help" id="rack-name" placeholder="rack-a-01" />
      <FormMessage id="rack-name-help">Use a stable rack identifier.</FormMessage>
    </Field>
  )}
  sources={[
    {
      label: 'React',
      language: 'tsx',
      code: String.raw`import '@tinyrack/ui/core/core.css';
import '@tinyrack/ui/components/form/form.css';
import { Field, FormMessage, Input, Label } from '@tinyrack/ui/components/form/react';

<Field>
  <Label htmlFor="rack-name">Rack name</Label>
  <Input aria-describedby="rack-name-help" id="rack-name" placeholder="rack-a-01" />
  <FormMessage id="rack-name-help">Use a stable rack identifier.</FormMessage>
</Field>;`,
    },
    {
      label: 'HTML',
      language: 'html',
      code: String.raw`<div class="tr-field" data-size="md">
  <label class="tr-label" for="rack-name">Rack name</label>
  <input class="tr-input" data-size="md" id="rack-name" aria-describedby="rack-name-help" />
  <p class="tr-form-message" data-variant="neutral" id="rack-name-help">
    Use a stable rack identifier.
  </p>
</div>

<label class="tr-checkbox" data-size="md">
  <input class="tr-checkbox-input" type="checkbox" checked />
  <span aria-hidden="true" class="tr-checkbox-control"></span>
  <span class="tr-checkbox-label">Monitoring enabled</span>
</label>`,
    },
  ]}
/>

## Guidance

<ComponentGuidance
  whenToUse={[
    'Compose each control as a complete field with a visible label and associated help or error text.',
    'Use the native control that matches the data and interaction: input, textarea, select, checkbox, radio, or switch.',
  ]}
  avoid={[
    'Do not use placeholder text as the only label or validation color as the only error signal.',
    'Do not replace native required, disabled, checked, or selected behavior with visual state alone.',
  ]}
  accessibility={[
    'Connect labels with htmlFor/id and connect help or error text with aria-describedby.',
    'Set required, disabled, and aria-invalid on the native control; keep invalid messages specific and visible.',
    'Wrap related radio or checkbox choices in a fieldset with a legend, and preserve native Tab, Space, arrow-key, and select keyboard behavior.',
  ]}
/>

## API

| Component | Main props | Note |
| --- | --- | --- |
| `Field` | `size` | Field layout wrapper. |
| `Label` | native label props | Use `htmlFor` or wrap the control. |
| `Input`, `Textarea`, `Select` | `size`, `invalid`, native props | Text controls share sizing and validation styling. |
| `Checkbox`, `Radio`, `Switch` | `size`, `invalid`, `disabled`, native input props | Native input remains focusable. |
| `RadioGroup` | `orientation`, fieldset props | Renders a `fieldset`. |
| `FormMessage` | `variant` | Renders supporting or error text. |
