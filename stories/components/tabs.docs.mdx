# Tabs

Accessible tabbed navigation for compact operational panels.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `tr-tabs` | `Tabs` | Required | Owns selection, size, orientation, and activation context. |
| Part | `tr-tabs-list`, `role="tablist"` | `TabsList` | Required | Give every list an accessible label. |
| Part | `tr-tabs-trigger`, `role="tab"` | `TabsTrigger` | Required | Uses native buttons with roving tab focus. |
| Part | `tr-tabs-panel`, `role="tabpanel"` | `TabsPanel` | Required | Inactive panels are hidden. |
| Size | `data-size` | `size` | `sm`, `md` (default), `lg` | Changes trigger height and panel padding. |
| Behavior | `data-orientation` and app keyboard logic | `orientation` | `horizontal` (default), `vertical` | Orientation changes layout and arrow-key mapping. |
| Behavior | App-controlled | `activationMode` | `automatic` (default), `manual` | Use manual activation when switching panels is expensive. |
| State | `aria-selected`, `data-active` | `value` or `defaultValue` | One selected value | Use exactly one controlled or uncontrolled selection mode. |
| Native | Div and button attributes | Native root, list, trigger, and panel props | Pass-through | Preserve generated IDs and ARIA relationships. |

## Usage

<ComponentExampleTabs
  id="tabs-basic"
  title="Basic tabs"
  description="Use Tabs for compact panels that switch between related operational views."
  previewLayout="stretch"
  preview={(
    <Tabs defaultValue="overview" activationMode="automatic">
      <TabsList aria-label="Rack sections">
        <TabsTrigger value="overview">Overview</TabsTrigger>
        <TabsTrigger value="logs">Logs</TabsTrigger>
      </TabsList>
      <TabsPanel value="overview">Rack A is healthy.</TabsPanel>
      <TabsPanel value="logs">03:18 backup completed.</TabsPanel>
    </Tabs>
  )}
  sources={[
    {
      label: 'React',
      language: 'tsx',
      code: String.raw`import '@tinyrack/ui/core/core.css';
import '@tinyrack/ui/components/tabs/tabs.css';
import { Tabs, TabsList, TabsPanel, TabsTrigger } from '@tinyrack/ui/components/tabs/react';

<Tabs defaultValue="overview" activationMode="automatic">
  <TabsList aria-label="Rack sections">
    <TabsTrigger value="overview">Overview</TabsTrigger>
    <TabsTrigger value="logs">Logs</TabsTrigger>
  </TabsList>
  <TabsPanel value="overview">Rack A is healthy.</TabsPanel>
  <TabsPanel value="logs">03:18 backup completed.</TabsPanel>
</Tabs>;`,
    },
    {
      label: 'HTML',
      language: 'html',
      code: String.raw`<div class="tr-tabs" data-size="md" data-orientation="horizontal">
  <div class="tr-tabs-list" role="tablist" aria-label="Rack sections">
    <button
      class="tr-tabs-trigger"
      role="tab"
      aria-selected="true"
      data-active="true"
    >
      Overview
    </button>
  </div>
  <div class="tr-tabs-panel" role="tabpanel" data-active="true">
    Rack A is healthy.
  </div>
</div>`,
    },
  ]}
/>

## Guidance

<ComponentGuidance
  whenToUse={[
    'Use tabs for a small set of peer views that belong to the same object or task.',
    'Use manual activation when changing panels performs expensive rendering or network work.',
  ]}
  avoid={[
    'Do not use tabs for primary site navigation, ordered steps, or content users need to compare simultaneously.',
    'Do not hide validation errors or required actions in an inactive panel without surfacing their state.',
  ]}
  accessibility={[
    'Give every TabsList an accessible label that describes the group rather than repeating the selected tab.',
    'Horizontal lists use Left and Right Arrow; vertical lists use Up and Down Arrow; Home and End move to the edges.',
    'Keep trigger labels concise and preserve the generated tab-to-panel ID relationships.',
  ]}
/>

## API

| Prop | Values | Note |
| --- | --- | --- |
| `value` | `string` | Controlled selected tab. |
| `defaultValue` | `string` | Initial selected tab for uncontrolled usage. |
| `onValueChange` | `(value) => void` | Called after click or keyboard activation. |
| `size` | `sm`, `md`, `lg` | Default `md`; sets `data-size`. |
| `orientation` | `horizontal`, `vertical` | Default `horizontal`; sets `data-orientation`. |
| `activationMode` | `automatic`, `manual` | Default `automatic`. |
| native props | div and button attributes | Passed through to root, list, trigger, and panel primitives. |
