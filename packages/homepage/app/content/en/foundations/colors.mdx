## Base colors

> Base color values should only be used to construct functional and component/pattern design tokens, never used directly in product code.

Base ramps are exported from `@tinyrack/ui/core` for token authors and tooling. They are intentionally absent from `core.css` and the Tailwind theme bridge so product UI stays on theme-aware functional roles.

## Functional colors

Functional colors are the default product API. Choose them by meaning rather than by whichever value happens to look correct in the current theme.

## Component/pattern tokens

Component tokens stay colocated with their owning CSS and use functional roles as fallbacks.

```css
.tr-alert {
  --_tr-alert-background: var(--tinyrack-surface-muted);
  background: var(--tr-alert-background, var(--_tr-alert-background));
}

.tr-alert[data-variant="danger"] {
  --_tr-alert-background: var(--tinyrack-danger-surface);
  --_tr-alert-border: var(--tinyrack-danger-border);
  --_tr-alert-color: var(--tinyrack-danger);
}
```

Action hierarchy and status meaning are separate contracts. Buttons use `secondary`, `primary`, and `danger`; feedback components use `neutral`, `info`, `success`, `warning`, and `danger`.

<div className="flex flex-wrap gap-2">
  <TRButton variant="secondary">Inspect</TRButton>
  <TRButton variant="primary">Deploy</TRButton>
  <TRButton variant="danger">Remove</TRButton>
</div>

## Custom primary theme

A product can replace the default achromatic action identity by overriding three functional roles. Base ramps do not need to change.

```css
[data-theme="rack-blue"] {
  --tinyrack-primary: #1d4ed8;
  --tinyrack-primary-hover: #1e40af;
  --tinyrack-on-primary: #ffffff;
}
```

## Do and don't

### Do: use a functional role in product UI

```css
.rack-status {
  color: var(--tinyrack-info);
  background: var(--tinyrack-info-surface);
  border-color: var(--tinyrack-info-border);
}
```

### Don't: select a Base value in product UI

```ts
import { tinyrackPalettes } from '@tinyrack/ui/core';

// Do not use this as a component or product style.
const rackStatusColor = tinyrackPalettes.blue[700];
```

Base values may be imported by token tooling, but rendered product UI should consume the functional CSS variables or their `tinyrack-*` Tailwind utilities.
