## 基本色

> Base color values should only be used to construct functional and component/pattern design tokens, never used directly in product code.

Base ramp は token author と tooling 向けに `@tinyrack/ui/core` から export されます。製品 UI が theme-aware functional role を使えるよう、`core.css` と Tailwind theme bridge には意図的に含めていません。

<div className="grid gap-4">
  {Object.entries(tinyrackPalettes).map(([paletteName, palette]) => (
    <section className="grid gap-3 border border-tinyrack-border p-4" key={paletteName} aria-label={`${paletteName} base colors`}>
      <h3 className="m-0 capitalize text-tinyrack-lg font-semibold leading-tinyrack-sm">{paletteName}</h3>
      <div className="grid grid-cols-2 gap-2 sm:grid-cols-4 lg:grid-cols-6" role="list">
        {Object.entries(palette).map(([step, value]) => (
          <div className="grid min-w-0 gap-2" key={step} role="listitem">
            <span aria-hidden="true" className="h-12 border border-tinyrack-border" style={{ backgroundColor: value }} />
            <span className="grid min-w-0 gap-1 text-tinyrack-xs"><strong>{step}</strong><TRCode>{value}</TRCode></span>
          </div>
        ))}
      </div>
    </section>
  ))}
</div>

## コンポーネント・パターントークン

Component token は所有する CSS と同じ場所に置き、fallback として functional role を使います。

```css
.tr-alert {
  --_tr-alert-background: var(--tinyrack-surface-muted);
  background: var(--tr-alert-background, var(--_tr-alert-background));
}

.tr-alert[data-variant="danger"] {
  --_tr-alert-background: var(--tinyrack-danger-surface);
  --_tr-alert-border: var(--tinyrack-danger-border);
  --_tr-alert-color: var(--tinyrack-danger);
}
```

Action hierarchy と status の意味は別の contract です。TRButton は `secondary`、`primary`、`danger`、feedback component は `neutral`、`info`、`success`、`warning`、`danger` を使います。

<div className="flex flex-wrap gap-2">
  <TRButton variant="secondary">確認</TRButton>
  <TRButton variant="primary">デプロイ</TRButton>
  <TRButton variant="danger">削除</TRButton>
</div>

## カスタム primary テーマ

製品は 3 つの functional role を override して既定の achromatic action identity を置き換えられます。Base ramp を変更する必要はありません。

```css
[data-theme="rack-blue"] {
  --tinyrack-primary: #1d4ed8;
  --tinyrack-primary-hover: #1e40af;
  --tinyrack-on-primary: #ffffff;
}
```

## 推奨と禁止

### 推奨: 製品 UI では functional role を使う

```css
.rack-status {
  color: var(--tinyrack-info);
  background: var(--tinyrack-info-surface);
  border-color: var(--tinyrack-info-border);
}
```

### 禁止: 製品 UI で Base 値を選ばない

```ts
import { tinyrackPalettes } from '@tinyrack/ui/core';

// Do not use this as a component or product style.
const rackStatusColor = tinyrackPalettes.blue[700];
```

Base value は token tooling で import できますが、レンダリングされる製品 UI は functional CSS variable または `tinyrack-*` Tailwind utility を使います。
