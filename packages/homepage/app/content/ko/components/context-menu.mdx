## 계약

`TRContextMenu`는 rack row, file, canvas 같은 콘텐츠에 명령을 연결합니다. 전용 trigger가 항상 명령 목록을 열 때는 `TRMenu`를 사용하세요. 이 Tinyrack wrapper는 Base UI의 위치 지정, 포커스, 닫기, React 19 ref, 네이티브 prop, 이벤트, `render` contract를 보존합니다. Playground control은 보이는 `label`, item `variant`, `disabledItem` 상태를 다루며 open 상태는 control에 노출하지 않고 동기화합니다.

## API

모든 part, prop type, state type은 `@tinyrack/ui/components/context-menu`에서 export됩니다. 스타일은 `@tinyrack/ui/components/context-menu.css`를 통해 선택적으로 불러옵니다.

### `TRContextMenu.Root`

제어 또는 비제어 open 상태, 방향, 포커스 순환, 닫기, root disabled 동작을 관리합니다.

### `TRContextMenu.Trigger`

오른쪽 클릭 또는 길게 누를 대상을 정의합니다. Shift+F10과 플랫폼 Context Menu 키를 지원하려면 포커스 가능하게 만드세요.

### `TRContextMenu.Portal`

overlay anatomy를 `document.body` 또는 사용자 지정 container로 옮깁니다. 외부 애니메이션에서 node를 유지하려면 `keepMounted`를 사용하세요.

### `TRContextMenu.Backdrop`

popup 뒤에 투명한 modal 상호작용 layer를 제공합니다.

### `TRContextMenu.Positioner`

root popup은 pointer 또는 keyboard 좌표에, nested popup은 submenu trigger 옆에 배치합니다. 명시적인 `anchor`는 pointer anchor보다 우선합니다.

### `TRContextMenu.Popup`

`menu`를 렌더링하고 keyboard 탐색과 typeahead를 관리하며 제한된 scroll viewport를 제공합니다.

### `TRContextMenu.Arrow`

popup과 anchor 사이에 선택적인 화살표를 렌더링합니다.

### `TRContextMenu.Group`

관련 명령을 묶습니다.

### `TRContextMenu.GroupLabel`

명령 group에 label을 제공합니다.

### `TRContextMenu.Item`

명령을 실행합니다. 필요에 따라 `disabled`, `closeOnClick`, Tinyrack `variant="danger"`를 사용하세요.

### `TRContextMenu.LinkItem`

`href`가 있는 semantic link 명령을 렌더링합니다. 이동할 때 menu도 닫으려면 `closeOnClick`을 설정하세요.

### `TRContextMenu.CheckboxItem`

제어 또는 비제어 checked 상태를 가진 boolean 명령입니다.

### `TRContextMenu.CheckboxItemIndicator`

item이 checked일 때 checkbox 선택 콘텐츠를 렌더링합니다.

### `TRContextMenu.RadioGroup`

상호 배타적인 명령 집합의 선택 값을 관리합니다.

### `TRContextMenu.RadioItem`

radio group의 한 값을 나타냅니다.

### `TRContextMenu.RadioItemIndicator`

item이 선택됐을 때 radio 선택 콘텐츠를 렌더링합니다.

### `TRContextMenu.SubmenuRoot`

nested menu의 open 상태와 Escape 동작을 관리합니다.

### `TRContextMenu.SubmenuTrigger`

pointer 또는 keyboard 탐색으로 nested menu를 엽니다.

### `TRContextMenu.Separator`

명령 group을 시각적으로 구분합니다.

터치 사용자가 오른쪽 클릭을 할 수 없을 때 같은 context-menu action을 dispatch하는 명시적인 버튼을 제공하세요.
