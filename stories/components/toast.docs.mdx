# Toast

Transient status notifications with semantic variants, actions, dismissal, and six viewport positions.

## Contract

| Axis | Values | Default |
| --- | --- | --- |
| `variant` | `neutral`, `info`, `success`, `warning`, `danger` | toast `type` or `neutral` |
| `position` | block start/end × inline start/center/end | `block-end-inline-end` |
| parts | `Provider`, `Portal`, `Viewport`, `Root`, `Title`, `Description`, `Action`, `Close` | compound API |

## Usage

<ComponentExampleTabs
  id="toast-basic"
  title="Managed notification"
  preview={<Toast.Provider><ToastDemo /></Toast.Provider>}
  sources={[{ label: 'React', language: 'tsx', code: String.raw`function ToastButton() {
  const manager = useToastManager();
  return <Button onClick={() => manager.add({
    title: 'Deployment complete',
    description: 'Rack A is healthy.',
    type: 'success',
  })}>Show toast</Button>;
}

<Toast.Provider>
  <ToastButton />
  <Toast.Portal>
    <Toast.Viewport>
      {manager.toasts.map((toast) => (
        <Toast.Root key={toast.id} toast={toast}>...</Toast.Root>
      ))}
    </Toast.Viewport>
  </Toast.Portal>
</Toast.Provider>` }]}
/>

## API

Create or consume a toast manager inside `Provider`; Base UI owns queueing, timers, swipe gestures, focus, and live-region semantics.
