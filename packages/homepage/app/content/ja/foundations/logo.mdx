## 配置

各 surface に lockup を 1 回だけ使い、composition の安定した端に置きます。left-to-right interface では top-left を推奨し、launch、loading、end-frame の文脈では中央配置が適しています。favicon や service avatar など繰り返す compact surface には mark を使います。

製品、project、partner の名前は lockup clear space の外に置きます。新しい logo を拡張・組版せず、**Tinyrack UI** を通常の interface text として書きます。

## アクセシビリティ

logo が link の唯一の content の場合は <TRCode>alt=&quot;Tinyrack&quot;</TRCode> を使い、link に簡潔なアクセシブルな名前を付けます。近くの text がすでに Tinyrack を示し image が冗長になる場合は空の alt attribute を使います。

```html
<a href="/">
  <img src="/brand/tinyrack-lockup.svg" alt="Tinyrack" width="156" height="38" />
</a>
```

rack の図を説明する追加の文言を alt text に入れません。アクセシブルな名前は brand を識別し、page copy が artwork を説明します。
