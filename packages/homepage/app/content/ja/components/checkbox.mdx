## コントラクト

| Axis | Values |
| --- | --- |
| state | unchecked, checked, `indeterminate` |
| availability | enabled, `readOnly`, `disabled` |
| size | `sm`, `md`, `lg` |
| form | `name`, `value`, `uncheckedValue`, `form`, `required`, reset、native validity |

表示ラベルまたはアクセシブルな名前を指定します。フォーカスされた checkbox は Space で切り替わり、Enter では native checkbox と同様に切り替わりません。値をフォーカス可能かつ送信可能に保つには `readOnly`、使用不可にして送信からも除外するには `disabled` を使います。

export function RequiredCheckbox() {
  const [attempted, setAttempted] = useState(false);
  const [checked, setChecked] = useState(false);
  const invalid = attempted && !checked;

  return (
    <TRForm noValidate onSubmit={(event) => { event.preventDefault(); event.currentTarget.checkValidity(); setAttempted(true); }}>
      <TRField.Root invalid={invalid}>
        <TRField.Label>
          <TRCheckbox.Root checked={checked} name="terms" onCheckedChange={setChecked} required>
            <TRCheckbox.Indicator aria-hidden="true">✓</TRCheckbox.Indicator>
          </TRCheckbox.Root>
          I accept the maintenance window.
        </TRField.Label>
        {invalid ? <TRField.Error match>Accept the maintenance window to continue.</TRField.Error> : null}
      </TRField.Root>
      <TRButton type="submit">Continue</TRButton>
    </TRForm>
  );
}` }]}

## API

### `TRCheckbox.Root`

| Prop | Type / デフォルト | 用途 |
| --- | --- | --- |
| `checked` / `defaultChecked` | `boolean` / `undefined`, `false` | controlled または uncontrolled の選択状態 |
| `indeterminate` | `boolean` / `false` | `aria-checked="mixed"` を公開します。ユーザーが mixed 状態を解消したら `onCheckedChange` で解除します。 |
| `onCheckedChange` | `(checked, details) => void` | 変更操作を通知します。controlled consumer は `checked` を更新します。 |
| `disabled` / `readOnly` | `boolean` / `false` | 使用不可、またはフォーカス可能で変更不可にします。 |
| `required` | `boolean` / `false` | native constraint validation に参加します。 |
| `name`, `value`, `uncheckedValue`, `form` | `string` | 外部 form と明示的な off 値を含む native form serialization を設定します。 |
| `inputRef` / `ref` | `Ref<HTMLInputElement>` / `Ref<HTMLElement>` | hidden native input または visual root にアクセスします。 |
| `uiSize` | `'sm' \| 'md' \| 'lg'` / `'md'` | visual control size を設定します。 |
| `render` | Base UI render prop | 動作と状態を維持して root host を置換します。 |

Uncontrolled root は所有 form の reset 時に `defaultChecked` へ戻ります。Controlled root は引き続き `checked` に従います。

### `TRCheckbox.Indicator`

Indicator は checked または indeterminate のときに描画されます。transition styling には `keepMounted`、mixed glyph を実際の状態に追従させるには `render(props, state)` と `state.indeterminate` を使います。Root と Indicator は render callback、関数形式の `className`/`style`、および `data-checked`、`data-unchecked`、`data-indeterminate`、`data-disabled`、`data-readonly`、`data-invalid` 属性を通じて Base UI state を公開します。

`TRCheckboxRoot`、`TRCheckboxIndicator`、`TRCheckboxRootProps`、`TRCheckboxIndicatorProps`、`TRCheckboxUiSize`、`TRCheckboxRootState`、`TRCheckboxIndicatorState` は `@tinyrack/ui/components/checkbox` から export されます。CSS は opt-in です。
