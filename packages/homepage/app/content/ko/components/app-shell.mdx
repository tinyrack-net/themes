## 계약

`Main`은 페이지의 단일 main landmark를 담당합니다. 선택한 breakpoint 이상에서 `Sidebar`는 고정되고 독립적으로 스크롤됩니다. `sidebarMode`로 expanded와 icon rail을 전환하며, `SidebarLabel`은 두 상태 모두에서 탐색 이름을 접근 가능하게 유지합니다. breakpoint 아래에서는 `mobileSidebar="drawer"`가 기존 Base UI TRDrawer 동작을 보존하고, `mobileSidebar="rail"`이 inline 고정 rail을 활성화합니다.

## API

`Root`는 controlled `sidebarMode`, uncontrolled `defaultSidebarMode`(기본 `expanded`), `onSidebarModeChange`를 제공합니다. 안정적인 계약은 `data-sidebar-mode`, `data-mobile-sidebar`, `--tr-app-shell-sidebar-width`, 기본 `4rem`인 `--tr-app-shell-sidebar-rail-width`입니다. 기본 `mobileSidebar="drawer"`는 modal에서 항상 전체 label을 보여주며, `mobileSidebar="rail"`은 Drawer·Backdrop·Close·Trigger 없이 같은 탐색을 inline으로 렌더링합니다. `SidebarToggle`은 `aria-expanded`를 갖는 ghost `sm` IconButton이고 `SidebarLabel`은 rail에서 제거되지 않고 시각적으로만 숨겨집니다.
