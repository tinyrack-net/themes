## コントラクト

| 軸 | 動作 |
| --- | --- |
| 状態 | 入力テキスト、選択値、open 状態は、それぞれ controlled または uncontrolled で使用できます。 |
| 入力構造 | native label または Field.Label → InputGroup → 任意の InputAdornment + Input + Clear + Trigger の順で、Icon は Trigger 内に置きます。 |
| 複数選択構造 | Chips → Value → Chip + ChipRemove を使用し、grid collection は Collection → Row → Item で構成します。 |
| popup 構造 | Portal → 任意の Backdrop + Positioner → Popup → Arrow + Status + List の順で、Group、Indicator、Separator、Collection、Empty を組み合わせます。 |
| 操作 | pointer、矢印キー、Home/End、Enter、Escape、filter、focus 復元、disabled item、read-only root は Base UI が処理します。 |
| form | Root は `name`、`form`、`required`、`disabled`、`readOnly`、object の直列化、native validation、submit、reset をサポートします。 |

## API

| surface | 用途 |
| --- | --- |
| `Root` | item、filter、single/multiple 値、入力テキスト、open 状態、validation、form 直列化、locale、object の比較と文字列変換を管理します。 |
| `InputGroup`, `InputAdornment`, `Input`, `Clear`, `Trigger`, `Icon` | 編集可能な control を構成します。`InputAdornment` は装飾で、`Icon` は `Trigger` 内に置きます。 |
| `Portal`, `Backdrop`, `Positioner`, `Popup`, `Arrow` | dismiss 可能な overlay を mount・配置します。Portal は custom container に対応し、inline 表示では省略できます。 |
| `Status`, `List`, `Group`, `GroupLabel`, `Collection`, `Row`, `Item`, `ItemIndicator`, `Separator`, `Empty` | list/grid collection とアクセシブルな feedback を描画します。 |
| `Chips`, `Value`, `Chip`, `ChipRemove` | 複数の選択値を表示・削除します。 |
| `useFilter`, `useFilteredItems` | locale 対応の `contains`、`startsWith`、`endsWith` predicate を作成し、Root 内で generic な filtered item 配列を読み取ります。 |

各 part には個別の `TRCombobox*` export と props/state type があります。Root action、状態変更と highlight の detail/reason type、`TRComboboxFilter`、`TRComboboxFilterOptions` も export します。

編集可能な `Input` には native `label` または `TRField.Label` を使用します。`TRCombobox.Label` は trigger-only select 構造専用です。Disabled root/item は変更できず、read-only root は focus できますが入力や clear はできません。Object 値は安定した identity を保ち、`{ label, value }` 規約で足りない場合は `itemToStringLabel`、`itemToStringValue`、`isItemEqualToValue` を指定します。

Combobox module は SSR-safe です。Portal は hydration 後に mount され、server/client の controlled 初期値を一致させる必要があります。スタイルは `combobox.css` で別途提供されます。`--tr-combobox-input-background`、`--tr-combobox-content-background`、`--tr-combobox-content-max-height`、`--tr-combobox-content-max-width`、`--tr-combobox-option-min-height` で appearance を調整でき、light/dark の semantic token に fallback します。
