## 핵심 속성

이전과 다음 목적지는 데이터로 전달해요. `renderLink`는 컴포넌트를 라우팅 상태에 묶지 않고 라우터 요소를 끼워 넣어요. 목적지가 하나 이상 있을 때만 landmark를 렌더링해요.

| 속성 | 설명 |
| --- | --- |
| 방향 | 이전 링크는 `rel="prev"`, 다음 링크는 `rel="next"`를 노출해요 |
| 사용 가능 상태 | 없는 목적지는 생략하고, 보이되 쓸 수 없게 하려면 `disabled`를 설정해요 |
| 라우팅 | `renderLink(destination, direction)`는 anchor를 중첩하지 않고 라우터 링크를 합성해요 |
| 이벤트 | `onNavigate(destination, direction)`는 활성 링크에서만 실행돼요 |

## API

`previous`와 `next`는 `{ path, title, label?, description?, disabled? }`를 받아요. 방향 텍스트와 접근 가능한 이름은 `previousLabel`, `nextLabel`, `previousAriaLabel`, `nextAriaLabel`, landmark `label`로 지역화할 수 있어요. `renderLink`는 목적지와 `previous | next` 방향을 받아 React Router 같은 클라이언트 라우터를 연결해요. `onNavigate`도 활성 링크가 실행된 뒤 같은 값을 받아요.

루트는 네이티브 `nav` 속성, `className`, `style`, 이벤트와 React 19 `ref`를 보존해요. 긴 콘텐츠는 카드 안에서 줄바꿈되고 반응형 breakpoint에서 1열에서 2열로 전환돼요. 스타일은 Tinyrack semantic color, spacing, typography, focus, opacity, border 및 motion token을 사용해요.
