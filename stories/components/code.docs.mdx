# Code

Inline code styling for commands, paths, environment variables, and compact API names.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `tr-code` on `<code>` | `<Code>` | Required component surface | Use for inline commands, paths, tokens, and API names. |
| Behavior | Native inline text flow | `children` | Inline content; wraps at long strings | Move multiline examples to CodeBlock. |
| Native | Native `<code>` attributes | `HTMLAttributes<HTMLElement>` | Passed through | Keep native code semantics within the surrounding sentence. |

## Guidance

<ComponentGuidance
  whenToUse={[
    'Identify a short command, file path, environment variable, token, or API name within prose.',
    'Distinguish literal values from the sentence that explains how to use them.',
  ]}
  avoid={[
    'Do not use inline code for multiline commands, logs, or source examples; use CodeBlock instead.',
    'Do not apply code styling only for visual emphasis when the text is not a literal technical value.',
  ]}
  accessibility={[
    'Keep the native code element so assistive technology and copy-and-paste preserve the literal value.',
    'Write punctuation and instructions outside the Code component, and verify long paths wrap without obscuring nearby prose.',
  ]}
/>

## API

| Prop | Values | Note |
| --- | --- | --- |
| `className` | `string` | Merged with `tr-code`. |
| native code props | `HTMLAttributes<HTMLElement>` | Passed through to `<code>`. |
