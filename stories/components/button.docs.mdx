# Button

Native button behavior with Tinyrack size, variant, and appearance axes.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `tr-btn` on `<button>` | `<Button>` | Required text-button surface | Use for commands with visible text. |
| Component | `tr-icon-btn` on `<button>` | `<IconButton>` | Icon-only button surface | Provide an accessible name for every icon action. |
| Size | `data-size` | `size` | `sm`, `md`, `lg`; default `md` | Use `md` for routine actions and change size only for layout density. |
| Variant | `data-variant` | `variant` | `neutral`, `primary`, `danger`; default `neutral` | Reserve `primary` for the main action and `danger` for destructive intent. |
| Variant | `data-appearance` | `appearance` | `solid`, `outline`, `ghost`; default `solid` | Tune emphasis without changing action semantics. |
| Behavior | `aria-label` | `label` on `IconButton` | Required for `IconButton` | Name the command, not the icon artwork. |
| State | Native `disabled` attribute | `disabled` | Default `false` | Use native disabled behavior when an action is unavailable. |
| Native | Native `<button>` attributes | `ButtonHTMLAttributes<HTMLButtonElement>` | Passed through; `type` defaults to `button` | Set `type="submit"` explicitly for form submission. |

## Usage

  id="button-basic"
  title="Basic action"
  description="Use a native button contract for commands, with an icon-only button only when the label names the action."

<Button size="md" variant="primary" appearance="solid">
  Deploy
</Button>;

<IconButton label="Refresh rack">
  <RefreshCw aria-hidden="true" size={18} />
</IconButton>;`,
    },
    {
      label: 'HTML',
      language: 'html',
      code: String.raw`<button
  class="tr-btn"
  data-size="md"
  data-variant="primary"
  data-appearance="solid"
>
  Deploy
</button>

<button class="tr-icon-btn" aria-label="Refresh rack">
  <!-- icon svg -->
</button>`,
    },
  ]}
/>

## Guidance

<ComponentGuidance
  whenToUse={[
    'Trigger an immediate command such as deploy, refresh, retry, or remove.',
    'Use IconButton for familiar compact actions when the visible layout cannot accommodate text.',
  ]}
  avoid={[
    'Do not use a button for navigation; use Link so browser link behavior remains available.',
    'Do not show several primary buttons in one action group or use danger styling for harmless actions.',
  ]}
  accessibility={[
    'Keep native button semantics, keyboard focus, and disabled behavior; set type="submit" only for intentional form submission.',
    'Give IconButton a concise action label such as "Refresh rack" and mark a decorative icon aria-hidden="true".',
  ]}
/>

## API

| Prop | Values | Note |
| --- | --- | --- |
| `size` | `sm`, `md`, `lg` | Default `md`; sets `data-size`. |
| `variant` | `neutral`, `primary`, `danger` | Default `neutral`; sets `data-variant`. |
| `appearance` | `solid`, `outline`, `ghost` | Default `solid`; sets `data-appearance`. |
| `label` | `string` | Required for `IconButton` accessible names. |
| Icon child | `ReactNode` | Use `aria-hidden="true"` when `label` names the action. |
| `disabled` | `boolean` | Native disabled state. |
| `type` | `button`, `submit`, `reset` | Default `button`. |
| native button props | `ButtonHTMLAttributes<HTMLButtonElement>` | Passed through to `<button>`. |
