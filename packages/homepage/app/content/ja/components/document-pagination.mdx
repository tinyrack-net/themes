## コントラクト

前後の遷移先はデータとして渡します。`renderLink` はコンポーネントをルーティング状態に結合せず、ルーター要素を注入します。遷移先が 1 つ以上ある場合だけ landmark を描画します。

| 軸 | コントラクト |
| --- | --- |
| 方向 | 前のリンクは `rel="prev"`、次のリンクは `rel="next"` を公開します |
| 利用可否 | 存在しない遷移先は省略し、表示したまま無効にする場合は `disabled` を設定します |
| ルーティング | `renderLink(destination, direction)` は anchor を入れ子にせずルーターリンクを合成します |
| イベント | `onNavigate(destination, direction)` は有効なリンクでのみ実行されます |

## API

`previous` と `next` は `{ path, title, label?, description?, disabled? }` を受け取ります。方向表示とアクセシブル名は `previousLabel`、`nextLabel`、`previousAriaLabel`、`nextAriaLabel`、landmark の `label` でローカライズできます。`renderLink` は遷移先と `previous | next` 方向を受け取り、React Router などに利用できます。`onNavigate` も有効なリンクの操作後に同じ値を受け取ります。

root はネイティブ `nav` 属性、`className`、`style`、イベント、React 19 の `ref` を保持します。長い内容はカード内で折り返され、レスポンシブ breakpoint で 1 列から 2 列に切り替わります。スタイルは Tinyrack の semantic color、spacing、typography、focus、opacity、border、motion token を使用します。
