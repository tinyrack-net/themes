## スケール

ブレークポイントトークンは、レイアウトで利用できるスペースが増える位置を定義します。モバイルファーストで構成し、コンテンツに必要な時点でだけ <TRCode>{'*-min'}</TRCode> 規則を追加します。

<BreakpointReference locale="ja" />

## 選択基準

- 特定の端末が始まる位置ではなく、コンテンツがタスクを実行しにくくなる位置にブレークポイントを追加します。
- ページシェルとアプリケーション領域にはビューポートブレークポイントを使います。再利用コンポーネントの適応は、そのコンポーネントやコンテナ内で処理します。
- 視覚的なリフローには CSS を優先します。動作、データ読み込み、操作モードまで変える必要がある場合だけ TypeScript で共通値を読みます。

## TypeScript

```ts
import { tinyrackBreakpoints } from '@tinyrack/ui/core';

const desktopQuery = `(width >= ${tinyrackBreakpoints.lg})`;
```

## CSS と Tailwind

Tinyrack CSS は Tailwind の <TRCode>@variant</TRCode> ディレクティブを使います。<TRCode>@tinyrack/ui/core.css</TRCode> をコンポーネントスタイルより先に読み込むと、Tailwind が共通ブレークポイントスケールを解決します。

```css
@variant md {
  /* Tinyrack source authoring */
}
```

Tailwind は <TRCode>--breakpoint-xs</TRCode> から <TRCode>--breakpoint-xl</TRCode> まで同じ値を使用します。

```tsx
<section className="grid gap-4 md:grid-cols-2 max-md:grid-cols-1" />
```

モバイルファーストの規則には、生成された <TRCode>xs:</TRCode> から <TRCode>xl:</TRCode> までのバリアントを使います。しきい値より小さい範囲には <TRCode>max-md:</TRCode> のようなバリアントを使ってください。CSS の基準は <TRCode>--breakpoint-*</TRCode>、TypeScript の基準は <TRCode>tinyrackBreakpoints</TRCode> です。

## レスポンシブ品質確認

変更したレイアウトは、ブレークポイントの直前、境界上、直後で確認します。390px のモバイルビューポート、200% ズーム、長い翻訳ラベル、キーボードのフォーカス順序を含め、レイアウトが折りたたまれても必須アクションが消えないことを検証します。
