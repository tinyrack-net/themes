## API

highlighting은 hydration 후에 실행됩니다. 원본 source는 SSR-safe하며 Shiki가 로드되기 전에도 표시됩니다. 강조된 block은 `tinyrack-light`에서 `github-light-high-contrast`, `tinyrack-dark`에서 `github-dark-high-contrast`를 자동으로 사용합니다. `code`나 `language`를 변경하면 오래된 작업을 취소해 이전 async 결과가 현재 source를 덮어쓰지 못하게 합니다.

| Prop | 타입 | 기본값 | 설명 |
| --- | --- | --- | --- |
| `code` | `string` | 필수 | semantic `<pre><code>` 안에 표시할 source입니다. |
| `language` | `BundledLanguage` | `undefined` | 점진적 Shiki highlighting을 활성화합니다. 생략하면 plain text입니다. |
| `wrap` | `boolean` | `false` | 가로 overflow 대신 긴 줄을 줄바꿈합니다. |
| `ref` | `Ref<HTMLPreElement>` | — | native `pre` 요소를 참조합니다. |
| `className`, `style` | native `pre` props | — | consumer class, inline style, component token override를 적용합니다. |

그 밖의 native `pre` attribute와 event도 전달됩니다. 주변 문맥이 source를 식별하지 못하면 `aria-label`을 추가하세요. highlighting은 접근 가능한 원문을 바꾸지 않으며 language 로딩이 실패해도 SSR fallback은 유지됩니다.

## 스타일

| Token | 용도 |
| --- | --- |
| `--tr-code-block-background` | highlighted block을 포함한 surface |
| `--tr-code-block-border` | border 색상 |
| `--tr-code-block-border-width` | border 너비 |
| `--tr-code-block-color` | fallback source 색상 |
| `--tr-code-block-font-family` | monospace font family |
| `--tr-code-block-font-size` | source 크기 |
| `--tr-code-block-line-height` | source line height |
| `--tr-code-block-padding-x` | inline padding |
| `--tr-code-block-padding-y` | block padding |
| `--tr-code-block-radius` | corner radius |

`wrap={false}`이면 긴 줄을 가로 overflow로 표시합니다. `wrap`이면 긴 token도 사용 가능한 너비 안에서 줄바꿈하며 두 모드 모두 parent 경계 안에 머뭅니다.
