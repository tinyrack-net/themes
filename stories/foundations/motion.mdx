# Motion

Motion should make a state change easier to follow, then get out of the way.

## Principle

Choose motion by the kind of change, not by preference or decoration.

### Reduced motion

Reduced motion removes decorative travel, not the state change or its meaning.

<ReducedMotionPreview />

## Implementation

Components consume semantic duration and easing variables. Keep one transition at the component boundary, and let reduced-motion preferences remove decorative travel.

```css
.rack-layer {
  transition:
    opacity var(--tinyrack-duration-normal) var(--tinyrack-ease-out),
    transform var(--tinyrack-duration-normal) var(--tinyrack-ease-out);
}

@media (prefers-reduced-motion: reduce) {
  .rack-layer {
    transition-duration: 1ms;
    transform: none;
  }
}
```

Use `--tinyrack-duration-*` for system-wide timing and component-level `--tr-*` variables only for a deliberate local exception.
