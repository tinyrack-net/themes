## コントラクト

`Main` はページ内で唯一の main landmark を担当します。選択した breakpoint 以上では `Sidebar` が固定され、独立してスクロールします。`sidebarMode` で expanded と icon rail を切り替え、`SidebarLabel` はどちらの状態でもナビゲーション名をアクセシブルに保ちます。breakpoint 未満では `mobileSidebar="drawer"` が従来の Base UI TRDrawer を維持し、`mobileSidebar="rail"` が inline 固定 rail を有効にします。

## API

`Root` は `breakpoint`（`sm` または `lg`）、`layout`（`header-first` または `sidebar-first`）、`mobileSidebar`（`drawer` または `rail`）を受け取ります。Drawer state は `open`、`defaultOpen`、`onOpenChange`、sidebar width は `sidebarMode`、`defaultSidebarMode`、`onSidebarModeChange` で制御します。contained preview や embedded app では `portalContainer`、popup のスタイルには `drawerPopupClassName` を使います。

`Header` は header landmark を描画します。`Sidebar` は独立してスクロールする `aside` で、mobile dialog にもアクセシブルな名前を渡します。`Trigger` は mobile drawer を開き、`Close` は閉じます。`SidebarToggle` は focus を維持して `aria-expanded` を公開しながら expanded/rail mode を切り替えます。`SidebarLabel` は rail でも削除されず視覚的に隠れます。`Main` は main landmark と Base UI の `render` コントラクトを提供します。

安定したスタイルサーフェスは `--tr-app-shell-background`、`--tr-app-shell-sidebar-background`、`--tr-app-shell-sidebar-width`、`--tr-app-shell-sidebar-rail-width`、`--tr-app-shell-drawer-block-size` です。Root state は `data-breakpoint`、`data-layout`、`data-mobile-sidebar`、`data-sidebar-mode` に反映されます。

SSR では module 評価時に browser global を読まず、desktop/static structure を描画してから hydration します。その後 client が選択した media query を適用するため、application render 内で `window` に応じて child markup を変えないでください。Modal drawer は focus を閉じ込め、Escape または backdrop で閉じ、focus を `Trigger` に戻し、open 中の page scroll を防ぎます。
