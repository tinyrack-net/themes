# AppShell

A responsive application frame with a static desktop sidebar and modal mobile navigation.

## Contract

`Main` owns the page's single main landmark. `Sidebar` is sticky with its own scroll position at and above the selected breakpoint, so document scrolling does not move or reset its navigation. Below the breakpoint it uses Base UI Drawer for focus, dismissal, scroll lock, and focus return. The `open` state only affects that mobile Drawer; it does not hide the desktop sidebar.

## API

`Root` accepts controlled or uncontrolled open state, `sm` or `lg` breakpoints, and `header-first` or `sidebar-first` layout. `Trigger` and `Close` default to the `lg` IconButton size and require an accessible name; an explicit `size` still wins. On desktop, `Sidebar` stays sticky within the viewport while its plain `ScrollArea` scrolls independently from the document. On mobile it becomes a modal Drawer. Put visual padding on a child inside `Sidebar`, not on `Sidebar` itself, so the scrollbar stays aligned with the outer edge.
