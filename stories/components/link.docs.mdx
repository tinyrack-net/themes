# Link

Native anchor behavior with Tinyrack text-link styling.

## Contract

| Category | CSS / HTML | React | Values & default | Guidance |
| --- | --- | --- | --- | --- |
| Component | `tr-link` on `<a>` | `<Link>` | Required component surface | Use native anchors for navigation. |
| Variant | `data-variant` | `variant` | `neutral`, `primary`, `danger`; default `primary` | Use `primary` for routine navigation and `danger` only for risky destinations. |
| Behavior | `data-underline` | `underline` | `none`, `hover`, `always`; default `hover` | Preserve a recognizable link affordance in prose. |
| Native | `href` and native `<a>` attributes | `AnchorHTMLAttributes<HTMLAnchorElement>` | Passed through; `href` is required for navigation | Keep browser open, copy, and new-tab behavior. |

  ]}

## Guidance

<ComponentGuidance
  whenToUse={[
    'Navigate to another page, route, file, or external resource while preserving browser link behavior.',
    'Map markdown links in React or Astro MDX so authored prose uses the same link treatment.',
  ]}
  avoid={[
    'Do not use Link for commands that change state; use Button instead.',
    'Do not remove the underline in dense prose unless surrounding context makes every link unmistakable.',
  ]}
  accessibility={[
    'Use destination-specific link text such as "rack inventory" instead of generic text such as "click here".',
    'Keep a real href and a visible keyboard focus indicator; if a link opens a new context, communicate that in its text or nearby copy.',
  ]}
/>

## API

| Prop | Values | Note |
| --- | --- | --- |
| `variant` | `neutral`, `primary`, `danger` | Default `primary`; sets `data-variant`. |
| `underline` | `none`, `hover`, `always` | Default `hover`; sets `data-underline`. |
| `href` | `string` | Native anchor href. |
| `className` | `string` | Merged with `tr-link`. |
| native anchor props | `AnchorHTMLAttributes<HTMLAnchorElement>` | Passed through to `<a>`. |
